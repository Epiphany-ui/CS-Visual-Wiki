CS-Visual-learn 前端配套完整接口文档（适配全页面/全业务链路）
文档说明
1. 本文档基于项目22个前端界面、全页面跳转逻辑、完整业务链路梳理，包含前端所有需要调用的后端接口，无遗漏、无冗余。
2. 所有接口为前端专供，返回字段完全适配页面渲染、交互跳转、状态更新，可直接支撑学习、创作、社区、个人中心全流程落地。
3. 接口统一规范：请求方式（GET/POST/PUT/DELETE）、接口路径、请求参数、响应参数、业务场景一一对应。
4. 基础域名统一前缀：/api/v1
5. 公共返回字段：code(状态码)、msg(提示信息)、data(业务数据)
一、登录注册模块接口（适配页面：登录页）
1.1 用户登录接口
请求方式：POST
接口路径：/user/login
业务场景：登录页用户账号密码登录，获取登录凭证与用户基础信息
请求参数：
username：string，用户名/账号
password：string，密码
响应参数：
token：string，登录令牌
userId：number，用户ID
nickname：string，用户昵称
avatar：string，头像地址
1.2 用户注册接口
请求方式：POST
接口路径：/user/register
业务场景：登录页新用户注册
请求参数：username、password、nickname
响应参数：success状态、提示信息、userId
1.3 获取当前登录用户信息
请求方式：GET
接口路径：/user/info
业务场景：全局页面校验登录状态、展示用户信息
响应参数：用户基础信息、登录状态、创作者标识
1.4 用户退出登录
请求方式：POST
接口路径：/user/logout
业务场景：全局退出登录，清空登录态
二、首页模块接口（适配页面：首页）
2.1 获取首页热门轮播动画
请求方式：GET
接口路径：/home/carousel
业务场景：首页渲染热门动画轮播
响应参数：作品ID、封面图、标题、播放量、作者信息
2.2 获取首页知识点分类列表
请求方式：GET
接口路径：/home/category
业务场景：首页展示学科分类导航（算法/数学/数据结构等）
响应参数：分类ID、分类名称、分类图标、知识点数量
2.3 获取首页精选作品、最新作品
请求方式：GET
接口路径：/home/work/list
业务场景：首页渲染精选、最新公开作品
请求参数：type（精选/最新）、page、size
响应参数：作品列表、分页总数
三、搜索模块接口（适配页面：全局搜索结果页）
3.1 全局统一搜索接口
请求方式：GET
接口路径：/search/all
业务场景：首页搜索框全局搜索（知识点/作品/模板/用户）
请求参数：keyword、page、size
响应参数：
knowledgeList：知识点词条列表
workList：动画作品列表
templateList：模板列表
userList：创作者列表
四、百科知识模块接口（适配页面：百科分类列表页、百科详情页、动画播放器）
4.1 获取百科分类知识点列表
请求方式：GET
接口路径：/knowledge/category/list
业务场景：百科分类列表页渲染对应分类下所有知识点
请求参数：categoryId、difficulty（难度筛选）、page、size
响应参数：知识点ID、标题、简介、难度、配套动画数量
4.2 获取知识点百科详情
请求方式：GET
接口路径：/knowledge/detail
业务场景：百科详情页渲染完整词条内容
请求参数：knowledgeId
响应参数：定义、核心原理、过程示例、复杂度、常见误区、来源、难度
4.3 获取知识点配套动画列表
请求方式：GET
接口路径：/knowledge/animation/list
业务场景：百科详情页加载内嵌动画播放器资源
请求参数：knowledgeId
响应参数：动画ID、视频地址、标题、时长、播放次数
4.4 获取相关推荐知识点
请求方式：GET
接口路径：/knowledge/recommend
业务场景：百科详情页推送前置、关联、拓展知识点
请求参数：knowledgeId
响应参数：关联知识点列表
4.5 知识点收藏/取消收藏
请求方式：POST
接口路径：/knowledge/collect
业务场景：用户收藏/取消收藏知识点
请求参数：knowledgeId、isCollect（true/false）
4.6 学习打卡 & 进度记录
请求方式：POST
接口路径：/study/checkin
业务场景：用户学习知识点后打卡，更新学习进度

五、模板创作模块接口（适配页面：模板列表页、模板参数配置页、渲染加载页、成品预览页）
5.1 获取模板分类列表
请求方式：GET
接口路径：/template/list
业务场景：模板库列表页渲染所有可用创作模板
请求参数：category、page、size
响应参数：模板ID、模板名称、封面、简介、使用次数、参数配置项
5.2 获取模板详情及参数配置Schema
请求方式：GET
接口路径：/template/detail
业务场景：模板参数配置页加载模板介绍、可配置参数
请求参数：templateId
响应参数：模板详情、参数表单配置、默认参数、预览视频
5.3 模板参数生成动画（核心渲染接口）
请求方式：POST
接口路径：/template/generate
业务场景：用户修改参数后，一键生成动画，支撑渲染加载态、成品预览页
请求参数：templateId、customParams（用户自定义参数对象）
响应参数：taskId（渲染任务ID）、renderStatus、previewUrl、manimCode
5.4 查询渲染任务状态
请求方式：GET
接口路径：/render/status
业务场景：轮询获取动画渲染进度，更新页面加载状态
请求参数：taskId
响应参数：进度百分比、状态（渲染中/成功/失败）、成品资源地址
5.5 模板作品保存/发布
请求方式：POST
接口路径：/template/work/save
业务场景：成品预览页保存私密作品、发布公开作品
请求参数：taskId、workTitle、workDesc、isPublic（公开状态）
响应参数：workId、作品预览地址
5.6 作品资源导出接口
请求方式：GET
接口路径：/work/export
业务场景：成品预览页导出MP4/GIF/源码
请求参数：workId、exportType（mp4/gif/code）
响应参数：下载链接

CS-Visual-learn 产品原型 · 页面交互流程图（文字箭头版）
说明：以下为纯页面跳转交互逻辑，对标产品原型交互，每条流程独立互不干扰，全部采用「页面→操作→下一级页面」箭头流程，无技术内容，可直接用于原型梳理、前端开发、需求对接。
流程一：新用户登录 & 首页浏览流程
登录页面 →【登录/注册】→ 首页（发现页）
首页 →【点击搜索框】→ 全局搜索结果页
首页 →【点击知识点分类】→ 百科分类列表页
首页 →【点击精选动画】→ 作品详情页
首页 →【点击快速创作】→ 创作沙箱页
流程二：知识学习主线流程（核心学习链路）
首页 / 搜索页 →【进入知识点词条】→ 百科详情页
百科详情页 →【播放动画】→ 内嵌动画播放器
百科详情页 →【点击相关知识点】→ 新百科详情页
百科详情页 →【收藏】→ 存入个人收藏（无页面跳转）
百科详情页 →【学习打卡】→ 学习数据更新（无页面跳转）
流程三：零代码模板创作流程（小白用户）
首页导航 →【模板库】→ 模板列表页
模板列表页 →【选择对应模板】→ 模板详情 & 参数配置页
参数配置页 →【修改参数/预览】→ 实时刷新动画预览
参数配置页 →【一键生成动画】→ 渲染加载中状态 → 成品预览页
成品预览页 →【保存作品】→ 个人作品列表页
成品预览页 →【发布公开】→ 画廊/社区公开作品页
成品预览页 →【深度编辑】→ 跳转创作沙箱页（带入当前代码）
成品预览页 →【导出视频/GIF】→ 本地下载
流程四：AI 沙箱自定义创作流程（进阶用户）
首页导航 / 快速入口 →【创作沙箱】→ 沙箱主界面（对话+代码+预览三栏）
沙箱页 →【输入自然语言需求】→ AI生成代码 → 自动渲染动画预览
沙箱页 →【手动修改代码】→ 手动渲染 → 刷新预览
沙箱页 →【渲染报错】→ AI自动修复代码 → 重新预览
沙箱页 →【版本回退】→ 切换历史代码版本 + 对应动画
沙箱页 →【保存】→ 存入个人草稿/作品
沙箱页 →【发布】→ 作品发布配置页 → 公开至画廊
流程五：社区浏览 & 二次创作 Fork 流程
首页导航 →【画廊/社区】→ 作品列表/排行榜页
社区列表页 →【点击作品】→ 作品详情页
作品详情页 →【点赞/收藏/评论】→ 实时数据更新（无跳转）
作品详情页 →【Fork 二次创作】→ 自动复制代码 → 跳转沙箱编辑页
沙箱编辑页（Fork来源）→【修改创作】→ 重新渲染 → 发布新作品（带溯源）
作品详情页 →【点击作者头像】→ 创作者个人主页
流程六：个人中心全页面流转流程
首页右上角 →【我的】→ 个人中心主页
个人中心 →【我的作品】→ 个人作品列表页（私密/公开）
个人中心 →【我的收藏】→ 收藏列表（词条+模板+作品）
个人中心 →【浏览历史】→ 历史记录列表
个人中心 →【学习数据】→ 学习统计 & 打卡记录页
个人中心 →【创作者数据】→ 作品数据统计页
个人中心 →【账号设置】→ 资料编辑页
补充：全局通用跳转逻辑
任意页面 →【顶部导航栏】→ 可随时跳转：首页 / 百科 / 模板库 / 画廊 / 社区 / 个人中心
任意作品页/沙箱页 →【返回】→ 回到上一级页面

一、产品概述
1.1 产品定位
CS-Visual-Learn 是一款面向计算机科学与数学领域的交互式知识可视化学习平台。基于 Manim 动画引擎与大语言模型技术，将教材中抽象的算法、数据结构、数学定理转化为直观可交互的动画，同时提供零代码 / 低代码的动画创作能力，打造 "学习 - 理解 - 创作 - 分享" 的完整闭环。
1.2 核心价值主张
对学习者：告别 "看书就懂，做题就废"，通过动画直观理解抽象概念，降低 CS / 数学的学习门槛
对创作者：不需要掌握复杂的 Manim 语法，通过自然语言或模板参数，快速生成专业级教学动画
对教育者：一键生成可视化教学素材，降低课件制作成本

1.4 产品愿景
成为 CS / 数学领域的 "可视化维基百科"，让每一个抽象概念都有对应的直观动画，让知识看得见。

四、核心业务流程
4.1 学习流程（普通用户最常用）
plaintext
用户进入首页 → 搜索知识点 → 阅读百科词条 → 播放配套动画
→ 拖动进度条反复观看 → 学习相关词条 → （可选）沙箱中调试参数 → 打卡完成
4.2 零代码创作流程（普通用户）
plaintext
进入模板库 → 选择模板 → 填写参数 → 点击生成 → 预览调整
→ 导出/保存作品
4.3 AI 创作流程（进阶用户）
plaintext
进入沙箱 → 输入自然语言需求 → AI生成代码 → 渲染预览
→ AI自动修复错误/多轮修改 → 手动调整代码 → 保存发布
4.4 二次创作流程
plaintext
画廊看到作品 → Fork到自己沙箱 → 修改代码/参数 → 重新渲染
→ 发布新作品（自动标注来源）
五、核心技术能力
AI 代码生成引擎：RAG 增强的 Manim 代码生成，多轮对话修改，自动错误修复
Manim 渲染引擎：异步队列渲染，多质量支持，缓存机制，资源隔离
RAG 知识库系统：百科 + 优质代码向量检索，增量更新，提升生成准确率
参数化模板引擎：占位符替换，参数校验，零代码生成
异步任务调度：任务状态管理，失败重试，进度推送，优先级队列
六、核心设计原则
学习优先，创作其次：首先是学习工具，其次才是创作工具
零代码优先：能用模板解决的不让用户写代码，能用自然语言解决的不让用户学语法
内容质量优先：官方内容严格把关，社区内容靠推荐算法筛选
渐进式复杂度：看动画→调参数→改模板→写代码，难度逐步提升
核心引擎零修改：ai_engine.py 保持稳定，新功能在外层封装
