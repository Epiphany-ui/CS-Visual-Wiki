# 红黑树

## 一、定义
红黑树（Red-Black Tree）是一种自平衡的二叉查找树（BST），它在每个节点上增加一个存储位表示节点的颜色（红色或黑色），通过对任何一条从根到叶子的路径上各节点着色方式的约束，确保树的高度近似平衡。红黑树满足以下五条性质：

1. 每个节点要么是红色，要么是黑色。
2. 根节点是黑色。
3. 每个叶子节点（NIL节点，即空节点）都是黑色。
4. 如果一个节点是红色，则它的两个子节点都是黑色（即不能出现连续的红色节点）。
5. 对于每个节点，从该节点到其所有后代叶子节点的简单路径上，均包含相同数目的黑色节点（称为“黑高”）。

这些性质共同保证了红黑树的高度最多为 \(2 \log_2(n+1)\)，其中 \(n\) 为内部节点数。

## 二、核心原理
红黑树的核心思想是通过颜色约束和旋转操作来维持平衡，避免二叉查找树退化为链表。其工作机制基于以下要点：

1. **颜色标记与平衡约束**：红色节点用于吸收插入或删除操作带来的不平衡，黑色节点则维护路径上的黑高一致性。性质4和5共同限制了树的高度，确保最坏情况下查找路径长度不超过最优路径的两倍。

2. **旋转操作**：包括左旋和右旋，用于调整树的结构而不破坏二叉查找树的有序性。旋转操作在 \(O(1)\) 时间内完成，是维持红黑树平衡的基本工具。

3. **插入修复**：插入新节点（初始为红色）后，若其父节点为红色，则违反性质4。通过重新着色和旋转（最多两次旋转）进行修复，修复过程沿树向上进行，最多 \(O(\log n)\) 步。

4. **删除修复**：删除节点后，若导致黑高失衡（即删除的节点为黑色），则通过“双黑”概念处理。修复过程涉及兄弟节点的颜色和子节点结构，通过旋转和着色恢复性质，最多需要三次旋转。

5. **与AVL树的区别**：红黑树允许左右子树高度差最多一倍（而非AVL的严格平衡），因此插入和删除操作的平均旋转次数更少，适用于频繁修改的场景。

## 三、过程/示例
以插入序列 \([10, 20, 30, 15]\) 为例，演示红黑树的构建过程（NIL叶子节点省略，实际存在）：

**初始状态**：空树。

**步骤1：插入10**  
10作为根节点，根据性质2，将其着色为黑色。  
树结构：`10(黑)`

**步骤2：插入20**  
20 > 10，作为10的右子节点，初始为红色。  
树结构：`10(黑) -> 右: 20(红)`  
满足所有性质，无需调整。

**步骤3：插入30**  
30 > 20，作为20的右子节点，初始为红色。此时出现连续红色（20红，30红），违反性质4。  
修复过程：  
- 父节点20为红色，叔节点（10的左子）为NIL（黑色），祖父节点10为黑色。  
- 执行左旋：以10为支点左旋，20成为新根，10成为20的左子。  
- 重新着色：20变为黑色，10变为红色。  
结果树：`20(黑) -> 左: 10(红), 右: 30(红)`  
满足所有性质。

**步骤4：插入15**  
15 > 10且 < 20，作为10的右子节点，初始为红色。此时出现连续红色（10红，15红），违反性质4。  
修复过程：  
- 父节点10为红色，叔节点30为红色，祖父节点20为黑色。  
- 将父节点10和叔节点30变为黑色，祖父节点20变为红色。  
- 由于20是根节点，根据性质2，将其重新着色为黑色。  
结果树：`20(黑) -> 左: 10(黑), 右: 30(黑)；10(黑) -> 右: 15(红)`  
所有性质满足，插入完成。

## 四、复杂度/性质分析
- **时间复杂度**：
  - 查找：\(O(\log n)\)（最好、最坏、平均），因为树高始终为 \(O(\log n)\)。
  - 插入：\(O(\log n)\)，包括查找位置 \(O(\log n)\) 和修复操作（最多两次旋转，\(O(1)\)）。
  - 删除：\(O(\log n)\)，包括查找节点 \(O(\log n)\) 和修复操作（最多三次旋转，\(O(1)\)）。
- **空间复杂度**：\(O(n)\)，每个节点额外存储颜色信息（1位），与节点数线性相关。
- **稳定性**：红黑树不是稳定的排序结构，因为旋转操作可能改变相等元素的相对顺序（若允许重复键）。
- **收敛性**：插入和删除修复过程均沿树向上进行，每次操作后问题规模缩小，保证在 \(O(\log n)\) 步内收敛。
- **与AVL树对比**：红黑树插入/删除的平均旋转次数更少（AVL需严格平衡，旋转更频繁），但查找效率略低（AVL树高更小）。红黑树更适合写操作频繁的场景。

## 五、常见误区
1. **混淆红黑树与AVL树**：认为红黑树是严格平衡的。实际上红黑树是“近似平衡”（黑高平衡），允许左右子树高度差最多一倍，而AVL树要求左右子树高度差不超过1。
2. **忽略NIL叶子节点**：误认为红黑树的叶子节点是普通节点。红黑树中所有叶子节点都是黑色的NIL节点，性质5中的路径计数包含这些NIL节点。
3. **插入时直接着色为黑色**：新插入节点必须为红色，否则会直接破坏性质5（增加黑高），导致更复杂的修复。
4. **删除时忽略“双黑”概念**：删除黑色节点后，其位置会留下一个“双黑”节点（表示该路径缺少一个黑色），需要通过兄弟节点的颜色和子节点结构进行修复，而非简单重新着色。
5. **认为旋转次数与树高成正比**：红黑树的插入和删除修复中，旋转次数是常数（最多2-3次），而重新着色可能沿路径进行 \(O(\log n)\) 次，但总时间复杂度仍为 \(O(\log n)\)。
