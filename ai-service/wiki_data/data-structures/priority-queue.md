# 优先队列

## 一、定义

优先队列（Priority Queue）是一种抽象数据类型（ADT），其每个元素关联一个优先级值，支持的核心操作为：插入（Insert）一个元素，以及取出（Extract-Max / Extract-Min）当前优先级最高（或最低）的元素。与普通队列的先进先出（FIFO）不同，优先队列的出队顺序由元素的优先级决定。

形式上，设优先队列 `Q` 存储元素 `x` 及其优先级 `p(x)`，则其基本操作可描述为：

- `Insert(Q, x, p)`：将元素 `x` 以优先级 `p` 插入 `Q`。
- `Extract-Max(Q)`：返回并删除 `Q` 中优先级最大的元素。
- `Peek-Max(Q)`（或 `Maximum(Q)`）：返回但不删除优先级最大的元素。
- `Increase-Key(Q, x, new_p)`：将元素 `x` 的优先级提升为 `new_p`（假设 `new_p ≥ 当前优先级`）。

优先队列通常使用**二叉堆（Binary Heap）**实现，但也可用平衡二叉搜索树、斐波那契堆等数据结构实现。

## 二、核心原理

1. **基于完全二叉树的结构**：二叉堆是一棵完全二叉树，所有层除最后一层外完全填满，且最后一层节点尽可能靠左。这种结构使得堆可以用数组高效存储：对于下标 `i` 的节点，其左子节点下标为 `2i`，右子节点为 `2i+1`，父节点为 `⌊i/2⌋`。

2. **堆序性质（Heap Property）**：对于最大堆，每个节点的值都大于或等于其子节点的值；对于最小堆，每个节点的值都小于或等于其子节点的值。这一性质保证了根节点始终是全局最大（或最小）元素。

3. **上浮（Swim / Sift-Up）与下沉（Sink / Sift-Down）**：插入操作将新元素放在数组末尾，然后通过“上浮”操作将其与父节点比较并交换，直到恢复堆序。取出操作将根节点与末尾元素交换，删除原根节点，然后对新的根节点执行“下沉”操作，与较大的子节点交换，直到堆序恢复。

4. **动态调整**：`Increase-Key` 操作先修改指定元素的优先级，然后对该元素执行上浮操作，确保堆序性质不被破坏。这一操作是 Dijkstra 最短路径算法和 Prim 最小生成树算法的核心。

## 三、过程/示例

假设我们有一个初始为空的**最大堆**优先队列，依次插入元素：`[5, 3, 8, 1, 6]`。

**步骤 1：插入 5**  
数组：`[5]`，堆结构：`5`（根）

**步骤 2：插入 3**  
数组：`[5, 3]`，堆结构：  
```
    5
   /
  3
```
无需调整，因为 5 ≥ 3。

**步骤 3：插入 8**  
数组：`[5, 3, 8]`，堆结构：  
```
    5
   / \
  3   8
```
8 的父节点是 5，8 > 5，违反堆序，执行上浮：交换 8 和 5。  
数组变为：`[8, 3, 5]`，堆结构：  
```
    8
   / \
  3   5
```

**步骤 4：插入 1**  
数组：`[8, 3, 5, 1]`，堆结构：  
```
    8
   / \
  3   5
 /
1
```
1 的父节点是 3，1 ≤ 3，无需调整。

**步骤 5：插入 6**  
数组：`[8, 3, 5, 1, 6]`，堆结构：  
```
    8
   / \
  3   5
 / \
1   6
```
6 的父节点是 3，6 > 3，上浮：交换 6 和 3。  
数组变为：`[8, 6, 5, 1, 3]`，堆结构：  
```
    8
   / \
  6   5
 / \
1   3
```
6 的新父节点是 8，6 ≤ 8，调整结束。

**步骤 6：Extract-Max**  
取出根节点 8，将末尾元素 3 移到根位置：`[3, 6, 5, 1]`。  
对根节点 3 执行下沉：与较大的子节点 6 交换，得 `[6, 3, 5, 1]`。  
3 的新子节点为 1（左），3 > 1，无需再交换。最终堆：  
```
    6
   / \
  3   5
 /
1
```
返回 8。

## 四、复杂度/性质分析

| 操作 | 二叉堆实现的时间复杂度 | 说明 |
|------|------------------------|------|
| Insert | O(log n) | 最坏情况下需从叶节点上浮到根 |
| Extract-Max | O(log n) | 最坏情况下需从根下沉到叶 |
| Peek-Max | O(1) | 直接返回数组第一个元素 |
| Increase-Key | O(log n) | 上浮操作 |
| 建堆（Build-Heap） | O(n) | 从无序数组自底向上下沉，非 O(n log n) |

- **空间复杂度**：O(n)，使用数组存储 n 个元素。
- **稳定性**：优先队列本身不保证稳定性。当多个元素优先级相同时，取出顺序取决于实现（通常按插入顺序或数组下标顺序，但标准定义不保证）。
- **收敛性**：在算法应用中（如 Dijkstra），优先队列保证每次取出当前最小距离节点，从而保证算法正确收敛。
- **平均情况**：插入和删除的平均时间复杂度仍为 O(log n)，因为堆的高度为 ⌊log₂n⌋。

## 五、常见误区

1. **混淆优先队列与普通队列**：普通队列严格遵循 FIFO，而优先队列的出队顺序由优先级决定，与入队时间无关。初学者常误以为优先队列“先进先出”。

2. **认为堆排序就是优先队列**：堆排序利用堆结构进行排序，但优先队列是一个抽象数据类型，堆只是其一种实现方式。优先队列支持动态插入和删除，而堆排序是静态排序算法。

3. **忽略 Increase-Key 的方向性**：`Increase-Key` 只允许增大优先级（对于最大堆），若需减小优先级，应使用 `Decrease-Key`（常见于最小堆）。错误地调用会导致堆序被破坏。

4. **误以为所有优先队列实现都是 O(log n)**：二叉堆是 O(log n)，但斐波那契堆的 Insert 和 Decrease-Key 均摊 O(1)，Extract-Min 为 O(log n)。不同实现有不同性能特征。

5. **混淆堆与优先队列的“最大/最小”**：最大堆用于实现“取出最大元素”的优先队列，最小堆用于“取出最小元素”。初学者常误以为优先队列只能取最小值（或最大值），实际上两者皆可，取决于堆的类型。
