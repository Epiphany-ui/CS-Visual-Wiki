# B树

## 一、定义

B树（B-Tree）是一种自平衡的多路搜索树，专为磁盘等直接存取辅助存储设备而设计。它通过维持每个节点包含多个键值和多个子节点指针，显著降低树的高度，从而减少磁盘I/O次数。B树是数据库系统和文件系统中索引结构的核心基础。

**数学定义**：一棵阶数为m（m≥2）的B树满足以下条件：
- 每个节点最多包含m-1个键值和m个子节点指针。
- 除根节点外，每个节点至少包含⌈m/2⌉-1个键值。
- 所有叶子节点位于同一层。

**节点结构**：一个B树节点可表示为：
```
Node {
    int n;                    // 当前键值个数
    KeyType keys[m-1];        // 键值数组（有序）
    Node* children[m];        // 子节点指针数组
    bool leaf;                // 是否为叶子节点
}
```

## 二、核心原理

1. **多路平衡机制**：B树通过限制每个节点中键值的数量范围（[⌈m/2⌉-1, m-1]），确保树始终保持平衡。插入和删除操作通过节点分裂与合并来维持这一约束。

2. **搜索路径优化**：与二叉搜索树不同，B树每个节点包含多个键值，搜索时先在节点内部进行二分查找确定方向，然后沿对应子节点继续搜索。这大幅降低了树的高度，对于大规模数据，树高通常为O(logₘN)。

3. **节点分裂与合并**：插入时若节点键值数达到m-1，则将该节点分裂为两个节点，并将中间键值提升至父节点。删除时若节点键值数低于⌈m/2⌉-1，则尝试从兄弟节点借键值或与兄弟节点合并。

4. **磁盘友好设计**：B树的节点大小通常与磁盘块大小对齐（如4KB），使得每次节点访问对应一次磁盘I/O。由于树高很小，查找任意键值只需少量磁盘访问。

5. **顺序访问支持**：叶子节点通常通过链表相连（B+树变体），支持高效的范围查询和顺序遍历。

## 三、过程/示例

以阶数m=3的B树为例，依次插入键值序列：10, 20, 5, 6, 12, 30, 7, 17。

**步骤1**：插入10, 20。根节点为[10,20]，未满。

**步骤2**：插入5。节点变为[5,10,20]，键值数=3=m，触发分裂。中间键值10提升为新根，左右子节点分别为[5]和[20]。
```
        [10]
       /    \
    [5]     [20]
```

**步骤3**：插入6。找到[5]节点，插入后为[5,6]，未满。

**步骤4**：插入12。找到[20]节点，插入后为[12,20]，未满。

**步骤5**：插入30。找到[12,20]节点，插入后为[12,20,30]，键值数=3，触发分裂。中间键值20提升至父节点，父节点变为[10,20]。
```
        [10,20]
       /   |   \
    [5,6] [12] [30]
```

**步骤6**：插入7。找到[5,6]节点，插入后为[5,6,7]，键值数=3，触发分裂。中间键值6提升至父节点，父节点变为[6,10,20]，键值数=3，继续分裂。中间键值10提升为新根。
```
          [10]
         /    \
      [6]      [20]
     /   \     /   \
   [5]  [7]  [12]  [30]
```

**步骤7**：插入17。找到[12]节点，插入后为[12,17]，未满。最终B树结构如上。

## 四、复杂度/性质分析

**时间复杂度**：
- 搜索：O(logₘN)，其中m为阶数，N为键值总数。最坏情况树高为log_{⌈m/2⌉}(N)。
- 插入：O(logₘN)，需要沿路径向下查找插入位置，并可能向上回溯进行分裂。
- 删除：O(logₘN)，需要向下查找并可能向上回溯进行合并或借键。

**空间复杂度**：O(N)，每个键值存储一次。节点内部存在空间浪费（未填满的键值槽），但通常可接受。

**稳定性**：B树不是稳定结构，相同键值的相对顺序可能因分裂操作而改变。

**磁盘I/O复杂度**：对于磁盘存储，B树的优势在于每次节点访问对应一次I/O。树高h ≈ log_{⌈m/2⌉}(N)，对于m=1000，N=10⁹，h≤3，即最多3次磁盘I/O即可完成查找。

**节点利用率**：每个节点至少半满（⌈m/2⌉-1个键值），平均利用率约为69%（ln2）。

## 五、常见误区

1. **B树与B+树混淆**：B+树是B树的变体，所有数据仅存储在叶子节点，内部节点只存索引键值，且叶子节点通过链表相连。B树的数据可存储在任何节点中。B+树更适合范围查询和顺序遍历。

2. **阶数m的含义误解**：阶数m通常指最大子节点数，而非最大键值数。最大键值数为m-1。部分教材定义不同，需注意上下文。

3. **B树与二叉搜索树的性能比较**：B树并非在所有场景下都比二叉搜索树快。对于内存中的小数据集，二叉搜索树（如红黑树）可能更快，因为B树节点内搜索需要额外开销。B树的优势主要体现在磁盘I/O场景。

4. **节点分裂的时机错误**：节点在插入后键值数达到m时才分裂，而非达到m-1时立即分裂。分裂操作将中间键值提升至父节点，左右各保留⌈m/2⌉-1个键值。

5. **删除操作的复杂性低估**：删除操作比插入更复杂，需要处理多种情况：从叶子节点删除、从内部节点删除（需找前驱或后继替换）、节点欠载后的借键或合并。初学者常忽略合并后父节点可能继续欠载的连锁反应。
