# 循环队列

## 一、定义
循环队列（Circular Queue）是一种线性数据结构，它使用固定大小的数组来存储元素，并通过逻辑上将数组的首尾相连来克服普通队列“假溢出”问题。其核心思想是：当队尾指针到达数组末尾时，若数组前端有空位，则队尾指针循环回到数组开头继续入队。

**核心数学表示**：
- 数组长度：`capacity`
- 队头指针：`front`（指向第一个元素）
- 队尾指针：`rear`（指向最后一个元素的下一个位置）
- 队列长度：`(rear - front + capacity) % capacity`
- 判空条件：`front == rear`
- 判满条件：`(rear + 1) % capacity == front`

**入队操作伪代码**：
```
ENQUEUE(Q, x):
    if (Q.rear + 1) % Q.capacity == Q.front:
        error "队列已满"
    Q.array[Q.rear] = x
    Q.rear = (Q.rear + 1) % Q.capacity
```

**出队操作伪代码**：
```
DEQUEUE(Q):
    if Q.front == Q.rear:
        error "队列为空"
    x = Q.array[Q.front]
    Q.front = (Q.front + 1) % Q.capacity
    return x
```

## 二、核心原理
1. **环形复用机制**：通过取模运算 `% capacity` 实现指针的循环移动，使数组空间可以被重复利用，避免普通队列中“前端空位无法使用”的问题。

2. **空一格的判满策略**：循环队列通常牺牲一个存储单元来区分队空和队满。当 `(rear + 1) % capacity == front` 时判定为满，此时数组中实际存储了 `capacity - 1` 个元素。这种设计避免了使用额外变量记录队列长度。

3. **双指针独立移动**：`front` 和 `rear` 指针各自独立移动，入队只动 `rear`，出队只动 `front`，两者在循环空间中相对位置的变化反映了队列状态。

4. **模运算的周期性**：所有指针操作都基于模 `capacity` 运算，使得指针值在 `[0, capacity-1]` 范围内循环，形成逻辑上的环形结构。

5. **恒定容量约束**：循环队列的容量在初始化时固定，无法动态扩展，这是其与链式队列的重要区别。

## 三、过程/示例
假设创建一个容量为5的循环队列（数组下标0-4），初始状态：`front = 0, rear = 0`，队列为空。

**步骤1：入队A、B、C**
- 入队A：`array[0]=A, rear=(0+1)%5=1`
- 入队B：`array[1]=B, rear=(1+1)%5=2`
- 入队C：`array[2]=C, rear=(2+1)%5=3`
当前状态：`front=0, rear=3`，数组：[A, B, C, _, _]

**步骤2：出队两次**
- 出队：取出`array[0]=A`，`front=(0+1)%5=1`
- 出队：取出`array[1]=B`，`front=(1+1)%5=2`
当前状态：`front=2, rear=3`，数组：[_, _, C, _, _]

**步骤3：入队D、E、F**
- 入队D：`array[3]=D, rear=(3+1)%5=4`
- 入队E：`array[4]=E, rear=(4+1)%5=0`
- 入队F：检查`(rear+1)%5 == front?` → `(0+1)%5=1 ≠ 2`，可入队。`array[0]=F, rear=(0+1)%5=1`
当前状态：`front=2, rear=1`，数组：[F, _, C, D, E]

**步骤4：尝试入队G**
- 检查`(rear+1)%5 == front?` → `(1+1)%5=2 == front=2`，队列已满，入队失败。

**步骤5：出队所有元素**
- 出队C：`front=3`
- 出队D：`front=4`
- 出队E：`front=0`
- 出队F：`front=1`
此时`front=1, rear=1`，队列为空。

## 四、复杂度/性质分析
**时间复杂度**：
- 入队操作：O(1)，仅涉及赋值和指针移动
- 出队操作：O(1)，仅涉及取值和指针移动
- 判空/判满：O(1)
- 所有操作均为常数时间，无最好/最坏/平均之分

**空间复杂度**：
- O(capacity)，使用固定大小的数组存储元素
- 实际有效存储容量为 `capacity - 1`（因空一格策略）

**稳定性**：
- 循环队列是稳定的数据结构，元素出队顺序严格遵循先进先出（FIFO）原则，不会改变元素的相对顺序。

**收敛性**：
- 指针在模运算下呈现周期性行为，周期为 `capacity`
- 队列状态空间有限（最多 `capacity` 种不同状态），任何操作序列最终都会进入循环或稳定状态

**与普通队列对比**：
- 普通队列：出队后前端空间无法复用，空间利用率低
- 循环队列：空间可复用，但牺牲一个存储单元

## 五、常见误区
1. **混淆判空与判满条件**：初学者常误以为 `front == rear` 是队满标志，实际上这是队空标志。队满条件是 `(rear + 1) % capacity == front`。

2. **忽略模运算的边界情况**：当 `rear` 或 `front` 为 `capacity-1` 时，加1后应变为0而非 `capacity`。例如 `(4+1)%5=0` 而非5。

3. **误认为循环队列可以存储 `capacity` 个元素**：由于空一格策略，循环队列最多存储 `capacity-1` 个元素。若试图存储 `capacity` 个元素，则无法区分队空和队满状态。

4. **指针更新顺序错误**：入队时应先赋值再移动 `rear`，出队时应先取值再移动 `front`。若顺序颠倒，会导致数据错位或丢失。

5. **与链式队列混淆**：链式队列无容量限制，可动态增长；循环队列容量固定，但操作更高效（无内存分配开销）。两者在判满处理上完全不同。
