# 数组

## 一、定义
数组（Array）是一种线性数据结构，用于在连续内存空间中存储相同类型元素的有限集合。每个元素通过索引（Index）直接访问，索引通常从0开始。数学上，数组可表示为映射 \( A: \{0,1,\dots,n-1\} \to T \)，其中 \( n \) 为数组长度，\( T \) 为元素类型。伪代码定义如下：

```
Array: 
  - 元素类型 T
  - 长度 n
  - 存储单元连续
  - 访问操作: A[i] 返回第 i 个元素 (0 ≤ i < n)
```

## 二、核心原理
1. **连续内存分配**：数组所有元素在内存中按顺序连续存放。设基地址为 \( \text{base} \)，元素大小为 \( s \)，则元素 \( A[i] \) 的地址为 \( \text{base} + i \times s \)。这一特性使得访问任意元素的时间为常数。

2. **随机访问**：通过索引直接计算地址，无需遍历。这是数组最核心的优势，时间复杂度为 \( O(1) \)。

3. **固定大小**：数组在创建时长度固定，不能动态增减。若需动态调整，需重新分配内存并复制元素（如动态数组的实现）。

4. **元素同质性**：所有元素必须为同一数据类型，保证内存布局规整和访问效率。

5. **缓存友好性**：由于空间局部性，连续访问数组元素时CPU缓存命中率高，性能优于链表等非连续结构。

## 三、过程/示例
考虑一个整数数组 `arr = [10, 20, 30, 40, 50]`，长度 \( n=5 \)，基地址为 `0x1000`，每个整数占4字节。

**步骤1：初始化**  
在内存中分配连续5×4=20字节空间，地址范围 `0x1000` 至 `0x1013`。  
`arr[0]` 位于 `0x1000`，`arr[1]` 位于 `0x1004`，依此类推。

**步骤2：访问元素 `arr[3]`**  
计算地址：`0x1000 + 3×4 = 0x100C`，直接读取该地址的值 `40`。  
无需遍历其他元素，一步完成。

**步骤3：修改元素 `arr[2] = 35`**  
计算地址：`0x1000 + 2×4 = 0x1008`，将值 `30` 覆盖为 `35`。  
数组变为 `[10, 20, 35, 40, 50]`。

**步骤4：遍历数组求和**  
从 `i=0` 到 `4`，依次读取 `arr[i]`，累加得 `10+20+35+40+50=155`。  
每次访问均为 \( O(1) \)，总时间 \( O(n) \)。

## 四、复杂度/性质分析
- **访问（Access）**：\( O(1) \) —— 直接计算地址，与数组大小无关。
- **搜索（Search）**：  
  - 无序数组：平均 \( O(n) \)，最坏 \( O(n) \)，最好 \( O(1) \)（目标在首位）。  
  - 有序数组：可用二分查找，\( O(\log n) \)。
- **插入/删除**：  
  - 在末尾：\( O(1) \)（若空间足够）。  
  - 在中间或开头：需移动后续元素，平均 \( O(n) \)，最坏 \( O(n) \)。
- **空间复杂度**：\( O(n) \)，仅存储元素本身，无额外指针开销（相比链表）。
- **稳定性**：数组本身不是排序算法，但作为存储结构，可支持稳定排序（如归并排序）。
- **局限性**：大小固定，动态扩展需复制整个数组（\( O(n) \) 时间），且可能造成内存碎片。

## 五、常见误区
1. **索引从1开始**：许多初学者误以为数组索引从1开始，导致越界或漏访。C、Java、Python等主流语言均从0开始，仅少数语言（如MATLAB、Lua）从1开始。

2. **数组与链表的混淆**：认为数组插入删除快（实际慢），或链表访问快（实际慢）。数组强于随机访问，链表强于动态插入删除。

3. **越界访问**：数组不自动检查索引范围（如C语言），访问 `A[n]` 可能读取非法内存，导致程序崩溃或安全漏洞。Python等语言虽会报错，但性能代价需注意。

4. **数组大小可变**：误以为普通数组可以动态增长。实际静态数组大小固定，动态数组（如Python list、Java ArrayList）是封装了自动扩容的抽象，底层仍涉及复制。

5. **多维数组的内存布局**：认为二维数组是“数组的数组”即行优先存储，但某些语言（如Fortran）采用列优先。混淆会导致缓存性能下降或数据错乱。
