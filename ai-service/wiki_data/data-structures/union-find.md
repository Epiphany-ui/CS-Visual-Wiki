# 并查集

## 一、定义
并查集（Disjoint Set Union，简称DSU）是一种用于管理不相交集合的数据结构，支持两种核心操作：**查找**（Find）和**合并**（Union）。查找操作用于确定某个元素属于哪个集合，通常返回该集合的代表元；合并操作将两个不同的集合合并为一个集合。并查集的核心数学表示为：给定一个有限集合 \( S = \{1, 2, \dots, n\} \)，将其划分为若干互不相交的子集 \( S_1, S_2, \dots, S_k \)，满足 \( \bigcup_{i=1}^k S_i = S \) 且 \( S_i \cap S_j = \emptyset \)（\( i \neq j \)）。并查集通过树形结构或数组实现，典型伪代码如下：

```
初始化：parent[i] = i, rank[i] = 0
Find(x):
    if parent[x] != x:
        parent[x] = Find(parent[x])  // 路径压缩
    return parent[x]
Union(x, y):
    rootX = Find(x), rootY = Find(y)
    if rootX == rootY: return
    if rank[rootX] < rank[rootY]:
        parent[rootX] = rootY
    else if rank[rootX] > rank[rootY]:
        parent[rootY] = rootX
    else:
        parent[rootY] = rootX
        rank[rootX] += 1
```

## 二、核心原理
并查集的核心思想是用树形结构表示集合，每个集合的根节点作为代表元。其工作机制基于以下关键性质：

1. **树形表示**：每个元素有一个父指针，根节点的父指针指向自身。初始时每个元素自成一棵树，即每个元素都是自己的根。
2. **路径压缩**：在查找操作中，将路径上所有节点直接连接到根节点，从而大幅降低后续查找的深度。这是并查集高效的关键优化之一。
3. **按秩合并**：合并时，将秩（树的高度或大小）较小的树挂接到秩较大的树上，避免树过度生长。秩通常存储为树高的上界。
4. **不相交性**：任意两个集合的元素互不重叠，合并操作保证集合的互斥性，不会产生元素同时属于多个集合的情况。
5. **动态性**：并查集支持动态添加新元素（自成一集），但不支持拆分集合，因此适用于元素关系逐渐合并的场景。

## 三、过程/示例
假设有5个元素：1, 2, 3, 4, 5，初始状态每个元素自成一棵树。用数组 `parent` 表示父节点，`rank` 表示秩。

**初始状态**：
- parent = [1, 2, 3, 4, 5]（索引从1开始）
- rank = [0, 0, 0, 0, 0]

**步骤1：合并(1, 2)**  
Find(1)=1, Find(2)=2，秩相同，将2挂到1下。  
parent = [1, 1, 3, 4, 5]，rank[1] = 1。

**步骤2：合并(3, 4)**  
Find(3)=3, Find(4)=4，秩相同，将4挂到3下。  
parent = [1, 1, 3, 3, 5]，rank[3] = 1。

**步骤3：合并(1, 3)**  
Find(1)=1（根），Find(3)=3（根），秩均为1，将3挂到1下。  
parent = [1, 1, 1, 3, 5]，rank[1] = 2。

**步骤4：查找(4)**  
Find(4)：parent[4]=3，parent[3]=1，parent[1]=1，路径压缩将4直接连到1。  
parent = [1, 1, 1, 1, 5]，rank不变。

**步骤5：合并(2, 5)**  
Find(2)=1，Find(5)=5，秩rank[1]=2 > rank[5]=0，将5挂到1下。  
parent = [1, 1, 1, 1, 1]，rank[1]仍为2。

最终所有元素属于同一集合，根为1。

## 四、复杂度/性质分析
并查集的时间复杂度在引入路径压缩和按秩合并后接近常数。具体分析如下：

- **时间复杂度**：单次操作（Find或Union）的均摊时间复杂度为 \( O(\alpha(n)) \)，其中 \( \alpha(n) \) 是阿克曼函数的反函数，对于所有实际输入（\( n \leq 10^{600} \)），\( \alpha(n) \leq 5 \)。最坏情况（无优化）下，单次操作可达 \( O(n) \)。
- **空间复杂度**：需要两个长度为 \( n \) 的数组（parent和rank），空间复杂度为 \( O(n) \)。
- **稳定性**：并查集不涉及排序，无稳定性概念。其操作结果唯一确定，不受输入顺序影响（合并顺序会影响树结构，但最终集合划分相同）。
- **收敛性**：合并操作单调减少集合数量，最多 \( n-1 \) 次合并后所有元素归为一集，过程必然收敛。
- **性质**：路径压缩和按秩合并共同保证了近乎线性的性能，是并查集实用性的基石。

## 五、常见误区
1. **混淆根节点与代表元**：根节点是树结构的根，代表元是集合的标识。在并查集中两者一致，但初学者可能误以为任意元素都可作为代表元，实际上只有根节点才是。
2. **忽略路径压缩的递归实现**：路径压缩通常用递归实现，但递归深度在极端情况下可能导致栈溢出。迭代实现可避免此问题，但教材中常以递归为例。
3. **误认为合并操作可逆**：并查集不支持拆分集合，一旦合并就无法撤销。这与动态连通性问题不同，后者可能需要删除边。
4. **混淆秩与树高**：秩是树高的上界，并非精确高度。路径压缩后实际高度可能小于秩，但秩仍用于合并决策，保证算法正确性。
5. **与图论中连通分量混淆**：并查集用于动态维护连通分量，但图论中连通分量是静态概念。并查集更强调在线处理，而图论算法（如DFS）通常离线计算。
