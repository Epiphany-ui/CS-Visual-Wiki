# 线段树

## 一、定义

线段树（Segment Tree）是一种用于存储区间或线段信息的二叉树数据结构。它允许在 **O(log n)** 时间复杂度内完成区间查询（如求和、最大值、最小值）和单点更新操作。对于长度为 n 的数组，线段树将每个区间 [l, r] 上的信息存储在一个节点中，通过递归二分的方式构建。

核心数学定义：设原数组为 A[1..n]，线段树的每个节点对应一个区间 [l, r]，节点存储的值 f(l, r) 满足：
- 若 l = r，则 f(l, r) = A[l]
- 若 l < r，则 f(l, r) = combine(f(l, mid), f(mid+1, r))，其中 mid = ⌊(l+r)/2⌋

combine 函数根据具体问题定义，如求和时为加法，求最大值时为 max 运算。

## 二、核心原理

1. **递归二分结构**：线段树采用分治思想，将整个区间不断二分，直到每个叶子节点对应单个元素。树的高度为 ⌈log₂n⌉，保证了查询和更新的对数时间复杂度。

2. **节点存储聚合信息**：每个内部节点存储其对应子区间的聚合结果（如区间和、区间最大值等）。这种预计算使得任意区间查询可以通过合并 O(log n) 个节点的信息完成。

3. **懒标记（Lazy Propagation）机制**：为支持区间更新（如区间内所有元素加一个值），引入懒标记。更新时只标记需要更新的节点，推迟向下传递，直到后续查询需要时才真正更新子节点，将区间更新的复杂度也控制在 O(log n)。

4. **完全二叉树存储**：通常用数组实现线段树，根节点下标为 1，节点 i 的左子节点为 2i，右子节点为 2i+1。数组大小一般设为 4n 以确保足够空间。

5. **可扩展性**：线段树不仅支持求和、最值等基本操作，还可扩展支持区间乘法、区间赋值、区间翻转等复杂操作，只需修改 combine 函数和懒标记逻辑。

## 三、过程/示例

以数组 A = [1, 3, 5, 7, 9, 11]（n=6）为例，构建求和线段树并演示区间查询 [2, 5] 的过程。

**构建过程**：
1. 根节点对应区间 [1, 6]，存储和 = 1+3+5+7+9+11 = 36
2. 左子节点 [1, 3]，和 = 1+3+5 = 9；右子节点 [4, 6]，和 = 7+9+11 = 27
3. 继续二分：[1, 2] 和 = 4，[3, 3] 和 = 5，[4, 5] 和 = 16，[6, 6] 和 = 11
4. 叶子节点：[1, 1]=1，[2, 2]=3，[3, 3]=5，[4, 4]=7，[5, 5]=9，[6, 6]=11

**查询区间 [2, 5] 的步骤**：
1. 从根节点 [1, 6] 开始，查询区间 [2, 5] 与当前节点区间不完全重合
2. 进入左子节点 [1, 3]，查询区间 [2, 5] 与其有交集 [2, 3]
3. 在 [1, 3] 节点，继续进入右子节点 [2, 3]（完全包含在查询区间内），直接返回其和 4
4. 回到根节点，进入右子节点 [4, 6]，查询区间 [2, 5] 与其有交集 [4, 5]
5. 在 [4, 6] 节点，进入左子节点 [4, 5]（完全包含），返回其和 16
6. 合并结果：4 + 16 = 20

验证：A[2]+A[3]+A[4]+A[5] = 3+5+7+9 = 24？注意此处 [2,5] 包含下标 2,3,4,5，实际应为 3+5+7+9=24。上述示例中 [2,3] 和应为 3+5=8，[4,5] 和应为 7+9=16，总和 24。此修正说明构建时 [1,2] 和应为 4（1+3），[2,3] 和应为 8（3+5）。

## 四、复杂度/性质分析

**时间复杂度**：
- 构建：O(n)，每个节点访问一次
- 单点查询/更新：O(log n)
- 区间查询：O(log n)，最多访问 4log n 个节点
- 区间更新（带懒标记）：O(log n)

**空间复杂度**：
- 数组实现：O(4n)，实际使用 4n 大小的数组
- 指针实现：O(2n)，但常数较大

**关键性质**：
- 线段树是静态结构，构建后形状不变（除非动态开点）
- 查询结果与操作顺序无关（满足结合律）
- 对于 n 个元素，树中节点总数不超过 4n
- 懒标记的引入使得区间更新与区间查询具有相同的复杂度

**局限性**：
- 不支持高效的插入和删除操作（平衡树更合适）
- 对于动态变化的区间（如区间分裂合并），需要更复杂的数据结构

## 五、常见误区

1. **数组大小设置错误**：初学者常误以为 2n 足够，实际上线段树需要 4n 空间。最坏情况下（n 不是 2 的幂），树高为 ⌈log₂n⌉+1，节点数可达 4n-1。

2. **混淆线段树与树状数组**：树状数组（Fenwick Tree）实现更简单，但只能处理可逆运算（如求和、异或），且不支持区间更新（需差分技巧）。线段树适用范围更广，但常数更大。

3. **懒标记未正确下推**：在区间查询或更新时，必须先将当前节点的懒标记下推到子节点，否则会导致子节点数据不一致。常见错误是在查询时忘记下推。

4. **区间边界处理错误**：线段树通常使用闭区间 [l, r]，递归时 mid = (l+r)/2，左子区间 [l, mid]，右子区间 [mid+1, r]。边界处理不当会导致死循环或漏算。

5. **误认为线段树支持所有区间操作**：线段树要求操作满足结合律（如加法、最大值、GCD）。不满足结合律的操作（如中位数、众数）无法直接用线段树高效处理。
