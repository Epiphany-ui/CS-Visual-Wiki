# AVL树

## 一、定义
AVL树是一种自平衡的二叉搜索树（Binary Search Tree, BST），由苏联数学家Georgy Adelson-Velsky和Evgenii Landis于1962年提出。其核心定义是：对于树中的任意节点，其左子树和右子树的高度之差的绝对值不超过1。该高度差称为平衡因子（Balance Factor, BF），定义为：

\[
BF(node) = height(node.left) - height(node.right)
\]

其中，空子树的高度定义为-1。AVL树要求在每次插入或删除操作后，通过旋转操作维持所有节点的平衡因子在{-1, 0, 1}范围内。

## 二、核心原理
1. **平衡因子约束**：每个节点的平衡因子必须为-1、0或1。若某节点平衡因子绝对值超过1，则称该节点“失衡”，需立即调整。

2. **旋转操作**：当插入或删除导致失衡时，通过四种旋转恢复平衡：
   - **LL旋转**（右单旋）：在左孩子的左子树插入导致失衡。
   - **RR旋转**（左单旋）：在右孩子的右子树插入导致失衡。
   - **LR旋转**（先左后右双旋）：在左孩子的右子树插入导致失衡。
   - **RL旋转**（先右后左双旋）：在右孩子的左子树插入导致失衡。

3. **递归调整**：插入或删除操作后，从修改节点向上回溯至根节点，沿途检查并修复所有失衡节点。每次旋转后，子树高度恢复为插入前的高度，因此上层节点无需继续检查。

4. **高度与节点数关系**：高度为h的AVL树最少节点数满足递推关系：\(N(h) = N(h-1) + N(h-2) + 1\)，其中\(N(0)=1, N(1)=2\)。该递推与斐波那契数列相关，因此\(h \approx 1.44 \log_2(n+2)\)。

## 三、过程/示例
以插入序列[3, 2, 1, 4, 5]为例，演示AVL树的构建过程：

1. **插入3**：空树，根节点为3，BF=0。
2. **插入2**：作为3的左孩子，节点3的BF=1（左高1），仍平衡。
3. **插入1**：作为2的左孩子，节点2的BF=1，节点3的BF=2（左子树高度2，右子树高度0）。节点3失衡，属于LL情况，执行右单旋：
   - 将节点2提升为根，节点3变为2的右孩子。
   - 树结构变为：2(根)的左孩子为1，右孩子为3。所有节点BF=0。
4. **插入4**：从根2开始，4>2，进入右子树3；4>3，作为3的右孩子。节点3的BF=-1，节点2的BF=-1，均平衡。
5. **插入5**：从根2开始，5>2，进入右子树3；5>3，进入右子树4；作为4的右孩子。节点4的BF=-1，节点3的BF=-2（右子树高度2，左子树高度0），节点3失衡，属于RR情况，执行左单旋：
   - 将节点4提升为节点3的位置，节点3变为4的左孩子。
   - 最终树结构：根为2，左孩子1，右孩子4；4的左孩子3，右孩子5。所有节点BF=0。

## 四、复杂度/性质分析
- **时间复杂度**：
  - 查找：\(O(\log n)\)，最坏情况即树高\(1.44\log_2(n+2)\)。
  - 插入/删除：\(O(\log n)\)，包括查找位置和最多\(O(\log n)\)次旋转（每次旋转为\(O(1)\)）。
- **空间复杂度**：\(O(n)\)，每个节点存储值、左右指针和高度（或平衡因子）。
- **稳定性**：AVL树不是稳定结构，相同键值的插入顺序会影响树形态，但平衡性保证性能稳定。
- **与红黑树对比**：AVL树更严格平衡，查找更快（\(1.44\log n\) vs \(2\log n\)），但插入/删除旋转更多（红黑树最多3次旋转）。适用于查找密集型应用。

## 五、常见误区
1. **平衡因子计算错误**：误将空树高度视为0而非-1，导致平衡因子计算偏差。例如，叶子节点的左右子树高度均为-1，BF=0。
2. **旋转后忘记更新高度**：旋转操作后必须更新受影响节点的高度，否则后续平衡因子计算错误。
3. **混淆LL/LR情况**：当左孩子平衡因子为1时插入左子树为LL，左孩子平衡因子为-1时插入右子树为LR。需先检查孩子节点的平衡因子再决定旋转类型。
4. **删除操作忽略回溯**：删除节点可能导致多个祖先失衡，需从删除位置向上回溯至根，不能仅修复最近失衡点。
5. **与二叉搜索树混淆**：AVL树首先必须是BST（左<根<右），平衡调整不能破坏BST性质。旋转时需注意子树的中序顺序不变。
