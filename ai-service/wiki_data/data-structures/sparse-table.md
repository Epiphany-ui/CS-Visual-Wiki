# 稀疏表

## 一、定义

稀疏表（Sparse Table）是一种用于解决**可重复贡献问题**（Range Query with Idempotent Operation）的静态数据结构。它通过预处理一个二维数组 `st[i][j]`，使得对于任意区间 `[l, r]`，可以在 **O(1)** 时间内完成查询。其核心数学定义为：

```
st[i][j] = op( st[i][j-1], st[i + 2^(j-1)][j-1] )
```

其中 `op` 是一个满足**幂等性**的运算（如最小值、最大值、最大公约数、按位与/或等），`st[i][j]` 表示从下标 `i` 开始、长度为 `2^j` 的区间上的运算结果。预处理后，查询区间 `[l, r]` 的答案为：

```
k = floor(log2(r - l + 1))
ans = op( st[l][k], st[r - 2^k + 1][k] )
```

稀疏表不支持动态修改，仅适用于**静态数组**上的区间查询。

## 二、核心原理

1. **倍增思想**：稀疏表基于二进制分解，将任意长度的区间拆分为两个长度均为 2 的幂的子区间。由于运算的幂等性，这两个子区间可以重叠而不影响结果。

2. **预处理递推**：构建过程采用动态规划。第一层 `j=0` 时，`st[i][0] = a[i]`（原数组元素）。对于 `j >= 1`，每个长度为 `2^j` 的区间由两个长度为 `2^(j-1)` 的区间合并得到，递推公式如上。

3. **幂等性要求**：运算必须满足 `op(x, x) = x`。这保证了重叠区间查询的正确性。典型幂等运算包括：最小值、最大值、gcd、lcm、按位与、按位或。**区间和、区间积不满足幂等性**，因此不能用稀疏表处理。

4. **查询的二进制分解**：对于任意区间长度 `len = r - l + 1`，取最大的 `k` 使得 `2^k <= len`。则区间 `[l, r]` 可被 `[l, l+2^k-1]` 和 `[r-2^k+1, r]` 两个子区间完全覆盖（可能有重叠），利用幂等性合并即可。

5. **空间换时间**：稀疏表以 O(n log n) 的预处理时间和空间，换取 O(1) 的查询时间，是典型的静态离线优化策略。

## 三、过程/示例

以数组 `A = [3, 1, 4, 1, 5, 9, 2, 6]`（下标从 0 开始），查询区间最小值（RMQ）为例。

**步骤1：初始化**  
`j=0`：`st[i][0] = A[i]`  
```
st[0][0]=3, st[1][0]=1, st[2][0]=4, st[3][0]=1,
st[4][0]=5, st[5][0]=9, st[6][0]=2, st[7][0]=6
```

**步骤2：递推 j=1**（区间长度 2）  
`st[i][1] = min(st[i][0], st[i+1][0])`  
```
st[0][1]=min(3,1)=1, st[1][1]=min(1,4)=1,
st[2][1]=min(4,1)=1, st[3][1]=min(1,5)=1,
st[4][1]=min(5,9)=5, st[5][1]=min(9,2)=2,
st[6][1]=min(2,6)=2
```

**步骤3：递推 j=2**（区间长度 4）  
`st[i][2] = min(st[i][1], st[i+2][1])`  
```
st[0][2]=min(1,1)=1, st[1][2]=min(1,1)=1,
st[2][2]=min(1,5)=1, st[3][2]=min(1,2)=1,
st[4][2]=min(5,2)=2
```

**步骤4：递推 j=3**（区间长度 8）  
`st[0][3] = min(st[0][2], st[4][2]) = min(1,2) = 1`

**步骤5：查询区间 [1, 6]**  
长度 `len = 6-1+1 = 6`，`k = floor(log2(6)) = 2`（因为 2^2=4 ≤ 6 < 8）  
左区间：`st[1][2] = 1`（覆盖 [1,4]）  
右区间：`st[6-4+1][2] = st[3][2] = 1`（覆盖 [3,6]）  
结果：`min(1, 1) = 1`，正确。

## 四、复杂度/性质分析

- **时间复杂度**：
  - 预处理：O(n log n)，其中 n 为数组长度，log n 为最大层数。
  - 单次查询：O(1)，仅需两次数组访问和一次运算。
  - 建表时每个元素参与 log n 次合并。

- **空间复杂度**：O(n log n)。二维数组大小为 `n × (⌊log₂n⌋+1)`。对于 n=10⁶，log₂n≈20，空间约 80 MB（若存储 4 字节整数）。

- **稳定性**：查询结果完全确定，无随机性；预处理过程也确定。

- **适用性**：
  - 仅适用于**静态数组**（无修改操作）。
  - 仅适用于**幂等运算**（如 min、max、gcd、按位与/或）。
  - 对于非幂等运算（如区间和），应使用线段树或前缀和。

- **与线段树的对比**：
  - 稀疏表：查询 O(1)，预处理 O(n log n)，不支持更新。
  - 线段树：查询 O(log n)，支持更新 O(log n)，空间 O(4n)。

## 五、常见误区

1. **误用于非幂等运算**：初学者常试图用稀疏表求区间和或区间乘积。由于 `sum(x, x) ≠ x`，重叠区间会导致重复计算，结果错误。区间和应使用前缀和或线段树。

2. **查询时 k 的计算错误**：`k = floor(log2(r-l+1))`，而非 `ceil`。若取 `ceil`，则 `2^k` 可能超过区间长度，导致访问越界或覆盖范围超出。

3. **忽略下标边界**：递推时 `st[i][j]` 依赖 `st[i+2^(j-1)][j-1]`，需确保 `i+2^(j-1) < n`，否则数组越界。通常只计算到 `i + 2^j ≤ n`。

4. **与线段树概念混淆**：稀疏表是静态结构，线段树是动态结构。若题目涉及数组元素的修改，应使用线段树或树状数组，而非稀疏表。

5. **认为稀疏表可处理任意区间查询**：实际上仅支持幂等运算。对于区间众数、中位数等复杂查询，稀疏表无法直接应用，需结合其他数据结构。
