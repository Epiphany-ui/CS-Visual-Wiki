# 树状数组

## 一、定义
树状数组（Binary Indexed Tree，BIT），又称Fenwick树，是一种用于高效维护数列前缀和与单点更新的数据结构。它通过将数组索引映射为二进制表示中的最低有效位（Lowbit）来构建一棵逻辑上的树形结构，支持在O(log n)时间内完成前缀和查询和单点更新操作。

设原数组为a[1..n]，树状数组为c[1..n]，则c[i]存储的是原数组中从索引i - lowbit(i) + 1到i的区间和，其中lowbit(i) = i & (-i)。核心操作如下：

- 单点更新：将a[x]增加delta，需更新所有包含x的c[i]：i = x; while(i ≤ n) { c[i] += delta; i += lowbit(i); }
- 前缀和查询：求a[1]到a[x]的和：sum = 0; i = x; while(i > 0) { sum += c[i]; i -= lowbit(i); }

## 二、核心原理
1. **二进制索引机制**：每个索引i的二进制表示决定了其管辖范围。lowbit(i)表示i的二进制中最低位1所对应的数值，c[i]覆盖的区间长度为lowbit(i)，即从i-lowbit(i)+1到i。

2. **树形层次结构**：树状数组并非物理上的树，而是通过索引的二进制运算形成逻辑树。对于任意i，其父节点为i+lowbit(i)，子节点为i-lowbit(i)。更新时沿父链向上传播，查询时沿子链向下累加。

3. **前缀和的可分解性**：任意前缀[1, x]可唯一分解为若干个不相交的lowbit区间。例如x=7（二进制111），可分解为[1,4]、[5,6]、[7,7]，对应c[4]、c[6]、c[7]。查询时通过不断减去lowbit即可获得这些区间。

4. **差分思想扩展**：树状数组不仅支持单点更新+区间查询，还可通过维护差分数组实现区间更新+单点查询，或通过维护两个树状数组实现区间更新+区间查询。

5. **与线段树的关系**：树状数组可视为线段树的简化版本，仅支持前缀操作，但常数更小、实现更简洁。它本质上是线段树的一种特殊形式，去掉了右子树，只保留左子树信息。

## 三、过程/示例
以数组a = [3, 2, 5, 1, 4]（索引从1开始）为例，构建树状数组c。

**初始化**：c全为0，依次对每个位置执行单点更新。

1. 更新a[1]=3：i=1，lowbit(1)=1，更新c[1]+=3，i+=1得2，c[2]+=3，i+=2得4，c[4]+=3，i+=4得8>5停止。此时c=[3,3,0,3,0]。

2. 更新a[2]=2：i=2，c[2]+=2得5，i+=2得4，c[4]+=2得5，i+=4得8停止。c=[3,5,0,5,0]。

3. 更新a[3]=5：i=3，c[3]+=5，i+=1得4，c[4]+=5得10，i+=4得8停止。c=[3,5,5,10,0]。

4. 更新a[4]=1：i=4，c[4]+=1得11，i+=4得8停止。c=[3,5,5,11,0]。

5. 更新a[5]=4：i=5，c[5]+=4，i+=1得6>5停止。c=[3,5,5,11,4]。

**查询前缀和**：求前3个元素和sum(3)。

- i=3，sum+=c[3]=5，i-=lowbit(3)=1得2
- i=2，sum+=c[2]=5，sum=10，i-=2得0，结束
- 结果10，验证a[1]+a[2]+a[3]=3+2+5=10，正确。

**查询区间和**：求[2,4]的和 = sum(4)-sum(1)。

- sum(4)：i=4，sum+=c[4]=11，i-=4得0，结果11
- sum(1)：i=1，sum+=c[1]=3，i-=1得0，结果3
- 区间和=11-3=8，验证2+5+1=8，正确。

## 四、复杂度/性质分析
- **时间复杂度**：单点更新和前缀和查询均为O(log n)，其中n为数组长度。建树可通过n次更新实现，复杂度O(n log n)；也可通过线性构造达到O(n)。区间查询需两次前缀和操作，仍为O(log n)。所有操作的最坏、最好、平均情况均为O(log n)，无退化情况。

- **空间复杂度**：O(n)，仅需一个与原数组等长的辅助数组。

- **稳定性**：树状数组是确定性数据结构，不涉及随机化或近似，结果精确。

- **扩展性质**：支持可逆操作（如加减），但不支持不可逆操作（如取最大值）。对于不可逆操作，需使用线段树。

- **与线段树对比**：树状数组常数更小（约1/3），代码更短，但功能受限——只能处理前缀可分解的问题。线段树支持任意区间操作和多种运算，但实现复杂、常数大。

## 五、常见误区
1. **索引从0开始**：树状数组要求索引从1开始，否则lowbit(0)=0会导致死循环。若原数组从0开始，需整体偏移1。

2. **混淆lowbit方向**：更新时i+=lowbit(i)向上传播，查询时i-=lowbit(i)向下累加。初学者常将两者方向弄反。

3. **区间查询直接使用c数组**：c[i]存储的是区间和而非单点值，求区间[l,r]和必须用sum(r)-sum(l-1)，不能直接累加c[l]到c[r]。

4. **与线段树功能混淆**：树状数组不能直接求区间最大值、最小值，也不能处理区间赋值操作。若需这些功能，应使用线段树或分块。

5. **忽略数组越界**：更新时i+lowbit(i)可能超过n，需加边界判断；查询时i-lowbit(i)可能小于0，但while(i>0)已处理。
