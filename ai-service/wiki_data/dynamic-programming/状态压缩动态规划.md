# 状态压缩动态规划

## 一、定义

状态压缩动态规划（State Compression Dynamic Programming，简称状压DP）是一种利用二进制位运算来表示和操作集合状态的动态规划方法。其核心思想是将一个集合的子集用二进制整数表示，其中第 \(i\) 位为 1 表示该元素在集合中，为 0 表示不在。状压DP通常用于解决状态空间为 \(2^n\) 量级的组合优化问题，其基本递推形式可表示为：

\[
dp[S] = \min_{i \in S} \left\{ dp[S \setminus \{i\}] + cost(i, S) \right\}
\]

其中 \(S\) 是一个二进制掩码表示的集合，\(dp[S]\) 表示达到状态 \(S\) 的最优值，\(cost(i, S)\) 表示在状态 \(S\) 下加入元素 \(i\) 的代价。

## 二、核心原理

1. **状态编码与位运算**：将集合的每个子集映射为一个唯一的二进制整数。利用位运算（如 `&`、`|`、`^`、`<<`、`>>`）可以高效地进行集合的交、并、差、判断包含关系等操作，这是状压DP高效性的基础。

2. **状态转移的枚举策略**：通常采用“枚举子集”或“枚举最后一个加入的元素”两种方式。前者用于需要遍历所有子集的场景，后者适用于路径类问题（如旅行商问题）。转移时通过位运算快速生成新状态。

3. **无后效性保证**：状压DP要求状态定义满足无后效性，即当前状态一旦确定，后续决策只依赖于当前状态本身，而不依赖于到达该状态的具体路径。二进制掩码天然记录了所有已选元素，保证了信息的完备性。

4. **空间压缩与滚动数组**：由于状态数随元素个数指数增长（\(2^n\)），当 \(n\) 较大时（通常 \(n \leq 20\)），需注意内存限制。部分问题可通过滚动数组或只保留必要维度的状态来优化空间。

5. **预处理优化**：对于频繁使用的子集信息（如子集和、子集内元素间的代价），可预先计算并存储，避免在DP过程中重复计算，从而降低常数因子。

## 三、过程/示例

**问题**：给定一个带权有向完全图，顶点编号为 0, 1, 2，求从顶点 0 出发、恰好经过每个顶点一次并回到 0 的最短路径（旅行商问题，TSP）。

**输入**：邻接矩阵 \(dist[3][3] = \begin{bmatrix} 0 & 2 & 9 \\ 1 & 0 & 6 \\ 8 & 7 & 0 \end{bmatrix}\)

**状态定义**：\(dp[S][v]\) 表示当前已访问的顶点集合为 \(S\)（二进制掩码），且当前位于顶点 \(v\) 时的最短路径长度。初始状态 \(dp[\{0\}][0] = 0\)，其余为无穷大。

**执行步骤**：

1. 初始化：\(dp[1][0] = 0\)（二进制 001 表示只访问了顶点 0）。

2. 枚举所有状态 \(S\) 和当前顶点 \(v\)：
   - 状态 \(S=3\)（011，已访问 0 和 1），当前顶点 \(v=1\)：
     - 从 \(S \setminus \{1\} = 1\)（001）且上一顶点为 0 转移：\(dp[3][1] = dp[1][0] + dist[0][1] = 0 + 2 = 2\)
   - 状态 \(S=5\)（101，已访问 0 和 2），当前顶点 \(v=2\)：
     - 从 \(S \setminus \{2\} = 1\)（001）且上一顶点为 0 转移：\(dp[5][2] = dp[1][0] + dist[0][2] = 0 + 9 = 9\)
   - 状态 \(S=7\)（111，已访问所有顶点），当前顶点 \(v=1\)：
     - 从 \(S \setminus \{1\} = 5\)（101）且上一顶点为 2 转移：\(dp[7][1] = dp[5][2] + dist[2][1] = 9 + 7 = 16\)
     - 从 \(S \setminus \{1\} = 3\)（011）且上一顶点为 0 转移：\(dp[7][1] = \min(16, dp[3][0] + dist[0][1])\)，但 \(dp[3][0]\) 未定义（因为状态 3 中当前顶点不能为 0，0 已在集合中但非当前顶点），故忽略。
   - 状态 \(S=7\)，当前顶点 \(v=2\)：
     - 从 \(S \setminus \{2\} = 3\)（011）且上一顶点为 1 转移：\(dp[7][2] = dp[3][1] + dist[1][2] = 2 + 6 = 8\)

3. 最终答案：\(\min_{v \neq 0} \{ dp[7][v] + dist[v][0] \} = \min(16 + dist[1][0], 8 + dist[2][0]) = \min(16+1, 8+8) = 16\)

**结果**：最短路径长度为 16，对应路径 0→1→2→0。

## 四、复杂度/性质分析

- **时间复杂度**：最坏情况为 \(O(n^2 \cdot 2^n)\)，其中 \(n\) 为元素个数。对于每个状态 \(S\)（共 \(2^n\) 个），需要枚举当前顶点 \(v\)（\(n\) 种可能）和上一顶点 \(u\)（\(n\) 种可能），故总操作数为 \(O(n^2 \cdot 2^n)\)。通过预处理邻接信息可优化常数，但渐进复杂度不变。

- **空间复杂度**：通常为 \(O(n \cdot 2^n)\)，用于存储 \(dp[S][v]\) 数组。当 \(n=20\) 时，约需 \(20 \times 2^{20} \approx 20\) MB 内存（假设每个状态用 4 字节整数），在可接受范围内。对于 \(n>20\) 的问题，需考虑启发式或近似算法。

- **稳定性**：算法本身是确定性的，给定相同输入和状态定义，输出结果唯一。数值稳定性取决于代价函数，若代价为整数则无精度问题。

- **收敛性**：由于状态空间有限且转移方向明确（从小集合到大集合），DP 过程必然在有限步内终止，不存在不收敛的情况。

## 五、常见误区

1. **状态定义遗漏当前顶点**：在 TSP 等路径问题中，仅用 \(dp[S]\) 表示访问集合 \(S\) 的最优值是不够的，因为缺少“当前所在位置”信息，导致无法正确计算返回起点的代价。必须使用二维状态 \(dp[S][v]\)。

2. **位运算优先级错误**：在 C/C++ 中，位运算符优先级低于比较运算符。例如 `if (S & (1 << i) == 0)` 会被解析为 `if (S & ((1 << i) == 0))`，导致逻辑错误。正确写法应为 `if ((S & (1 << i)) == 0)`。

3. **忽略起点/终点的特殊处理**：许多问题要求从特定起点出发并回到起点，初学者容易忘记在最终答案中加上返回起点的代价，或错误地将起点视为普通顶点进行枚举。

4. **状态转移顺序混乱**：状压DP要求从小集合向大集合转移，若循环顺序不当（如先枚举顶点再枚举状态），可能导致使用未计算的状态值。正确做法是外层循环枚举状态 \(S\)，内层枚举当前顶点 \(v\)。

5. **与回溯法混淆**：状压DP与回溯法都能解决子集枚举问题，但回溯法通过递归剪枝搜索，时间复杂度通常为 \(O(n!)\)；状压DP通过状态复用将复杂度降为 \(O(n^2 \cdot 2^n)\)。初学者容易在 \(n\) 较小时误用回溯法，导致指数级时间开销。
