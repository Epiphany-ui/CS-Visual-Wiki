# 背包问题

## 一、定义
背包问题（Knapsack Problem）是一类经典的组合优化问题，其基本形式可描述为：给定一组物品，每个物品具有重量和价值两个属性，在总重量不超过背包容量限制的前提下，选择若干物品装入背包，使得装入物品的总价值最大化。

最经典的0-1背包问题的数学形式化定义为：
- 输入：物品数量n，背包容量W，物品重量数组w[1..n]，物品价值数组v[1..n]
- 输出：最大化总价值V = Σv[i]·x[i]，满足Σw[i]·x[i] ≤ W，其中x[i]∈{0,1}表示是否选取第i个物品

核心递推关系（动态规划解法）：
```
dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])  当j ≥ w[i]
dp[i][j] = dp[i-1][j]                                当j < w[i]
```
其中dp[i][j]表示前i个物品在容量为j时的最大价值。

## 二、核心原理
1. **最优子结构性质**：背包问题的最优解包含其子问题的最优解。若已知最优解中是否包含第i个物品，则剩余物品的选择方案必须是剩余容量下的最优解。这一性质是动态规划方法的基础。

2. **重叠子问题**：在递归求解过程中，相同的子问题（特定物品集合和剩余容量）会被多次遇到。动态规划通过表格存储中间结果，避免重复计算，这是其高效性的关键。

3. **状态转移机制**：对于每个物品，决策只有"取"或"不取"两种选择。状态转移方程体现了这种二元决策：不取时继承前i-1个物品的结果，取时则需为当前物品腾出空间并加上其价值。

4. **无后效性**：当前状态一旦确定，后续决策只依赖于当前状态（剩余容量和已考虑的物品），而不关心如何达到该状态。这使得问题可以按阶段顺序求解。

5. **贪心策略的失效**：与分数背包问题不同，0-1背包问题不能简单地按单位价值贪心选择，因为物品不可分割，局部最优可能导致全局次优。

## 三、过程/示例
考虑一个具体实例：背包容量W=10，有4个物品：
- 物品1：重量6，价值30
- 物品2：重量3，价值14
- 物品3：重量4，价值16
- 物品4：重量2，价值9

动态规划求解过程（构建dp[5][11]表格，初始化为0）：

**步骤1**：处理物品1（w=6, v=30）
- 容量0-5：无法装入，dp[1][j]=0
- 容量6-10：可以装入，dp[1][j]=30

**步骤2**：处理物品2（w=3, v=14）
- 容量0-2：无法装入，dp[2][j]=dp[1][j]
- 容量3-5：可装物品2，dp[2][3]=max(0, 0+14)=14；dp[2][4]=max(0, 0+14)=14；dp[2][5]=max(0, 0+14)=14
- 容量6-8：可装物品2或物品1，dp[2][6]=max(30, 0+14)=30；dp[2][7]=max(30, 14+14)=30；dp[2][8]=max(30, 14+14)=30
- 容量9-10：可同时装物品1和2，dp[2][9]=max(30, 14+30)=44；dp[2][10]=max(30, 14+30)=44

**步骤3**：处理物品3（w=4, v=16）
- 容量4：dp[3][4]=max(dp[2][4]=14, dp[2][0]+16=16)=16
- 容量7：dp[3][7]=max(dp[2][7]=30, dp[2][3]+16=14+16=30)=30
- 容量9：dp[3][9]=max(dp[2][9]=44, dp[2][5]+16=14+16=30)=44
- 容量10：dp[3][10]=max(dp[2][10]=44, dp[2][6]+16=30+16=46)=46

**步骤4**：处理物品4（w=2, v=9）
- 容量10：dp[4][10]=max(dp[3][10]=46, dp[3][8]+9=30+9=39)=46

最终最优价值为46，对应选择物品1和物品3（30+16=46）。

## 四、复杂度/性质分析
**时间复杂度**：
- 动态规划解法：O(n·W)，其中n为物品数量，W为背包容量。这是伪多项式时间复杂度，因为W的规模是输入的数值而非位数。
- 回溯法（暴力搜索）：O(2^n)，指数级复杂度。
- 分支限界法：最坏情况仍为O(2^n)，但平均性能优于暴力搜索。

**空间复杂度**：
- 基础动态规划：O(n·W)，使用二维数组。
- 空间优化版本：O(W)，仅使用一维数组，但需注意遍历顺序（容量需逆序遍历）。

**关键性质**：
- 0-1背包问题是NP完全的，不存在多项式时间算法（除非P=NP）。
- 分数背包问题（物品可分割）可在O(n log n)时间内通过贪心算法求解。
- 完全背包问题（物品无限取用）同样可用动态规划，但内层循环需正序遍历容量。

## 五、常见误区
1. **混淆0-1背包与分数背包**：初学者常误以为0-1背包也可用贪心算法（按单位价值排序），但整数约束使贪心失效。例如物品A（w=5,v=10）和B（w=4,v=8），容量5时贪心选A得10，但实际最优是B（若还有C（w=1,v=1）则选B+C=9>10）。

2. **动态规划遍历顺序错误**：0-1背包的一维优化中，容量必须逆序遍历，否则物品会被多次选取（变成完全背包）。这是最常见的编码错误。

3. **忽略初始化条件**：dp数组的初始值设置错误，例如要求恰好装满时需初始化为负无穷，而非0。不同约束条件对应不同的初始化策略。

4. **状态定义混淆**：将dp[i][j]错误理解为"前i个物品恰好装满容量j"与"不超过容量j"混用，导致递推关系错误。

5. **回溯路径理解偏差**：得到最优价值后，回溯选取物品时，误以为dp[i][j] > dp[i-1][j]就一定选了第i个物品，忽略了相等情况下的多种最优解可能性。
