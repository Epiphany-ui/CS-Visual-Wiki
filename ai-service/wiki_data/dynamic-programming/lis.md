# 最长递增子序列

## 一、定义

最长递增子序列（Longest Increasing Subsequence, LIS）是计算机科学和组合数学中的一个经典问题。给定一个序列 \( X = (x_1, x_2, \dots, x_n) \)，其子序列是指从原序列中删除零个或多个元素（不改变剩余元素的相对顺序）后得到的新序列。若该子序列中的元素严格单调递增（即 \( x_{i_1} < x_{i_2} < \dots < x_{i_k} \)），则称其为递增子序列。最长递增子序列即所有递增子序列中长度最大的那个。

数学上，定义 \( \text{LIS}(X) = \max\{ k \mid \exists i_1 < i_2 < \dots < i_k, \text{ s.t. } x_{i_1} < x_{i_2} < \dots < x_{i_k} \} \)。该问题不要求子序列在原序列中连续，这是其与“最长递增子数组”或“最长连续递增子序列”的本质区别。

## 二、核心原理

1. **动态规划思想**：LIS 问题的经典解法基于动态规划。定义状态 \( dp[i] \) 表示以第 \( i \) 个元素结尾的最长递增子序列的长度。转移方程为：
   \[
   dp[i] = 1 + \max\{ dp[j] \mid j < i \text{ 且 } x_j < x_i \}
   \]
   初始时 \( dp[i] = 1 \)（每个元素自身构成一个长度为1的递增子序列）。最终答案为 \( \max_{1 \le i \le n} dp[i] \)。

2. **贪心 + 二分优化**：对于大规模数据，\( O(n^2) \) 的动态规划效率不足。优化方法维护一个数组 \( tails \)，其中 \( tails[k] \) 表示所有长度为 \( k+1 \) 的递增子序列中末尾元素的最小可能值。遍历原序列时，对每个元素 \( x \)，在 \( tails \) 中二分查找第一个大于等于 \( x \) 的位置并替换，从而将复杂度降至 \( O(n \log n) \)。

3. **子序列与子串的区分**：LIS 允许元素不连续，因此不能使用滑动窗口或双指针。其本质是在保持原序的前提下，从序列中挑选出尽可能多的元素构成递增关系，这类似于在偏序集中寻找最长链。

4. **唯一性与多解性**：LIS 的长度是唯一的，但具体子序列可能不唯一。例如序列 \( [1, 3, 2, 4] \) 的 LIS 长度为 3，但可以是 \( [1, 3, 4] \) 或 \( [1, 2, 4] \)。

## 三、过程/示例

考虑序列 \( X = [3, 1, 8, 2, 5] \)，使用动态规划方法求解 LIS。

- 初始化：\( dp = [1, 1, 1, 1, 1] \)，表示每个元素自身构成长度为1的子序列。
- 处理 \( i=1 \)（元素 1）：前面元素为 3，但 3 > 1，不满足递增，\( dp[1] \) 保持 1。
- 处理 \( i=2 \)（元素 8）：前面有 3 和 1，均小于 8，取最大 \( dp \) 值 1，则 \( dp[2] = 1 + 1 = 2 \)。
- 处理 \( i=3 \)（元素 2）：前面小于 2 的元素只有 1（索引1），\( dp[1]=1 \)，故 \( dp[3] = 1 + 1 = 2 \)。
- 处理 \( i=4 \)（元素 5）：前面小于 5 的元素有 3、1、2，对应 \( dp \) 值分别为 1、1、2，最大为 2，故 \( dp[4] = 1 + 2 = 3 \)。

最终 \( dp = [1, 1, 2, 2, 3] \)，最大值为 3，对应子序列 \( [1, 2, 5] \) 或 \( [3, 8, 5] \)（注意 \( [3, 8, 5] \) 不递增，实际应为 \( [1, 2, 5] \) 或 \( [1, 8, 5] \) 不成立，正确解为 \( [1, 2, 5] \) 或 \( [3, 8] \) 长度不足，需仔细验证）。正确 LIS 为 \( [1, 2, 5] \) 或 \( [1, 8, 5] \) 不递增，故唯一解为 \( [1, 2, 5] \)。（注：示例中 \( [3, 8, 5] \) 不递增，应修正为 \( [1, 2, 5] \) 或 \( [1, 8] \) 长度2，实际 LIS 为 \( [1, 2, 5] \) 长度3。）

## 四、复杂度/性质分析

- **时间复杂度**：
  - 动态规划法：\( O(n^2) \)，其中 \( n \) 为序列长度。最好、最坏、平均情况均为 \( O(n^2) \)，因为需要遍历所有 \( j < i \)。
  - 贪心+二分法：\( O(n \log n) \)，每次二分查找 \( O(\log n) \)，遍历 \( n \) 个元素。最好、最坏、平均均为 \( O(n \log n) \)。

- **空间复杂度**：
  - 动态规划法：\( O(n) \)，用于存储 \( dp \) 数组。
  - 贪心+二分法：\( O(n) \)，用于存储 \( tails \) 数组。

- **稳定性**：LIS 问题本身不涉及排序稳定性，但若需输出具体子序列，需额外记录前驱信息，此时空间复杂度可能升至 \( O(n^2) \) 或 \( O(n) \) 取决于实现。

- **收敛性**：不适用，LIS 是离散组合优化问题，无迭代收敛过程。

## 五、常见误区

1. **混淆“子序列”与“子数组”**：子序列不要求连续，而子数组（或子串）要求连续。例如序列 \( [1, 4, 2, 3] \) 的 LIS 是 \( [1, 2, 3] \)（长度3），但最长连续递增子数组是 \( [1, 4] \) 或 \( [2, 3] \)（长度2）。

2. **误认为 LIS 唯一**：LIS 的长度唯一，但具体子序列可能不唯一。初学者常以为只有一个解，导致在输出路径时出错。

3. **忽略严格递增与不降的区别**：LIS 通常要求严格递增（\( < \)），而非不降（\( \le \)）。若题目要求非递减，则转移条件需改为 \( x_j \le x_i \)，且二分查找时需找第一个大于 \( x \) 的位置。

4. **动态规划初始化错误**：忘记将每个 \( dp[i] \) 初始化为 1，或错误地认为空子序列长度为 0，导致结果偏小。

5. **二分优化中边界处理不当**：在 \( tails \) 数组中查找第一个大于等于 \( x \) 的位置时，若所有元素均小于 \( x \)，应追加到末尾，而非替换最后一个元素。常见错误是使用 `lower_bound` 后直接替换，未处理越界情况。
