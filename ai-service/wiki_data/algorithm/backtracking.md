# 回溯算法

## 一、定义
回溯算法（Backtracking Algorithm）是一种通过系统地搜索解空间来求解问题的算法范式。它采用深度优先搜索策略，在问题的解空间树中逐步构造候选解，并在发现当前部分解无法扩展为有效解时，撤销上一步或上几步的选择，回退到之前的状态继续尝试其他可能性。回溯算法的核心可形式化为递归函数：

```
function backtrack(state):
    if state is a valid solution:
        record solution
        return
    for each candidate in generate_candidates(state):
        if is_valid(candidate):
            apply(candidate, state)
            backtrack(state)
            undo(candidate, state)
```

该算法本质上是对解空间树进行深度优先遍历，并利用剪枝函数（约束条件和限界条件）来避免无效搜索。

## 二、核心原理
1. **解空间树模型**：回溯算法将问题的所有可能解组织成一棵树形结构，每个节点代表问题的一个部分解，根节点为空解，叶子节点为完整解。算法从根节点出发，沿着树的分支逐步构造解。

2. **深度优先搜索**：回溯采用DFS策略，优先沿着一条路径深入探索，直到无法继续或找到解，再回溯到最近的分支点尝试其他路径。这种策略保证了系统性和完整性。

3. **剪枝机制**：在搜索过程中，通过约束函数（判断当前部分解是否可能扩展为有效解）和限界函数（判断当前部分解是否可能优于已知最优解）来提前终止无效分支的探索，这是回溯算法效率的关键。

4. **状态恢复（撤销操作）**：回溯的核心操作是“尝试-撤销”模式。当从当前状态返回时，必须精确撤销之前所做的修改，使状态恢复到尝试前的样子，这是保证搜索正确性的基础。

5. **递归与迭代实现**：回溯通常用递归实现，递归调用栈天然支持状态的保存与恢复。也可用显式栈实现迭代版本，但逻辑复杂度增加。

## 三、过程/示例
以经典的**N皇后问题**（N=4）为例，要求在4×4棋盘上放置4个皇后，使它们互不攻击（任意两个不在同一行、列或对角线上）。

**初始状态**：空棋盘，从第0行开始放置。

**步骤1**：在第0行尝试放置皇后。遍历第0行的4列，首先尝试列0。放置后状态：`[0]`（表示第0行皇后在第0列）。

**步骤2**：进入第1行。遍历列0-3，检查是否与已放置皇后冲突。列0与第0行同列冲突，列1在对角线上冲突，列2安全。放置皇后在列2。状态：`[0, 2]`。

**步骤3**：进入第2行。遍历列0-3：列0与第0行同列冲突，列1与第1行对角线冲突，列2与第1行同列冲突，列3与第0行对角线冲突。无合法位置，回溯。

**步骤4**：撤销第1行皇后，回到第1行尝试下一个候选列。第1行列2已尝试，尝试列3。检查：列3与第0行对角线冲突（行差1，列差3），不合法。第1行无更多候选，继续回溯到第0行。

**步骤5**：撤销第0行皇后，在第0行尝试列1。放置后状态：`[1]`。

**步骤6**：进入第1行。列0与第0行对角线冲突，列1同列冲突，列2安全。放置：`[1, 3]`。

**步骤7**：进入第2行。列0与第1行对角线冲突，列1与第0行对角线冲突，列2与第1行同列冲突，列3与第0行同列冲突。无合法位置，回溯。

**步骤8**：撤销第1行皇后，尝试第1行列3。列3与第0行对角线冲突，不合法。回溯到第0行。

**步骤9**：在第0行尝试列2。放置：`[2]`。第1行列0安全，放置：`[2, 0]`。第2行列3安全（检查与第0行对角线：行差2列差1；与第1行对角线：行差1列差3），放置：`[2, 0, 3]`。第3行列1安全，放置：`[2, 0, 3, 1]`。找到一个解。

继续搜索可得另一个解：`[1, 3, 0, 2]`。

## 四、复杂度/性质分析
1. **时间复杂度**：最坏情况下，回溯算法需要遍历解空间树的所有节点。对于N皇后问题，解空间大小为O(N!)，最坏时间复杂度为O(N!)。实际运行时间取决于剪枝效率，平均情况通常远优于最坏情况。对于子集和问题，最坏为O(2^N)。

2. **空间复杂度**：主要取决于递归调用栈的深度，通常为O(N)（N为问题规模）。此外，需要存储当前部分解，空间复杂度为O(N)。总空间复杂度为O(N)。

3. **稳定性**：回溯算法是确定性的，给定相同的输入和搜索顺序，输出结果完全一致。但搜索顺序会影响找到第一个解的速度。

4. **收敛性**：对于有限解空间的问题，回溯算法保证在有限步内终止（找到所有解或确认无解）。对于无限解空间的问题，需配合适当的剪枝条件。

5. **最优性**：回溯算法可以保证找到所有解（穷举搜索），或在找到第一个解时终止。结合限界函数（如分支限界法），可保证找到最优解。

## 五、常见误区
1. **混淆回溯与递归**：回溯是一种算法范式，递归是实现手段。并非所有递归都是回溯，只有包含“尝试-撤销-回溯”机制的递归才是回溯。简单的递归（如斐波那契数列）不涉及状态恢复。

2. **忘记状态恢复**：初学者常忘记在递归返回后撤销修改，导致状态污染，使后续搜索基于错误的状态。这是回溯实现中最常见的bug。

3. **剪枝条件不完整**：剪枝是回溯效率的关键，但初学者可能遗漏某些约束条件，导致搜索空间过大；或剪枝过于激进，错误地剪掉包含解的分支。

4. **混淆回溯与动态规划**：两者都涉及状态空间搜索，但动态规划通过重叠子问题的最优子结构来避免重复计算，而回溯通过剪枝避免无效搜索。回溯通常用于组合优化和约束满足问题，动态规划用于最优化问题。

5. **忽视搜索顺序的影响**：候选解的生成顺序会显著影响回溯效率。合理的顺序（如优先尝试约束最强的候选）可以大幅减少搜索空间，这是启发式搜索的核心思想。
