# KMP字符串匹配

## 一、定义

KMP字符串匹配（Knuth-Morris-Pratt算法）是一种高效的字符串匹配算法，由Donald Knuth、James H. Morris和Vaughan Pratt于1977年联合提出。该算法用于在文本串（text）中查找模式串（pattern）的所有出现位置，其核心思想是利用模式串自身的结构信息避免不必要的回溯。

给定文本串T[0..n-1]和模式串P[0..m-1]，KMP算法通过预处理模式串得到部分匹配表（也称为失败函数或前缀函数）π，其中π[i]表示P[0..i]的最长相等真前缀与真后缀的长度。匹配过程中，当字符失配时，模式串向右滑动π[i]个位置而非仅移动一个字符。

核心伪代码（预处理部分）：
```
COMPUTE-PREFIX-FUNCTION(P)
    m = length(P)
    π[0] = 0
    k = 0
    for q = 1 to m-1
        while k > 0 and P[k] ≠ P[q]
            k = π[k-1]
        if P[k] = P[q]
            k = k + 1
        π[q] = k
    return π
```

## 二、核心原理

1. **避免回溯思想**：传统暴力匹配在失配时，文本指针需回退到起始位置的下一个字符，导致O(nm)时间复杂度。KMP利用已匹配部分的信息，保证文本指针永不回溯，仅模式串指针根据π表调整。

2. **前缀函数的自指性质**：π[i]的值具有递归性——若已知π[i-1]=k，则π[i]最多为k+1（当P[i]=P[k]时），否则需递归检查π[k-1]。这种自指结构使得预处理可在O(m)时间内完成。

3. **滑动距离的确定性**：当匹配到第j个字符失配时，模式串向右滑动的距离为j - π[j-1]，其中π[j-1]是已匹配部分的最长公共前后缀长度。这保证了不会错过任何可能的匹配起始位置。

4. **最优子结构**：前缀函数满足最优子结构性质，即最长公共前后缀的求解可分解为子问题，这是动态规划思想在字符串处理中的经典应用。

## 三、过程/示例

以文本串T = "ababcabababc"和模式串P = "ababc"为例：

**步骤1：预处理π表**
- P[0]='a'，π[0]=0
- P[1]='b'，P[1]≠P[0]，π[1]=0
- P[2]='a'，P[2]=P[0]，π[2]=1
- P[3]='b'，P[3]=P[1]，π[3]=2
- P[4]='c'，P[4]≠P[2]且P[4]≠P[0]，π[4]=0
得到π = [0,0,1,2,0]

**步骤2：匹配过程**
- i=0, j=0: T[0]='a'=P[0]，匹配，i=1, j=1
- i=1, j=1: T[1]='b'=P[1]，匹配，i=2, j=2
- i=2, j=2: T[2]='a'=P[2]，匹配，i=3, j=3
- i=3, j=3: T[3]='b'=P[3]，匹配，i=4, j=4
- i=4, j=4: T[4]='c'=P[4]，匹配，i=5, j=5
- j=m=5，找到匹配位置0，j=π[4]=0

- i=5, j=0: T[5]='a'=P[0]，匹配，i=6, j=1
- i=6, j=1: T[6]='b'=P[1]，匹配，i=7, j=2
- i=7, j=2: T[7]='a'=P[2]，匹配，i=8, j=3
- i=8, j=3: T[8]='b'=P[3]，匹配，i=9, j=4
- i=9, j=4: T[9]='a'≠P[4]='c'，失配，j=π[3]=2
- i=9, j=2: T[9]='a'=P[2]，匹配，i=10, j=3
- i=10, j=3: T[10]='b'=P[3]，匹配，i=11, j=4
- i=11, j=4: T[11]='c'=P[4]，匹配，i=12, j=5
- j=m=5，找到匹配位置6

最终在位置0和6找到两处匹配。

## 四、复杂度/性质分析

**时间复杂度**：
- 预处理阶段：O(m)，其中m为模式串长度。每个字符最多被比较两次（一次在while循环中增加k，一次在if中）。
- 匹配阶段：O(n)，其中n为文本串长度。文本指针i严格递增，模式指针j最多增加n次，每次回退至少减少1，总操作次数为O(n)。
- 总体：O(n+m)，线性时间复杂度。

**空间复杂度**：O(m)，用于存储前缀函数π数组。

**最好情况**：O(n+m)，当模式串无重复前缀时，π表全为0，匹配过程退化为简单线性扫描。

**最坏情况**：仍为O(n+m)，例如模式串为"aaaaa"，文本串为"aaaabaaaaa"，虽然频繁失配，但总操作次数仍保持线性。

**稳定性**：算法是确定性的，对于相同的输入始终产生相同输出。

**正确性证明**：基于前缀函数的滑动引理——若P[0..k-1] = T[i-k..i-1]且P[k] ≠ T[i]，则下一个可能匹配位置为i - π[k-1]，该位置不会跳过任何有效匹配。

## 五、常见误区

1. **π[i]的含义混淆**：π[i]表示P[0..i]的最长公共前后缀长度，而非P[0..i]与P[0..i-1]的关系。初学者常误以为π[i]是前缀与整个字符串的比较结果。

2. **失配时j的更新错误**：当P[j]≠T[i]时，应令j=π[j-1]（当j>0时），而非j=π[j]或j=j-1。特别地，当j=0时直接i++。

3. **与暴力匹配的混淆**：KMP并非在所有情况下都比暴力匹配快。对于随机短模式串，暴力匹配的常数因子更小。KMP的优势在于模式串具有重复结构或文本串极长时。

4. **预处理与匹配的边界处理**：计算π表时，while循环的条件是k>0且P[k]≠P[q]，而非k≥0。匹配时，当j=m时表示找到完整匹配，此时应记录位置并令j=π[m-1]继续搜索。

5. **与Boyer-Moore算法的混淆**：BM算法从右向左匹配，利用坏字符和好后缀规则，在长模式串和字母表较大时通常更快；KMP从左向右匹配，理论保证线性时间，两者优化策略本质不同。
