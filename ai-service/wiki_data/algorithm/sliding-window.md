# 滑动窗口

## 一、定义
滑动窗口（Sliding Window）是一种在数组或字符串等线性结构上维护一个连续子区间（窗口）的算法技术。该窗口在数据结构上以固定或可变长度滑动，通过动态更新窗口内的信息来高效求解最优化或统计问题。其核心思想可形式化为：给定序列 \( A[0..n-1] \) 和窗口大小 \( k \)，定义窗口 \( W_i = \{A[i], A[i+1], ..., A[i+k-1]\} \)，滑动窗口算法通过增量更新 \( W_{i+1} \) 相对于 \( W_i \) 的变化来避免重复计算。

伪代码框架（可变窗口）：
```
left = 0
for right in range(n):
    更新窗口状态 (加入 A[right])
    while 窗口不满足约束条件:
        更新窗口状态 (移除 A[left])
        left += 1
    记录或更新结果
```

## 二、核心原理
1. **双指针协同**：滑动窗口通常由左右两个指针界定窗口边界。右指针负责扩展窗口以纳入新元素，左指针负责收缩窗口以移除元素，二者协同维持窗口的有效性。

2. **增量更新机制**：窗口滑动时，仅需处理移入和移出的两个边界元素，而非重新计算整个窗口。例如求和问题中，\( sum_{i+1} = sum_i - A[i] + A[i+k] \)，将时间复杂度从 \( O(nk) \) 降至 \( O(n) \)。

3. **单调性约束**：窗口的移动方向是单向的（从左到右），左右指针均不会回溯。这一性质保证了每个元素最多被加入和移除一次，是线性时间复杂度的理论基础。

4. **状态维护**：窗口内需要维护的数据结构（如哈希表、队列、计数器）必须支持高效的增删操作。常见维护内容包括：元素频率、和/积、最大值/最小值、不同元素个数等。

5. **适用条件**：滑动窗口适用于问题具有“单调性”或“连续性”特征，即窗口的扩展必然导致约束条件变严格（或变宽松），从而可以通过收缩窗口来恢复有效性。

## 三、过程/示例
**问题**：给定数组 `[2, 3, 1, 2, 4, 3]`，求元素和 ≥ 7 的最短连续子数组长度。

**步骤**：
1. 初始化 `left = 0`，`right = 0`，`sum = 0`，`min_len = ∞`。
2. `right = 0`：加入 `A[0]=2`，`sum = 2 < 7`，继续扩展。
3. `right = 1`：加入 `A[1]=3`，`sum = 5 < 7`，继续扩展。
4. `right = 2`：加入 `A[2]=1`，`sum = 6 < 7`，继续扩展。
5. `right = 3`：加入 `A[3]=2`，`sum = 8 ≥ 7`，记录 `min_len = 4`。收缩左边界：移除 `A[0]=2`，`sum = 6 < 7`，`left = 1`。
6. `right = 4`：加入 `A[4]=4`，`sum = 10 ≥ 7`，记录 `min_len = min(4, 4) = 4`。收缩：移除 `A[1]=3`，`sum = 7 ≥ 7`，记录 `min_len = min(4, 3) = 3`；再移除 `A[2]=1`，`sum = 6 < 7`，`left = 3`。
7. `right = 5`：加入 `A[5]=3`，`sum = 9 ≥ 7`，记录 `min_len = min(3, 3) = 3`。收缩：移除 `A[3]=2`，`sum = 7 ≥ 7`，记录 `min_len = min(3, 2) = 2`；再移除 `A[4]=4`，`sum = 3 < 7`，`left = 5`。
8. 遍历结束，返回 `min_len = 2`（子数组 `[4, 3]`）。

## 四、复杂度/性质分析
- **时间复杂度**：最好、最坏、平均均为 \( O(n) \)。每个元素最多被左指针和右指针各访问一次，增量更新操作为 \( O(1) \)。若窗口内维护复杂数据结构（如平衡树），则复杂度为 \( O(n \log k) \)。
- **空间复杂度**：通常为 \( O(1) \) 或 \( O(k) \)，取决于窗口内维护的数据结构。例如，维护字符频率需要 \( O(\Sigma) \) 空间（\(\Sigma\) 为字符集大小）。
- **稳定性**：滑动窗口算法本身是确定性的，对于相同输入总是产生相同输出。但若问题存在多个最优解，算法返回的解取决于实现细节（如是否优先记录更早出现的解）。
- **收敛性**：算法保证在有限步内终止。左右指针均单调递增，且右指针不超过序列长度 \( n \)，因此循环次数有限。
- **正确性**：基于“窗口收缩不会丢失最优解”的单调性原理。对于满足单调性的问题，滑动窗口能够遍历所有可能的有效窗口，从而保证找到全局最优解。

## 五、常见误区
1. **混淆固定窗口与可变窗口**：固定窗口（如求长度为k的子数组最大和）只需单指针滑动，而可变窗口需要双指针动态调整。初学者常将可变窗口问题错误地当作固定窗口处理。

2. **窗口收缩条件错误**：在可变窗口中，收缩条件必须严格对应问题的约束。例如“和≥target”时收缩，而非“和>target”时收缩，否则可能遗漏边界解。

3. **忘记更新结果的位置**：结果更新应在窗口满足约束条件时进行，而非在每次移动指针后。常见错误是在收缩循环内部更新结果，导致记录到无效窗口。

4. **与双指针/二分查找混淆**：滑动窗口是双指针的一种特殊形式，但双指针还包括相向指针（如两数之和）和快慢指针（如链表环检测）。滑动窗口强调窗口的连续性和单向移动，与二分查找的区间划分有本质区别。

5. **忽视窗口内数据结构的维护成本**：当窗口内需要维护最大值（如滑动窗口最大值问题）时，使用优先队列或单调队列的复杂度为 \( O(n \log k) \) 或 \( O(n) \)，而非简单的 \( O(n) \)。初学者容易低估维护成本。
