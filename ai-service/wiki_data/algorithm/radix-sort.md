# 基数排序

## 一、定义
基数排序（Radix Sort）是一种非比较型整数排序算法，它通过按位（digit-by-digit）分配和收集元素来完成排序。其核心思想是将待排序元素视为由多个数位组成的序列，从最低有效位（LSD）或最高有效位（MSD）开始，依次对每个数位执行稳定的排序（通常使用计数排序作为子过程）。基数排序的数学基础可表示为：对于一组整数，若其最大值为 \(d\) 位数，则经过 \(d\) 轮按位排序后，整个序列有序。

伪代码（LSD基数排序）：
```
RadixSort(A, d):
    for i = 0 to d-1:
        使用稳定排序（如计数排序）按第i位排序A
```

## 二、核心原理
1. **按位分解**：将排序问题分解为多个独立的子问题。每个元素被拆解为若干数位，排序过程仅关注当前位，忽略其他位的影响。
2. **稳定排序的必要性**：基数排序要求每轮子排序必须是稳定的（即相等元素的相对顺序不变）。这保证了高位排序后，低位已排好的顺序不会被破坏。
3. **LSD vs MSD**：LSD（Least Significant Digit）从最低位开始，逐步向高位推进，适合固定长度整数；MSD（Most Significant Digit）从最高位开始，通常需要递归处理桶内元素，适用于字符串或可变长度数据。
4. **桶分配机制**：每轮排序将元素分配到 \(k\) 个桶中（\(k\) 为基数，如十进制中 \(k=10\)），然后按桶顺序收集。桶的数量决定了算法的空间开销。
5. **线性时间特性**：由于每轮排序的时间复杂度为 \(O(n+k)\)，且轮数等于最大位数 \(d\)，总复杂度为 \(O(d(n+k))\)。当 \(d\) 和 \(k\) 为常数时，可视为线性时间。

## 三、过程/示例
以数组 `[329, 457, 657, 839, 436, 720, 355]` 为例，使用LSD基数排序（十进制，\(k=10\)）：

**第1轮（个位）**：
- 按个位数字分配到桶0-9：
  - 桶0: [720]
  - 桶5: [355]
  - 桶6: [436]
  - 桶7: [457, 657]
  - 桶9: [329, 839]
- 收集结果：`[720, 355, 436, 457, 657, 329, 839]`

**第2轮（十位）**：
- 按十位数字分配：
  - 桶2: [720, 329]
  - 桶3: [436, 839]
  - 桶5: [355, 457, 657]
- 收集结果：`[720, 329, 436, 839, 355, 457, 657]`

**第3轮（百位）**：
- 按百位数字分配：
  - 桶3: [329, 355]
  - 桶4: [436, 457]
  - 桶6: [657]
  - 桶7: [720]
  - 桶8: [839]
- 收集结果：`[329, 355, 436, 457, 657, 720, 839]`

最终数组有序。

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好/最坏/平均：\(O(d(n+k))\)，其中 \(d\) 为最大位数，\(k\) 为基数（如十进制 \(k=10\)，二进制 \(k=2\)）。当 \(d\) 和 \(k\) 为常数时，时间复杂度为 \(O(n)\)。
  - 与比较排序（如快速排序 \(O(n \log n)\)）相比，基数排序在特定条件下具有线性优势。
- **空间复杂度**：\(O(n+k)\)，主要来自每轮排序所需的桶数组和辅助存储。
- **稳定性**：稳定。前提是每轮子排序（如计数排序）是稳定的。
- **收敛性**：严格收敛。经过 \(d\) 轮排序后，序列必然完全有序，不存在概率性或渐进收敛问题。
- **适用性**：适用于整数、字符串等可分解为固定数位的数据。对于浮点数或负数，需进行预处理（如映射为非负整数）。

## 五、常见误区
1. **误认为基数排序是“比较排序”**：基数排序不通过元素间比较决定顺序，而是通过分配和收集，因此其复杂度下限不受 \(O(n \log n)\) 约束。
2. **忽略子排序的稳定性要求**：若使用不稳定的排序（如快速排序）作为子过程，则基数排序结果错误。必须使用计数排序、桶排序等稳定算法。
3. **混淆基数与进制**：基数 \(k\) 不一定是10，可以是2、16或256（字节排序）。选择更大的基数可减少轮数，但会增加每轮的空间开销。
4. **错误处理负数**：直接对负数按位排序会出错。常见解决方案包括：将所有数加上最小值的绝对值转为非负数，或使用补码表示法。
5. **认为MSD和LSD等价**：MSD需要递归处理每个桶，实现更复杂；LSD实现简单且稳定。MSD适用于可变长度字符串，而LSD更适合固定长度整数。
