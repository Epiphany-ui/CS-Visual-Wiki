# 编辑距离

## 一、定义
编辑距离（Edit Distance），又称Levenshtein距离，是衡量两个字符串之间差异程度的度量标准。它定义为将一个字符串转换为另一个字符串所需的最少单字符编辑操作次数。允许的编辑操作通常包括：插入一个字符、删除一个字符、替换一个字符。对于两个字符串A和B，其编辑距离记为ED(A, B)，满足以下递推关系：

设`dp[i][j]`表示字符串A的前i个字符与字符串B的前j个字符之间的编辑距离，则有：
```
dp[0][j] = j
dp[i][0] = i
dp[i][j] = min(
    dp[i-1][j] + 1,          // 删除A[i]
    dp[i][j-1] + 1,          // 插入B[j]
    dp[i-1][j-1] + cost      // 替换或匹配，若A[i]==B[j]则cost=0，否则cost=1
)
```
其中`1 ≤ i ≤ len(A)，1 ≤ j ≤ len(B)`。

## 二、核心原理
1. **动态规划思想**：编辑距离的计算基于最优子结构性质。两个字符串的编辑距离可以通过其子串的编辑距离递推得到，避免了重复计算。

2. **三种基本操作**：删除（从源字符串移除一个字符）、插入（向源字符串添加一个字符）、替换（将源字符串的一个字符改为目标字符串的对应字符）。每种操作代价均为1。

3. **边界条件处理**：当其中一个字符串为空时，编辑距离等于另一个字符串的长度（全部插入或删除操作）。这构成了动态规划表的初始行和列。

4. **代价函数设计**：匹配操作（字符相同）代价为0，替换操作（字符不同）代价为1。这种设计使得算法能够区分“无需操作”和“需要修改”的情况。

5. **最优性保证**：通过取三种操作的最小值，算法确保每一步都选择局部最优，最终累积得到全局最优解。这一性质由动态规划的最优性原理保证。

## 三、过程/示例
将字符串“KITTEN”转换为“SITTING”：

**步骤1：初始化动态规划表**
```
    ""  S  I  T  T  I  N  G
""  0  1  2  3  4  5  6  7
K   1
I   2
T   3
T   4
E   5
N   6
```

**步骤2：填充表格（部分关键步骤）**
- 位置(1,1)：比较K和S，不同，`cost=1`。`dp[1][1]=min(0+1, 1+1, 0+1)=1`
- 位置(2,2)：比较I和I，相同，`cost=0`。`dp[2][2]=min(1+1, 2+1, 1+0)=1`
- 位置(3,3)：比较T和T，相同，`cost=0`。`dp[3][3]=min(2+1, 3+1, 1+0)=1`
- 位置(4,4)：比较T和T，相同，`cost=0`。`dp[4][4]=min(1+1, 2+1, 1+0)=1`
- 位置(5,5)：比较E和I，不同，`cost=1`。`dp[5][5]=min(1+1, 2+1, 1+1)=2`
- 位置(6,6)：比较N和N，相同，`cost=0`。`dp[6][6]=min(2+1, 3+1, 2+0)=2`
- 位置(6,7)：比较N和G，不同，`cost=1`。`dp[6][7]=min(2+1, 3+1, 2+1)=3`

**步骤3：最终结果**
`dp[6][7]=3`，即“KITTEN”到“SITTING”的编辑距离为3（替换K→S，替换E→I，插入G）。

## 四、复杂度/性质分析
1. **时间复杂度**：最坏、最好、平均情况均为O(mn)，其中m和n分别为两个字符串的长度。因为需要填充整个二维表，每个单元格的计算时间为O(1)。

2. **空间复杂度**：基本实现为O(mn)。可通过滚动数组优化至O(min(m,n))，因为计算当前行仅依赖上一行和当前行前一列。

3. **空间优化性质**：使用两行数组（当前行和上一行）即可完成计算，空间复杂度降至O(min(m,n))。进一步优化可使用单行数组加额外变量保存左上角值。

4. **距离度量性质**：编辑距离满足三角不等式：ED(A,C) ≤ ED(A,B) + ED(B,C)。这使得它成为有效的度量空间，可用于聚类分析。

5. **对称性**：编辑距离是对称的，即ED(A,B)=ED(B,A)，因为逆操作（插入↔删除，替换↔替换）具有相同代价。

## 五、常见误区
1. **混淆编辑距离与汉明距离**：汉明距离仅适用于等长字符串，且只允许替换操作；编辑距离适用于任意长度字符串，支持插入、删除、替换三种操作。编辑距离是汉明距离的推广。

2. **忽略操作顺序的影响**：初学者常认为操作顺序不重要，但实际上不同顺序可能导致不同操作次数。动态规划保证找到最优顺序，而非任意顺序。

3. **错误理解替换操作**：替换操作等价于一次删除加一次插入，但代价为1而非2。这是编辑距离设计的关键——替换被视为单一原子操作。

4. **混淆编辑距离与最长公共子序列（LCS）**：编辑距离与LCS相关但不相同。对于仅允许插入和删除（不允许替换）的变体，ED(A,B)=len(A)+len(B)-2*LCS(A,B)。但标准编辑距离包含替换操作，关系更复杂。

5. **忽视字符权重差异**：标准编辑距离假设所有操作代价相同。实际应用中（如DNA序列比对），不同操作可能赋予不同权重（如替换代价更高）。初学者常忽略这一扩展可能性。
