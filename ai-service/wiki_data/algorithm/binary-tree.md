# 二叉树

## 一、定义

二叉树（Binary Tree）是一种特殊的树形数据结构，其中每个节点最多有两个子节点，分别称为左子节点和右子节点。从数学角度，二叉树可递归定义为一个有限节点集合，要么为空集（空树），要么由一个根节点和两棵互不相交的二叉树（左子树和右子树）组成。

形式化定义：设 \( T \) 为二叉树，则：
- \( T = \emptyset \)（空树），或
- \( T = (r, T_L, T_R) \)，其中 \( r \) 为根节点，\( T_L \) 和 \( T_R \) 分别为左子树和右子树。

在计算机科学中，二叉树的节点通常用如下结构表示（伪代码）：
```
class Node:
    value   // 节点值
    left    // 指向左子节点的指针/引用
    right   // 指向右子节点的指针/引用
```

## 二、核心原理

1. **递归结构**：二叉树的定义天然具有递归性——每个子树本身也是一棵二叉树。这种递归性质决定了大多数二叉树操作（如遍历、插入、删除）都适合用递归算法实现。

2. **层次与深度**：根节点位于第0层（或第1层，视约定而定）。节点的深度指从根到该节点的路径长度；树的高度指所有节点深度的最大值。一棵深度为 \( h \) 的二叉树最多有 \( 2^{h+1} - 1 \) 个节点（满二叉树情形）。

3. **遍历方式**：二叉树的遍历分为深度优先（DFS）和广度优先（BFS）。DFS包括前序（根-左-右）、中序（左-根-右）、后序（左-右-根）三种；BFS即层序遍历，按层次从上到下、从左到右访问节点。

4. **特殊形态**：满二叉树（所有非叶子节点都有两个子节点，且叶子节点在同一层）、完全二叉树（除最后一层外全满，最后一层节点从左到右排列）、二叉搜索树（BST，左子树所有节点值 < 根节点值 < 右子树所有节点值）等，各有不同的性质和应用场景。

5. **存储方式**：可用链式存储（节点含左右指针）或顺序存储（用数组，适用于完全二叉树，节点 \( i \) 的左子为 \( 2i+1 \)，右子为 \( 2i+2 \)）。

## 三、过程/示例

以构建一棵二叉搜索树（BST）为例，给定数组 `[5, 3, 7, 2, 4, 6, 8]`，演示插入过程：

1. **插入5**：树为空，5成为根节点。
2. **插入3**：3 < 5，进入左子树（空），3成为5的左子节点。
3. **插入7**：7 > 5，进入右子树（空），7成为5的右子节点。
4. **插入2**：2 < 5 → 左子树（节点3），2 < 3 → 进入3的左子树（空），2成为3的左子节点。
5. **插入4**：4 < 5 → 左子树（节点3），4 > 3 → 进入3的右子树（空），4成为3的右子节点。
6. **插入6**：6 > 5 → 右子树（节点7），6 < 7 → 进入7的左子树（空），6成为7的左子节点。
7. **插入8**：8 > 5 → 右子树（节点7），8 > 7 → 进入7的右子树（空），8成为7的右子节点。

最终树结构：
```
        5
       / \
      3   7
     / \ / \
    2  4 6  8
```
对该树进行中序遍历（左-根-右），输出顺序为 `2, 3, 4, 5, 6, 7, 8`，即有序序列——这正是BST的核心性质。

## 四、复杂度/性质分析

1. **时间复杂度**：
   - **查找/插入/删除**（针对BST）：最好情况 \( O(\log n) \)（树平衡时），最坏情况 \( O(n) \)（树退化为链表时，如按升序插入）。平均情况 \( O(\log n) \)（假设随机插入）。
   - **遍历**：前序、中序、后序、层序遍历均为 \( O(n) \)，因为每个节点恰好访问一次。

2. **空间复杂度**：
   - 递归遍历：最坏 \( O(h) \)，\( h \) 为树高（递归调用栈深度）。平衡树 \( O(\log n) \)，退化树 \( O(n) \)。
   - 层序遍历（使用队列）：\( O(w) \)，\( w \) 为树的最大宽度，最坏 \( O(n) \)。
   - 存储结构：链式存储 \( O(n) \)，顺序存储（完全二叉树）\( O(n) \)。

3. **稳定性**：二叉树本身不是排序算法，但作为数据结构，BST的插入操作不保证稳定性（相等元素处理方式取决于实现）。

4. **收敛性**：在平衡二叉树（如AVL树、红黑树）中，通过旋转操作保证树高始终为 \( O(\log n) \)，从而保证操作效率收敛于对数级别。

## 五、常见误区

1. **二叉树与二叉搜索树混淆**：二叉树只规定节点度数不超过2，不要求任何顺序关系；二叉搜索树是二叉树的一种特殊形式，要求左<根<右。所有BST都是二叉树，反之不成立。

2. **树的高度与深度混淆**：高度通常指从根到最远叶子节点的边数（或节点数，视定义），深度指从根到某节点的边数。不同教材定义可能不同，需注意上下文。

3. **满二叉树与完全二叉树混淆**：满二叉树要求所有非叶子节点都有两个子节点；完全二叉树只要求除最后一层外全满，且最后一层节点从左到右连续排列。完全二叉树不一定是满的。

4. **递归遍历与迭代遍历混淆**：递归实现简洁但可能栈溢出（深度过大时）；迭代实现（显式使用栈）更复杂但可控。初学者常误以为递归是唯一方式。

5. **二叉树的存储误解**：并非所有二叉树都适合用数组存储。顺序存储仅对完全二叉树高效（无空间浪费），对一般二叉树会留下大量空位，造成空间浪费。链式存储是通用方案。
