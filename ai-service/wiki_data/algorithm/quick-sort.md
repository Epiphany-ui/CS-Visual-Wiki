# 快速排序

## 一、定义
快速排序（Quicksort）是一种基于分治策略的比较排序算法。其核心思想是：从待排序序列中选取一个元素作为“基准”（pivot），通过一趟扫描将序列划分为两个子序列，使得左子序列所有元素均不大于基准，右子序列所有元素均不小于基准，然后递归地对两个子序列进行排序。其核心递归关系可表示为：

\[
\text{Quicksort}(A, p, r) = 
\begin{cases} 
\text{void}, & \text{if } p \geq r \\
\text{Partition}(A, p, r) + \text{Quicksort}(A, p, q-1) + \text{Quicksort}(A, q+1, r), & \text{otherwise}
\end{cases}
\]

其中，Partition 函数返回基准元素最终所在的位置索引 \(q\)。

## 二、核心原理
1. **分治思想**：快速排序将原问题分解为两个规模更小的子问题（左右子序列），子问题相互独立且与原问题性质相同，通过递归求解完成整体排序。
2. **基准选择**：基准的选取直接影响算法性能。常见策略包括：固定选取第一个或最后一个元素、随机选取、三数取中法（median-of-three）等。
3. **划分过程**：核心操作是 Partition，它通过双指针（或单向扫描）遍历序列，将小于基准的元素交换到左侧，大于基准的交换到右侧，最终将基准置于正确位置。
4. **原地排序**：快速排序是原地排序算法，除递归调用栈外，仅需常数级别的额外空间。
5. **不稳定性**：由于划分过程中元素可能发生远距离交换，相等元素的相对顺序无法保证，因此快速排序是不稳定的排序算法。

## 三、过程/示例
以数组 `[3, 7, 8, 5, 2, 1, 9, 5, 4]` 为例，采用“取最后一个元素为基准”的经典 Lomuto 划分方案。

**初始状态**：`[3, 7, 8, 5, 2, 1, 9, 5, 4]`，基准 pivot = 4。

**第一趟划分**：
- 设置 i = p-1 = -1，j 从 p 到 r-1 遍历。
- j=0 (3)：3 ≤ 4，i++ 得 i=0，交换 A[0] 与 A[0]（不变）。
- j=1 (7)：7 > 4，不操作。
- j=2 (8)：8 > 4，不操作。
- j=3 (5)：5 > 4，不操作。
- j=4 (2)：2 ≤ 4，i++ 得 i=1，交换 A[1] 与 A[4] → `[3, 2, 8, 5, 7, 1, 9, 5, 4]`。
- j=5 (1)：1 ≤ 4，i++ 得 i=2，交换 A[2] 与 A[5] → `[3, 2, 1, 5, 7, 8, 9, 5, 4]`。
- j=6 (9)：9 > 4，不操作。
- j=7 (5)：5 > 4，不操作。
- 遍历结束，交换 A[i+1] 与 A[r]：i+1=3，交换 A[3] (5) 与 A[8] (4) → `[3, 2, 1, 4, 7, 8, 9, 5, 5]`。
- 基准 4 已归位，返回 q=3。

**递归左子序列** `[3, 2, 1]`：
- 基准=1，划分后得 `[1, 2, 3]`，基准归位。
- 递归左子序列 `[]`（空），右子序列 `[2, 3]`。
- 对 `[2, 3]`：基准=3，划分后得 `[2, 3]`，基准归位。
- 递归左子序列 `[2]`（单元素，已有序）。

**递归右子序列** `[7, 8, 9, 5, 5]`：
- 基准=5，划分后得 `[5, 5, 7, 8, 9]`，基准归位。
- 递归左子序列 `[5]`（单元素），右子序列 `[7, 8, 9]`。
- 对 `[7, 8, 9]`：基准=9，划分后得 `[7, 8, 9]`，递归完成。

**最终结果**：`[1, 2, 3, 4, 5, 5, 7, 8, 9]`。

## 四、复杂度/性质分析
- **时间复杂度**：
  - **最好情况**：每次划分都能将序列均匀分割，递归树高度为 \( \log n \)，每层划分总代价为 \( O(n) \)，总时间复杂度为 \( O(n \log n) \)。
  - **最坏情况**：每次划分都极度不平衡（如序列已有序且固定选首尾为基准），递归树高度为 \( n \)，总时间复杂度为 \( O(n^2) \)。
  - **平均情况**：对于随机排列的输入，期望时间复杂度为 \( O(n \log n) \)。
- **空间复杂度**：递归调用栈的深度决定空间开销。最好情况 \( O(\log n) \)，最坏情况 \( O(n) \)。通过尾递归优化可将最坏空间降至 \( O(\log n) \)。
- **稳定性**：不稳定。例如序列 `[2, 2, 1]`，以最后一个元素 1 为基准划分后，两个 2 的相对顺序可能改变。
- **收敛性**：快速排序保证有限步内终止，因为每次递归调用处理的子序列长度严格小于原序列。

## 五、常见误区
1. **误认为快速排序总是比归并排序快**：快速排序的平均性能虽优，但在最坏情况下退化为 \( O(n^2) \)，而归并排序始终为 \( O(n \log n) \)。实际应用中，快速排序的常数因子更小，但需注意避免最坏情况。
2. **混淆“原地排序”与“稳定性”**：快速排序是原地排序（额外空间少），但并非稳定排序。初学者常将两者等同。
3. **忽视基准选择的影响**：固定选取首尾元素在有序或逆序输入下会导致最坏情况。随机化选择或三数取中法可有效缓解。
4. **错误理解 Partition 的边界条件**：在实现时，容易混淆指针的初始值、循环终止条件以及最终交换基准的位置，导致数组越界或排序错误。
5. **误认为快速排序只能用于数组**：快速排序同样适用于链表，但 Partition 的实现方式需调整（如使用头插法或尾插法），且随机访问特性不如数组高效。
