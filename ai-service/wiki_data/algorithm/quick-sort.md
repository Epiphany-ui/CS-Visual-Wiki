# 快速排序

## 一、定义
快速排序（Quicksort）是一种基于分治策略的比较排序算法。其核心思想是：从待排序序列中选取一个元素作为“基准”（pivot），通过一趟扫描将序列划分为两个子序列，使得左子序列所有元素均不大于基准，右子序列所有元素均不小于基准，然后递归地对两个子序列进行同样的操作，直至整个序列有序。

快速排序的递归定义可表示为：
\[
\text{Quicksort}(A, l, r) = 
\begin{cases} 
\text{void}, & l \ge r \\
\text{Partition}(A, l, r) \to q, & \text{然后} \\
\text{Quicksort}(A, l, q-1), & \text{Quicksort}(A, q+1, r)
\end{cases}
\]

其中，Partition 函数返回基准元素的最终位置 \(q\)，并完成原地划分。

## 二、核心原理
1. **分治思想**：快速排序将原问题分解为两个规模更小的子问题，子问题相互独立且与原问题性质相同，通过递归求解子问题并合并结果得到原问题的解。
2. **基准选择**：基准的选择直接影响算法性能。常见策略包括固定选取第一个或最后一个元素、随机选取、三数取中法（选取左、中、右三个元素的中位数）等。
3. **划分过程**：划分是快速排序的核心操作。经典实现使用两个指针从两端向中间扫描，将小于基准的元素交换到左侧，大于基准的元素交换到右侧，最终将基准放到正确位置。
4. **原地排序**：快速排序通过交换元素实现原地排序，无需额外存储空间（递归栈除外），空间效率高。
5. **不稳定性**：快速排序是不稳定排序算法，即相等元素的相对顺序在排序后可能发生改变。

## 三、过程/示例
以数组 `[5, 3, 8, 4, 2, 7, 1, 6]` 为例，演示快速排序的完整过程（选取最后一个元素为基准）：

**初始数组**：`[5, 3, 8, 4, 2, 7, 1, 6]`，基准 = 6

**第一趟划分**：
- 左指针 i 从 0 开始，右指针 j 从 6 开始（排除基准）
- i 向右扫描，找到第一个大于基准的元素 8（位置2）
- j 向左扫描，找到第一个小于基准的元素 1（位置6）
- 交换 8 和 1：`[5, 3, 1, 4, 2, 7, 8, 6]`
- 继续扫描，i 找到 7（位置5），j 找到 2（位置4），此时 i > j，停止
- 将基准 6 与 i 位置的元素 7 交换：`[5, 3, 1, 4, 2, 6, 8, 7]`
- 基准 6 最终位于位置5，左子数组 `[5, 3, 1, 4, 2]`，右子数组 `[8, 7]`

**递归左子数组** `[5, 3, 1, 4, 2]`，基准 = 2：
- 划分后：`[1, 2, 3, 4, 5]`，基准 2 位于位置1
- 左子数组 `[1]`（已有序），右子数组 `[3, 4, 5]`

**递归右子数组** `[3, 4, 5]`，基准 = 5：
- 划分后：`[3, 4, 5]`，基准 5 位于位置4
- 左子数组 `[3, 4]`，继续递归

**递归子数组** `[3, 4]`，基准 = 4：
- 划分后：`[3, 4]`，基准 4 位于位置3
- 左子数组 `[3]`（已有序）

**递归右子数组** `[8, 7]`，基准 = 7：
- 划分后：`[7, 8]`，基准 7 位于位置6
- 左子数组 `[8]`（已有序）

**最终结果**：`[1, 2, 3, 4, 5, 6, 7, 8]`

## 四、复杂度/性质分析
1. **时间复杂度**：
   - **最好情况**：每次划分都将序列均匀分割，递归深度为 \(\log_2 n\)，每层划分需 \(O(n)\) 时间，总时间复杂度为 \(O(n \log n)\)。
   - **最坏情况**：每次划分都产生一个空子序列（如序列已有序且固定选首尾为基准），递归深度为 \(n\)，总时间复杂度为 \(O(n^2)\)。
   - **平均情况**：对于随机排列的输入，期望时间复杂度为 \(O(n \log n)\)，证明基于递归期望分析。

2. **空间复杂度**：快速排序是原地排序，但递归调用需要栈空间。最好情况 \(O(\log n)\)，最坏情况 \(O(n)\)。通过尾递归优化可将最坏空间复杂度降至 \(O(\log n)\)。

3. **稳定性**：快速排序是不稳定的。例如，数组 `[5, 3, 3', 2]` 以 3 为基准划分时，两个 3 的相对顺序可能改变。

4. **收敛性**：快速排序的递归过程保证有限步内终止，因为每次划分至少将基准元素放到最终位置，子序列规模严格递减。

## 五、常见误区
1. **混淆快速排序与归并排序**：两者都基于分治，但快速排序是原地排序且不稳定，归并排序需要额外空间但稳定。快速排序的划分操作在递归前进行，而归并排序的合并操作在递归后进行。

2. **误认为快速排序总是 \(O(n \log n)\)**：最坏情况 \(O(n^2)\) 确实存在，但通过随机化基准选择或三数取中法，实际应用中几乎不会出现最坏情况。

3. **忽略递归深度对空间的影响**：初学者常认为快速排序空间复杂度为 \(O(1)\)，实际上递归栈空间不可忽视，最坏情况下可能达到 \(O(n)\)。

4. **错误理解划分的终止条件**：经典 Lomuto 划分与 Hoare 划分的指针移动和终止条件不同，混淆可能导致无限循环或越界。例如，Hoare 划分中两个指针相遇时需特殊处理。

5. **认为快速排序适用于所有数据类型**：对于链表等非随机访问结构，快速排序效率较低，通常选择归并排序。此外，对于包含大量重复元素的数组，标准快速排序性能会退化，需采用三路划分优化。
