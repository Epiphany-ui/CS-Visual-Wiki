# 计数排序

## 一、定义
计数排序（Counting Sort）是一种非比较型整数排序算法，它通过统计每个元素出现的次数，利用元素的值直接确定其在输出数组中的位置，从而实现排序。其核心数学基础是前缀和（prefix sum）技术：给定输入数组 \(A[1..n]\)，其中元素均为非负整数且范围已知为 \([0, k]\)，计数排序首先统计每个值 \(v\) 出现的次数 \(C[v]\)，然后计算前缀和 \(C'[v] = \sum_{i=0}^{v} C[i]\)，最后根据 \(C'[v]\) 将每个元素放置到输出数组的正确位置。伪代码如下：

```
CountingSort(A, k)
    let C[0..k] be new array
    for i = 0 to k
        C[i] = 0
    for j = 1 to A.length
        C[A[j]] = C[A[j]] + 1
    // C[i] now contains the number of elements equal to i
    for i = 1 to k
        C[i] = C[i] + C[i-1]
    // C[i] now contains the number of elements ≤ i
    let B[1..A.length] be new array
    for j = A.length downto 1
        B[C[A[j]]] = A[j]
        C[A[j]] = C[A[j]] - 1
    return B
```

## 二、核心原理
1. **基于值的直接映射**：计数排序不通过比较元素大小来决定顺序，而是将元素的值作为索引，直接映射到计数数组的对应位置。这使得它能够突破比较排序 \(\Omega(n \log n)\) 的下界。

2. **计数与累加**：第一步统计每个值出现的频次；第二步通过前缀和将频次转换为每个值在输出数组中的“最后一个位置”。例如，若值 \(v\) 的累计计数为 \(p\)，则所有等于 \(v\) 的元素应占据输出数组的第 \(1\) 到第 \(p\) 个位置。

3. **反向遍历保证稳定性**：在将元素放入输出数组时，算法从原数组的末尾向前遍历。这一设计确保当多个元素具有相同值时，后出现的元素被放置在输出数组中更靠后的位置，从而维持了它们在原数组中的相对顺序（稳定性）。

4. **整数范围限制**：计数排序要求输入元素必须是整数，且取值范围 \(k\) 不能太大。当 \(k = O(n)\) 时，算法效率极高；若 \(k\) 远大于 \(n\)，则空间和时间开销会急剧增加。

5. **非原地排序**：计数排序需要额外的计数数组和输出数组，因此不是原地排序算法。

## 三、过程/示例
假设输入数组 \(A = [4, 2, 2, 8, 3, 3, 1]\)，元素范围 \([0, 8]\)，即 \(k=8\)。

**步骤1：初始化计数数组**  
创建 \(C[0..8]\)，全部置0。

**步骤2：统计频次**  
遍历 \(A\)：  
- \(A[1]=4\) → \(C[4] = 1\)  
- \(A[2]=2\) → \(C[2] = 1\)  
- \(A[3]=2\) → \(C[2] = 2\)  
- \(A[4]=8\) → \(C[8] = 1\)  
- \(A[5]=3\) → \(C[3] = 1\)  
- \(A[6]=3\) → \(C[3] = 2\)  
- \(A[7]=1\) → \(C[1] = 1\)  

得到 \(C = [0,1,2,2,1,0,0,0,1]\)。

**步骤3：计算前缀和**  
\(C[0]=0\)  
\(C[1]=0+1=1\)  
\(C[2]=1+2=3\)  
\(C[3]=3+2=5\)  
\(C[4]=5+1=6\)  
\(C[5]=6+0=6\)  
\(C[6]=6+0=6\)  
\(C[7]=6+0=6\)  
\(C[8]=6+1=7\)  

此时 \(C[i]\) 表示 ≤ i 的元素总数。

**步骤4：反向遍历放置元素**  
创建输出数组 \(B[1..7]\)，从 \(j=7\) 开始：  
- \(j=7, A[7]=1\)，\(C[1]=1\) → \(B[1]=1\)，\(C[1]=0\)  
- \(j=6, A[6]=3\)，\(C[3]=5\) → \(B[5]=3\)，\(C[3]=4\)  
- \(j=5, A[5]=3\)，\(C[3]=4\) → \(B[4]=3\)，\(C[3]=3\)  
- \(j=4, A[4]=8\)，\(C[8]=7\) → \(B[7]=8\)，\(C[8]=6\)  
- \(j=3, A[3]=2\)，\(C[2]=3\) → \(B[3]=2\)，\(C[2]=2\)  
- \(j=2, A[2]=2\)，\(C[2]=2\) → \(B[2]=2\)，\(C[2]=1\)  
- \(j=1, A[1]=4\)，\(C[4]=6\) → \(B[6]=4\)，\(C[4]=5\)  

最终 \(B = [1,2,2,3,3,4,8]\)，排序完成。

## 四、复杂度/性质分析
- **时间复杂度**：最好、最坏、平均情况均为 \(\Theta(n + k)\)，其中 \(n\) 为元素个数，\(k\) 为元素取值范围。当 \(k = O(n)\) 时，时间复杂度为 \(\Theta(n)\)，优于任何比较排序。
- **空间复杂度**：需要额外 \(O(k)\) 的计数数组和 \(O(n)\) 的输出数组，总空间复杂度为 \(O(n + k)\)。
- **稳定性**：稳定排序。反向遍历保证了相同值元素的相对顺序不变。
- **收敛性**：不适用。计数排序是确定性算法，不存在迭代收敛过程。
- **适用条件**：仅适用于整数或可映射为整数的数据，且取值范围不宜过大。对于浮点数或字符串，需先转换为整数键值。

## 五、常见误区
1. **误认为计数排序是原地排序**：许多初学者认为计数排序仅需一个计数数组，但实际上它需要额外的输出数组来存放排序结果，因此不是原地算法。

2. **混淆计数排序与桶排序**：计数排序是桶排序的一种特例——每个桶只存放一个值的所有元素。桶排序将数据分到多个桶中再分别排序，而计数排序直接利用值作为桶索引。

3. **忽略取值范围对性能的影响**：当 \(k\) 远大于 \(n\) 时（如对 \([1, 10^9]\) 内的10个整数排序），计数排序会浪费大量空间和时间，此时应改用比较排序。

4. **错误理解稳定性实现**：部分实现采用正向遍历放置元素，这会导致相同值元素的相对顺序反转，破坏稳定性。正确的做法是反向遍历原数组。

5. **认为计数排序只能处理非负整数**：通过偏移映射，计数排序可以处理负整数（如将所有元素加上最小值的绝对值），但前提是偏移后的范围仍可接受。
