# 栈

## 一、定义

栈（Stack）是一种受限的线性数据结构，其核心特征是**后进先出**（Last In, First Out, LIFO）。在栈中，所有元素的插入（入栈，Push）和删除（出栈，Pop）操作都只能在栈顶（Top）进行。栈底（Bottom）是固定的，不允许直接访问。

从抽象数据类型（ADT）的角度，栈可形式化定义为一个二元组 `(S, op)`，其中 `S` 是元素集合，`op` 是操作集合，主要包括：
- `Push(S, x)`: 将元素 `x` 插入栈顶。
- `Pop(S)`: 删除并返回栈顶元素。
- `Top(S)`: 返回栈顶元素但不删除。
- `IsEmpty(S)`: 判断栈是否为空。

若用数组实现，栈的数学状态可用栈顶指针 `top` 描述：`top` 指向当前栈顶元素的位置。初始时 `top = -1` 表示空栈。入栈时 `top ← top + 1`，出栈时 `top ← top - 1`。栈中元素个数为 `top + 1`。

## 二、核心原理

1. **LIFO 机制**：栈的核心思想是“后进先出”。最后进入栈的元素总是最先被取出。这一特性使得栈天然适合处理需要“回溯”或“撤销”的场景，如函数调用、括号匹配、表达式求值等。

2. **单端操作**：所有操作（插入、删除、读取）均限定在栈顶。这意味着栈无法像数组或链表那样随机访问中间元素。这种限制虽然降低了灵活性，但换来了操作的高效性和确定性。

3. **栈顶指针**：在数组实现中，栈顶指针 `top` 是核心状态变量。它记录了当前栈中元素的数量和位置。入栈时 `top` 增加，出栈时 `top` 减少。当 `top = -1` 时栈空，当 `top = MAX_SIZE - 1` 时栈满（若容量固定）。

4. **递归与栈的等价性**：任何递归算法都可以用栈来模拟。递归调用本质上是系统隐式维护了一个“调用栈”，每次函数调用将返回地址和局部变量压栈，返回时出栈。因此，栈是理解递归、深度优先搜索（DFS）等算法的基础。

5. **栈的两种实现**：栈可以用数组（顺序栈）或链表（链栈）实现。顺序栈访问快但容量固定；链栈动态扩容但需额外指针空间。两者在逻辑上完全等价，仅实现细节不同。

## 三、过程/示例

以括号匹配问题为例：检查字符串 `"({[]})"` 中的括号是否匹配。

**步骤演示**（栈初始为空，`top = -1`）：

1. 读取 `'('`：栈空，入栈。栈内容：`[ '(' ]`，`top = 0`。
2. 读取 `'{'`：栈顶为 `'('`，不是匹配对，入栈。栈内容：`[ '(', '{' ]`，`top = 1`。
3. 读取 `'['`：栈顶为 `'{'`，入栈。栈内容：`[ '(', '{', '[' ]`，`top = 2`。
4. 读取 `']'`：栈顶为 `'['`，匹配，出栈。栈内容：`[ '(', '{' ]`，`top = 1`。
5. 读取 `'}'`：栈顶为 `'{'`，匹配，出栈。栈内容：`[ '(' ]`，`top = 0`。
6. 读取 `')'`：栈顶为 `'('`，匹配，出栈。栈内容：`[]`，`top = -1`。
7. 字符串遍历结束，栈为空，匹配成功。

若输入为 `"({[})"`，则在第5步读取 `'}'` 时，栈顶为 `'['`，不匹配，立即报错。该过程展示了栈如何通过 LIFO 特性确保括号的嵌套顺序正确。

## 四、复杂度/性质分析

- **时间复杂度**：
  - 入栈（Push）、出栈（Pop）、取栈顶（Top）、判空（IsEmpty）：均为 **O(1)**，即常数时间。这是栈最突出的优势——所有基本操作与栈中元素个数无关。
  - 注意：若栈用动态数组实现，当容量不足时需扩容（通常翻倍），此时单次 Push 可能为 O(n)，但均摊后仍为 O(1)。

- **空间复杂度**：
  - 顺序栈：O(n)，其中 n 为栈的最大容量（或当前元素个数）。
  - 链栈：O(n)，每个节点额外存储指针，常数因子略大。

- **稳定性**：栈的操作不涉及元素比较或交换，因此“稳定性”概念不适用。但若从元素相对顺序看，栈的 LIFO 特性会反转元素顺序（先入栈的后出栈），这与队列的 FIFO 形成对比。

- **收敛性**：栈本身不涉及迭代或数值逼近，无收敛性问题。但在某些算法（如深度优先搜索）中，栈的深度决定了递归或搜索的深度，若栈空间不足可能导致栈溢出（Stack Overflow）。

## 五、常见误区

1. **混淆栈与队列**：栈是 LIFO（后进先出），队列是 FIFO（先进先出）。初学者常将两者混用，尤其在实现 BFS 和 DFS 时。记住：DFS 用栈（或递归），BFS 用队列。

2. **误认为栈可以随机访问**：栈只允许访问栈顶元素。试图通过下标访问栈中其他元素（如 `stack[2]`）是错误用法，违背栈的抽象定义。若需随机访问，应使用数组或列表。

3. **栈溢出与内存泄漏混淆**：栈溢出（Stack Overflow）通常指递归过深或局部变量过多导致系统调用栈空间耗尽，是运行时错误。而内存泄漏是堆内存未释放。两者机制不同，但初学者常混为一谈。

4. **忽视栈顶指针的边界条件**：在数组实现中，出栈前必须检查栈是否为空（`top >= 0`），入栈前必须检查栈是否已满（`top < MAX_SIZE - 1`）。忽略边界检查会导致数组越界或数据丢失。

5. **认为栈只能用于基本类型**：栈可以存储任意类型的数据，包括结构体、对象、函数指针等。在编译器中，栈用于存储函数调用帧（包含局部变量、返回地址等），这是栈在系统编程中的核心应用。
