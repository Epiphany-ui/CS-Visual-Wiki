# Bellman-Ford算法

## 一、定义

Bellman-Ford算法是一种用于求解**单源最短路径问题**的图论算法，由Richard Bellman和Lester Ford Jr.分别独立提出。与Dijkstra算法不同，Bellman-Ford算法能够处理图中存在**负权边**的情况，并能检测图中是否存在**负权回路**（即总权值为负的环）。

算法的核心思想是：对图中的所有边进行**V-1次**松弛操作（其中V为顶点数），每次松弛操作尝试通过一条边来缩短从源点到某个顶点的距离。若在第V次松弛后仍能更新距离，则说明图中存在负权回路。

**核心数学公式（松弛操作）：**

对于边 (u, v) 且权值为 w(u, v)，若 `dist[v] > dist[u] + w(u, v)`，则更新 `dist[v] = dist[u] + w(u, v)`。

**伪代码：**
```
初始化 dist[源点] = 0，其余 dist = ∞
for i = 1 to |V| - 1:
    for each edge (u, v) with weight w:
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
for each edge (u, v) with weight w:
    if dist[u] + w < dist[v]:
        return "存在负权回路"
return dist[]
```

## 二、核心原理

1. **动态规划思想**：Bellman-Ford算法本质上是动态规划在最短路径问题上的应用。`dist[v]` 表示从源点到顶点 v 的**最多经过 k 条边**的最短路径长度。外层循环每迭代一次，就允许路径中多包含一条边。

2. **松弛操作的传递性**：松弛操作是算法的基本单元。通过不断尝试用更短的路径替代当前已知路径，距离值逐渐收敛到真实最短路径。由于最短路径最多包含 V-1 条边（无环情况下），因此 V-1 次松弛足以保证所有顶点都找到最优解。

3. **负权回路检测机制**：算法在 V-1 次松弛后，再对所有边进行一次额外扫描。如果此时还能更新某个顶点的距离，则说明存在一条从源点可达的负权回路——因为理论上最短路径不应包含回路，而负权回路可以无限降低路径长度。

4. **对负权边的兼容性**：与Dijkstra算法的贪心策略不同，Bellman-Ford不依赖边权非负的假设。它通过全局多次松弛，允许负权边在后续迭代中“修正”之前被正权边“误导”的路径。

5. **无向图处理**：对于无向图中的负权边，算法会自动将其视为两条方向相反的有向边。但需注意，无向图中的负权边会立即构成负权回路（来回走一次即可），因此算法会正确检测并报告。

## 三、过程/示例

考虑以下有向图，顶点集为 {A, B, C, D}，源点为 A：
- A → B: 权值 4
- A → C: 权值 2
- B → C: 权值 -3
- C → D: 权值 3
- B → D: 权值 5

**初始化：**
dist[A] = 0, dist[B] = ∞, dist[C] = ∞, dist[D] = ∞

**第1次松弛（按边序处理）：**
- A→B: dist[B] = min(∞, 0+4) = 4
- A→C: dist[C] = min(∞, 0+2) = 2
- B→C: dist[C] = min(2, 4+(-3)) = 1
- C→D: dist[D] = min(∞, 2+3) = 5
- B→D: dist[D] = min(5, 4+5) = 5
结果：dist = [0, 4, 1, 5]

**第2次松弛：**
- A→B: dist[B] = min(4, 0+4) = 4
- A→C: dist[C] = min(1, 0+2) = 1
- B→C: dist[C] = min(1, 4+(-3)) = 1
- C→D: dist[D] = min(5, 1+3) = 4
- B→D: dist[D] = min(4, 4+5) = 4
结果：dist = [0, 4, 1, 4]

**第3次松弛（V-1=3次，顶点数4）：**
所有边均无法更新距离，dist 保持不变。

**负权回路检测（第4次松弛）：**
所有边均无法更新，无负权回路。

最终最短路径：A→B=4, A→C=1（经B）, A→D=4（经C）

## 四、复杂度/性质分析

**时间复杂度：**
- 最坏情况：O(V·E)，其中 V 为顶点数，E 为边数。需要进行 V-1 次外层循环，每次内层循环遍历所有 E 条边。
- 最好情况：O(E)，若图在第一次松弛后即收敛（如拓扑有序的DAG），但标准实现仍需 V-1 次循环。
- 平均情况：O(V·E)，与最坏情况相同。

**空间复杂度：** O(V)，仅需存储每个顶点的距离值。若需记录路径，则额外 O(V) 存储前驱节点。

**收敛性：** 算法保证在 V-1 次松弛后收敛到正确解（若无负权回路）。收敛速度取决于边的处理顺序，优化版本（如SPFA）使用队列可加速。

**稳定性：** 算法本身不保证稳定性，因为距离更新可能改变多个顶点的值，但最终结果唯一确定。

**与Dijkstra对比：** Bellman-Ford支持负权边但时间复杂度更高（O(VE) vs O(E+VlogV)），且能检测负权回路。

## 五、常见误区

1. **误认为V-1次松弛后所有距离一定正确**：只有在**不存在负权回路**的前提下，V-1次松弛才能保证正确性。若存在负权回路，距离可能无限减小，算法仅能检测其存在。

2. **混淆负权边与负权回路**：图中存在负权边是允许的，只要不构成负权回路，算法就能正确求解。负权回路是指**环的总权值为负**，而非单条边为负。

3. **忽视边的处理顺序对收敛速度的影响**：虽然最终结果与边序无关，但合理的边序（如按拓扑序）可减少松弛轮数。最坏情况下，若边序恰好与最短路径方向相反，可能需要V-1次才能收敛。

4. **误以为算法适用于所有图类型**：Bellman-Ford要求图是**有向图**。对于无向图，负权边会立即形成负权回路（来回走一次），算法会报告错误，但此时问题本身无解。

5. **与Floyd-Warshall算法混淆**：Bellman-Ford解决**单源**最短路径，而Floyd-Warshall解决**全源**最短路径。后者时间复杂度O(V³)，但能处理负权边（不能有负权回路）。两者应用场景不同。
