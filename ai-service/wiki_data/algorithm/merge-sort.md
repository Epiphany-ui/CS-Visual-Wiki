# 归并排序

## 一、定义
归并排序（Merge Sort）是一种基于分治策略的稳定排序算法。它将待排序序列递归地分成两个子序列，分别排序后再合并成一个有序序列。其核心操作是“合并两个已排序数组”，这一过程保证合并结果的有序性。

归并排序的递归形式可表示为：
```
MergeSort(A, p, r):
    if p < r:
        q = floor((p + r) / 2)
        MergeSort(A, p, q)
        MergeSort(A, q+1, r)
        Merge(A, p, q, r)
```
其中 `Merge(A, p, q, r)` 将已排序的 `A[p..q]` 和 `A[q+1..r]` 合并为有序的 `A[p..r]`。

## 二、核心原理
1. **分治思想**：将原问题分解为规模更小但结构相同的子问题，递归求解后合并结果。归并排序严格遵循“分-治-合”三阶段。
2. **合并操作**：合并两个有序数组时，使用双指针分别指向两个子数组的起始位置，比较当前元素，将较小者放入结果数组，直至一方耗尽，再将剩余元素全部复制。该操作保证合并后的数组全局有序。
3. **递归深度**：每次将序列对半分割，递归深度为 `⌈log₂n⌉`，这是分治策略效率的保证。
4. **稳定性**：合并时若遇到相等元素，优先取左侧子数组的元素，因此相等元素的相对顺序保持不变，归并排序是稳定的排序算法。
5. **非原地排序**：合并过程需要额外空间存储临时结果，因此归并排序不是原地排序算法。

## 三、过程/示例
以数组 `[38, 27, 43, 3, 9, 82, 10]` 为例，演示归并排序过程：

**第一步：递归分割**
- 初始数组：`[38, 27, 43, 3, 9, 82, 10]`
- 分割点：索引 `floor(0+6)/2 = 3`，分为左 `[38, 27, 43, 3]` 和右 `[9, 82, 10]`
- 继续分割左半：`[38, 27]` 和 `[43, 3]`
- 继续分割右半：`[9]` 和 `[82, 10]`
- 继续分割直到每个子数组只有一个元素：`[38]`, `[27]`, `[43]`, `[3]`, `[9]`, `[82]`, `[10]`

**第二步：自底向上合并**
- 合并 `[38]` 和 `[27]` → `[27, 38]`
- 合并 `[43]` 和 `[3]` → `[3, 43]`
- 合并 `[27, 38]` 和 `[3, 43]` → `[3, 27, 38, 43]`
- 合并 `[9]` 和 `[82]` → `[9, 82]`
- 合并 `[9, 82]` 和 `[10]` → `[9, 10, 82]`
- 最终合并 `[3, 27, 38, 43]` 和 `[9, 10, 82]` → `[3, 9, 10, 27, 38, 43, 82]`

合并过程详细演示（以最后一步为例）：
- 左指针指向 `3`，右指针指向 `9`，取 `3`
- 左指针指向 `27`，右指针指向 `9`，取 `9`
- 左指针指向 `27`，右指针指向 `10`，取 `10`
- 左指针指向 `27`，右指针指向 `82`，取 `27`
- 左指针指向 `38`，右指针指向 `82`，取 `38`
- 左指针指向 `43`，右指针指向 `82`，取 `43`
- 左数组耗尽，将右数组剩余 `82` 复制

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好情况：O(n log n)
  - 最坏情况：O(n log n)
  - 平均情况：O(n log n)
  无论输入数据如何，归并排序总是进行相同次数的分割和合并，因此时间复杂度稳定。
- **空间复杂度**：O(n)，需要与原始数组等长的辅助空间用于合并操作。递归调用栈占用 O(log n) 额外空间，但主导项为 O(n)。
- **稳定性**：稳定。合并时相等元素优先取自左子数组，不破坏原有顺序。
- **收敛性**：递归深度为 O(log n)，每次合并操作线性扫描，算法必然终止。
- **比较次数**：合并两个长度分别为 m 和 n 的有序数组时，最多需要 m+n-1 次比较，最少需要 min(m, n) 次比较。

## 五、常见误区
1. **混淆归并排序与快速排序**：两者都基于分治，但归并排序先分割后合并（“分-治-合”），快速排序先划分后递归（“治-分-合”）。归并排序稳定但需要额外空间，快速排序不稳定但可原地排序。
2. **误认为归并排序是原地排序**：合并操作必须使用额外数组，无法像插入排序那样在原始数组内完成。即使使用链表实现，也需要 O(log n) 的递归栈空间。
3. **忽视递归终止条件**：当子数组长度为 1 时，应直接返回而不执行合并操作。初学者常忘记处理 `p >= r` 的情况，导致无限递归或数组越界。
4. **合并时遗漏剩余元素**：合并过程中，当一个子数组遍历完毕后，必须将另一个子数组的剩余元素全部复制到结果中。遗漏此步骤会导致排序结果不完整。
5. **混淆自顶向下与自底向上实现**：自顶向下是递归实现，自底向上是迭代实现（先合并长度为 1 的子数组，再合并长度为 2、4……）。两者时间复杂度相同，但自底向上避免了递归调用栈的开销。
