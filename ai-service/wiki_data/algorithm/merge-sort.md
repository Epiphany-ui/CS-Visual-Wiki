# 归并排序

## 一、定义
归并排序（Merge Sort）是一种基于分治策略的排序算法，其核心思想是将待排序序列递归地划分为两个子序列，分别排序后再合并为一个有序序列。该算法由约翰·冯·诺依曼于1945年提出。

归并排序的核心数学描述如下：设待排序数组为A[1..n]，定义函数MergeSort(A, l, r)对子数组A[l..r]进行排序，其递归形式为：

```
MergeSort(A, l, r):
    if l < r:
        mid = ⌊(l+r)/2⌋
        MergeSort(A, l, mid)
        MergeSort(A, mid+1, r)
        Merge(A, l, mid, r)
```

其中Merge函数将两个已排序的子数组A[l..mid]和A[mid+1..r]合并为一个有序数组。

## 二、核心原理
归并排序的核心思想可归纳为以下要点：

1. **分治策略**：将原问题分解为规模更小的子问题，递归求解子问题，最后合并子问题的解得到原问题的解。归并排序严格遵循"分-治-合"三阶段。

2. **递归划分**：每次将当前数组从中间位置一分为二，直到每个子数组只包含一个元素（此时自然有序）。划分过程形成一棵完全二叉树，树高为⌈log₂n⌉。

3. **归并操作**：将两个已排序的子数组合并为一个有序数组。合并过程通过双指针技术实现，每次比较两个子数组当前元素，将较小者放入结果数组，时间复杂度为O(n)。

4. **稳定排序**：归并排序是稳定的排序算法，即相等元素的相对顺序在排序后保持不变。这一性质源于合并过程中，当左右子数组元素相等时，优先取左子数组元素。

5. **非原地排序**：归并排序需要额外的存储空间来存放合并结果，因此不是原地排序算法。标准实现的空间复杂度为O(n)。

## 三、过程/示例
以数组[38, 27, 43, 3, 9, 82, 10]为例，演示归并排序的完整执行过程：

**初始数组**：[38, 27, 43, 3, 9, 82, 10]

**第一步：递归划分**
- 划分点mid=3，分为左[38,27,43,3]和右[9,82,10]
- 左子数组继续划分：[38,27]和[43,3]
- 右子数组继续划分：[9]和[82,10]
- 继续划分直到单个元素：
  - [38],[27]；[43],[3]；[9]；[82],[10]

**第二步：归并合并**
- 合并[38]和[27] → [27,38]
- 合并[43]和[3] → [3,43]
- 合并[27,38]和[3,43] → [3,27,38,43]
- 合并[82]和[10] → [10,82]
- 合并[9]和[10,82] → [9,10,82]
- 合并[3,27,38,43]和[9,10,82] → [3,9,10,27,38,43,82]

**最终结果**：[3, 9, 10, 27, 38, 43, 82]

## 四、复杂度/性质分析
**时间复杂度**：
- 最好情况：O(n log n) — 无论输入数据如何，划分和合并的递归结构固定
- 最坏情况：O(n log n) — 同样不受输入数据影响
- 平均情况：O(n log n) — 归并排序的时间复杂度与输入数据的初始顺序无关

**空间复杂度**：
- 标准实现：O(n) — 需要与原始数组等长的辅助空间
- 原地归并排序：O(log n) — 通过复杂技巧可实现，但常数因子大，实际应用少

**稳定性**：稳定。合并操作中，当左右子数组元素相等时，优先取左子数组元素，保证了相等元素的相对顺序不变。

**收敛性**：归并排序的递归深度为⌈log₂n⌉，每次递归调用都会将问题规模减半，因此算法必然在有限步内终止。

**比较次数**：归并排序的比较次数在O(n log n)量级，具体为n⌈log₂n⌉ - 2^⌈log₂n⌉ + 1（最坏情况）。

## 五、常见误区
1. **混淆归并排序与快速排序**：两者都采用分治策略，但归并排序是"先分后合"，主要工作在合并阶段；快速排序是"先分后合"，主要工作在划分阶段。归并排序保证O(n log n)时间复杂度，快速排序最坏为O(n²)。

2. **误认为归并排序是原地排序**：标准归并排序需要O(n)额外空间，这是其与堆排序、快速排序的重要区别。虽然存在原地归并排序算法，但实现复杂且常数因子大。

3. **忽略递归终止条件**：初学者常忘记在递归函数中设置l≥r时的终止条件，导致无限递归。正确的终止条件是当子数组长度≤1时直接返回。

4. **合并操作中的索引错误**：合并时容易混淆左右子数组的起始和结束索引，特别是边界处理。常见错误包括数组越界、漏掉元素或重复处理元素。

5. **误认为归并排序适用于链表排序时空间复杂度仍为O(n)**：归并排序对链表排序时，空间复杂度可降为O(log n)（递归栈空间），因为链表不需要额外数组来存储合并结果，只需调整指针即可。
