# 二叉搜索树

## 一、定义
二叉搜索树（Binary Search Tree，BST）是一种特殊的二叉树结构，其中每个节点包含一个键值，并满足以下递归定义：对于树中的任意节点 \( x \)，其左子树中所有节点的键值均小于 \( x \) 的键值，其右子树中所有节点的键值均大于 \( x \) 的键值。数学上可表示为：

\[
\forall \text{ node } x, \quad \text{key}( \text{left}(x) ) < \text{key}(x) < \text{key}( \text{right}(x) )
\]

其中 \(\text{left}(x)\) 和 \(\text{right}(x)\) 分别表示节点 \( x \) 的左孩子和右孩子。该性质对树中每个节点递归成立。

## 二、核心原理
1. **有序性**：BST 的中序遍历（左-根-右）会得到一个严格递增的键值序列。这是 BST 最核心的性质，也是其名称中“搜索”二字的来源。
2. **递归结构**：BST 的每个子树本身也是一棵 BST。因此，所有操作（查找、插入、删除）都可以通过递归方式高效实现。
3. **查找机制**：从根节点开始，将目标值与当前节点键值比较。若相等则找到；若目标值较小，则进入左子树继续查找；若较大，则进入右子树。每次比较都将搜索范围缩小一半（理想情况下）。
4. **插入与删除**：插入时，按照查找规则找到空位并插入新节点。删除时需处理三种情况：删除叶子节点、删除只有一个子树的节点、删除有两个子树的节点（通常用前驱或后继节点替换）。
5. **动态维护**：BST 支持动态的插入和删除操作，无需预先知道数据规模，适合处理在线数据流。

## 三、过程/示例
以数组 \([5, 3, 7, 2, 4, 6, 8]\) 为例，演示构建 BST 的过程：

1. 插入 5：作为根节点。
2. 插入 3：3 < 5，进入左子树，左子树为空，插入为左孩子。
3. 插入 7：7 > 5，进入右子树，右子树为空，插入为右孩子。
4. 插入 2：2 < 5 → 左子树（节点 3），2 < 3 → 左子树为空，插入为节点 3 的左孩子。
5. 插入 4：4 < 5 → 左子树（节点 3），4 > 3 → 右子树为空，插入为节点 3 的右孩子。
6. 插入 6：6 > 5 → 右子树（节点 7），6 < 7 → 左子树为空，插入为节点 7 的左孩子。
7. 插入 8：8 > 5 → 右子树（节点 7），8 > 7 → 右子树为空，插入为节点 7 的右孩子。

最终树结构如下：
```
        5
       / \
      3   7
     / \ / \
    2  4 6  8
```
中序遍历结果为：2, 3, 4, 5, 6, 7, 8，严格递增。

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好情况（平衡树）：查找、插入、删除均为 \(O(\log n)\)，其中 \(n\) 为节点数。
  - 最坏情况（退化为链表，如插入有序序列）：所有操作退化为 \(O(n)\)。
  - 平均情况：随机插入序列下，树高期望为 \(O(\log n)\)，操作复杂度为 \(O(\log n)\)。
- **空间复杂度**：\(O(n)\)，用于存储所有节点。
- **稳定性**：BST 的插入和删除操作通常不保证稳定性（即相等键值的相对顺序可能改变）。若允许重复键值，需额外约定（如插入到左子树或右子树）。
- **收敛性**：对于查找操作，每次比较排除一半子树，搜索空间呈指数级收敛。最坏情况下收敛速度退化为线性。
- **与二分查找的关系**：BST 本质上是二分查找在动态数据结构上的实现，但二分查找要求有序数组，而 BST 支持动态插入和删除。

## 五、常见误区
1. **混淆 BST 与堆**：堆（Heap）只要求父节点大于（或小于）子节点，不要求左小右大，且中序遍历不保证有序。BST 则严格区分左右子树的大小关系。
2. **认为 BST 总是平衡的**：普通 BST 在插入有序数据时会退化为链表，导致性能严重下降。平衡二叉搜索树（如 AVL 树、红黑树）通过旋转机制保证树高为 \(O(\log n)\)。
3. **忽略重复键值处理**：标准 BST 定义要求键值唯一。若允许重复，需明确插入规则（如插入到左子树或右子树），否则会导致查找和删除的歧义。
4. **误认为中序遍历唯一确定树结构**：给定中序遍历序列，可以构造出多棵不同的 BST（例如，中序为 [1,2,3] 的 BST 有 5 种形态）。只有同时给定前序或后序遍历，才能唯一确定树结构。
5. **删除操作中的后继混淆**：删除有两个子树的节点时，常用右子树的最小节点（中序后继）或左子树的最大节点（中序前驱）替换。初学者容易误选为右子树的最大节点或左子树的最小节点，导致 BST 性质被破坏。
