# 链表

## 一、定义
链表（Linked List）是一种线性数据结构，其中元素通过指针（或引用）链接在一起，而非存储在连续的内存空间中。每个元素称为节点（Node），包含数据域和指向下一个节点的指针域。数学上，链表可定义为递归结构：一个链表要么为空（NULL），要么由一个节点和指向另一个链表的指针组成。伪代码表示为：

```
Node:
    data   // 数据域
    next   // 指针域，指向下一个节点

LinkedList:
    head   // 指向第一个节点的指针
```

## 二、核心原理
1. **非连续存储**：节点在内存中随机分布，通过指针建立逻辑顺序，克服数组需要连续内存的局限。
2. **动态大小**：链表大小可在运行时动态增长或收缩，无需预先分配固定容量。
3. **节点结构**：每个节点包含数据域和指针域。单链表中指针域仅指向后继节点；双链表中还包含指向前驱节点的指针；循环链表中尾节点指向头节点形成环。
4. **头指针**：通过头指针访问整个链表，头指针指向第一个节点。空链表头指针为NULL。
5. **插入/删除高效**：在已知位置插入或删除节点只需O(1)时间修改指针，无需移动其他元素。

## 三、过程/示例
以单链表为例，初始链表存储整数序列[3, 7, 2]，头指针head指向节点3。

**步骤1：遍历链表**  
从头节点开始，通过next指针依次访问：  
head → [3 | next] → [7 | next] → [2 | next] → NULL  
输出：3, 7, 2

**步骤2：在头部插入节点5**  
创建新节点[5 | ?]，将其next指向原头节点3，更新head指向新节点：  
head → [5 | next] → [3 | next] → [7 | next] → [2 | next] → NULL  
输出：5, 3, 7, 2

**步骤3：删除节点7**  
找到节点7的前驱节点3，将3的next指向节点2，释放节点7：  
head → [5 | next] → [3 | next] → [2 | next] → NULL  
输出：5, 3, 2

**步骤4：在节点3后插入节点9**  
创建新节点[9 | ?]，将其next指向节点2，将节点3的next指向节点9：  
head → [5 | next] → [3 | next] → [9 | next] → [2 | next] → NULL  
输出：5, 3, 9, 2

## 四、复杂度/性质分析
- **访问元素**：O(n)（必须从头遍历），最坏情况访问尾节点。
- **插入/删除**：已知位置时O(1)（仅修改指针）；未知位置时需先查找O(n)。
- **空间复杂度**：O(n)存储n个节点，每个节点额外存储指针（单链表1个，双链表2个）。
- **稳定性**：链表操作不改变相等元素的相对顺序，属于稳定数据结构。
- **内存开销**：比数组多存储指针，但避免了数组的预分配浪费和碎片化。
- **缓存性能**：由于非连续存储，链表对CPU缓存不友好，遍历速度通常慢于数组。

## 五、常见误区
1. **混淆数组与链表的索引**：数组支持O(1)随机访问，链表必须O(n)遍历。误以为链表也能通过索引快速访问是常见错误。
2. **指针丢失**：插入节点时若先修改当前节点的next指针，会导致后续节点无法访问。正确顺序是先让新节点指向后继，再修改前驱的next。
3. **空指针解引用**：遍历或操作链表时未检查节点是否为NULL，导致程序崩溃。例如在删除节点时未处理尾节点或空链表。
4. **内存泄漏**：删除节点后未释放内存（C/C++中需手动free/delete），或丢失头指针导致整个链表无法访问。
5. **循环链表死循环**：遍历循环链表时未设置终止条件（如计数器或标记），导致无限循环。需记录起始节点或使用快慢指针检测环。
