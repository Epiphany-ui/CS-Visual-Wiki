# Rabin-Karp算法

## 一、定义
Rabin-Karp算法是一种基于哈希技术的字符串匹配算法，由Michael O. Rabin和Richard M. Karp于1987年提出。该算法通过将模式串和文本子串映射为数值哈希值，利用哈希值的快速比较替代逐字符比较，从而实现高效的字符串匹配。其核心思想是：若两个字符串的哈希值不同，则它们必然不相等；若哈希值相同，则需进一步验证是否真正匹配（以处理哈希冲突）。

核心数学公式：对于长度为m的模式串P，定义其哈希值为：
\[
h(P) = (P[0] \times d^{m-1} + P[1] \times d^{m-2} + \cdots + P[m-1] \times d^0) \bmod q
\]
其中d为字符集大小（通常取256），q为一个较大的素数（用于减少哈希冲突）。

## 二、核心原理
1. **滚动哈希机制**：算法利用滑动窗口思想，在文本上从左到右移动长度为m的窗口。计算当前窗口的哈希值时，利用前一个窗口的哈希值通过递推公式快速得到，避免重复计算。递推公式为：
   \[
   h_{i+1} = (d \times (h_i - T[i] \times d^{m-1}) + T[i+m]) \bmod q
   \]
   其中T[i]为文本中第i个字符的数值编码。

2. **模运算与哈希冲突**：所有哈希计算均在模q下进行，q的选择直接影响算法性能。较大的素数q能降低哈希冲突概率，但会增加计算开销；较小的q则反之。当哈希值相等时，必须进行逐字符比较以确认是否真正匹配。

3. **预处理与在线匹配**：算法首先计算模式串的哈希值，然后对文本的每个长度为m的子串计算哈希值并比较。整个过程只需一次扫描文本，无需回溯，适合处理长文本和多个模式串的匹配。

4. **多模式扩展性**：Rabin-Karp算法天然支持同时匹配多个模式串——只需预先计算所有模式串的哈希值，并在扫描文本时与这些哈希值集合进行比对。

## 三、过程/示例
假设文本T = "235902314152673"，模式串P = "31415"，字符集为十进制数字（d=10），取q=13。

**步骤1**：计算模式串的哈希值
P = "31415" → 数值序列[3,1,4,1,5]
h(P) = (3×10^4 + 1×10^3 + 4×10^2 + 1×10^1 + 5×10^0) mod 13
= (30000 + 1000 + 400 + 10 + 5) mod 13 = 31415 mod 13 = 7

**步骤2**：计算文本第一个窗口"23590"的哈希值
h("23590") = (2×10^4 + 3×10^3 + 5×10^2 + 9×10^1 + 0) mod 13
= (20000 + 3000 + 500 + 90 + 0) mod 13 = 23590 mod 13 = 8
8 ≠ 7，不匹配。

**步骤3**：滚动到下一个窗口"35902"
利用递推公式：
h_new = (10 × (8 - 2×10^4 mod 13) + 2) mod 13
先计算10^4 mod 13 = 10000 mod 13 = 3
h_new = (10 × (8 - 2×3) + 2) mod 13 = (10 × 2 + 2) mod 13 = 22 mod 13 = 9
9 ≠ 7，继续。

**步骤4**：继续滚动直到窗口"31415"
当窗口滑动到位置5时，窗口内容为"31415"：
h("31415") = 7（与模式串哈希值相同）
此时进行逐字符比较：3=3,1=1,4=4,1=1,5=5 → 完全匹配，报告位置5。

**步骤5**：继续扫描剩余文本，直到结束。

## 四、复杂度/性质分析
1. **时间复杂度**：
   - 平均情况：O(n+m)，其中n为文本长度，m为模式串长度。预处理O(m)，扫描文本O(n)，每次哈希比较O(1)。
   - 最坏情况：O(nm)，当哈希冲突频繁发生时（如所有子串哈希值均与模式串相同），每次都需要O(m)的逐字符验证。
   - 最佳情况：O(n+m)，无哈希冲突时。

2. **空间复杂度**：O(1)，仅需存储模式串哈希值和少量辅助变量。

3. **关键性质**：
   - 非稳定算法：哈希冲突可能导致额外比较，但匹配结果正确。
   - 在线算法：可实时处理流式数据，无需预知文本长度。
   - 概率性质：通过选择大素数q，可将误报率降至极低（约1/q），但无法完全消除。

4. **与KMP算法对比**：Rabin-Karp在平均情况下与KMP（O(n+m)）相当，但最坏情况劣于KMP。优势在于多模式匹配场景，Rabin-Karp可同时处理k个模式串，时间复杂度为O(n+km)，而KMP需要分别匹配k次。

## 五、常见误区
1. **误认为哈希值相等即匹配**：哈希冲突是必然存在的，必须进行逐字符验证。初学者常忽略此步骤，导致假阳性错误。

2. **混淆滚动哈希与普通哈希**：滚动哈希利用前一个窗口的结果递推计算，而非每次重新计算整个子串。错误地重新计算会导致时间复杂度退化为O(nm)。

3. **忽视模运算的负数处理**：在递推公式中，减法操作可能产生负数，需进行模运算调整。例如在C语言中，需写成`(d * (h - T[i] * h_mul) + T[i+m]) % q`，并确保结果非负。

4. **错误选择模数q**：q若为2的幂（如2^32），会导致哈希值分布不均匀，增加冲突概率。应选择大素数，如10^9+7或2^61-1。

5. **混淆与朴素算法的关系**：Rabin-Karp并非完全替代朴素算法，而是通过哈希加速。当模式串长度m很小时（如m=1），滚动哈希的优势不明显，朴素算法反而更简单高效。
