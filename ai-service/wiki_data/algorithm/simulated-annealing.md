# 模拟退火算法

## 一、定义
模拟退火算法（Simulated Annealing, SA）是一种基于蒙特卡洛迭代求解策略的随机优化算法。其核心思想源于固体退火过程：将固体加热至高温后缓慢冷却，使内部粒子逐渐趋于有序状态，最终达到能量最低的基态。算法通过引入随机扰动和概率接受准则（Metropolis准则），在搜索过程中以一定概率接受劣质解，从而避免陷入局部最优。

**核心数学公式**（Metropolis准则）：
\[
P(\Delta E) = \begin{cases}
1, & \Delta E < 0 \\
e^{-\Delta E / T}, & \Delta E \geq 0
\end{cases}
\]
其中，\(\Delta E = E(x_{\text{new}}) - E(x_{\text{current}})\) 为新旧解的能量差（目标函数差值），\(T\) 为当前温度。

**伪代码框架**：
```
初始化温度 T_0，初始解 x_0
while (T > T_min):
    for i in range(L):  # L为每个温度下的迭代次数
        生成邻域解 x_new
        计算 ΔE = f(x_new) - f(x_current)
        if ΔE < 0 or random() < exp(-ΔE/T):
            x_current = x_new
    T = T * α  # 降温，α为冷却系数
输出最优解
```

## 二、核心原理
1. **退火过程的数学映射**：将优化问题的目标函数视为固体能量函数，控制参数（温度）作为搜索策略的调节器。高温时算法具有强随机性，低温时趋于确定性搜索。

2. **Metropolis接受准则**：这是算法跳出局部最优的关键机制。当新解更优时无条件接受；当新解更差时，以概率 \(e^{-\Delta E/T}\) 接受。该概率随温度降低而减小，使算法在高温阶段充分探索解空间，低温阶段精细开发。

3. **温度调度策略**：降温速率 \(\alpha\)（通常取0.8-0.99）控制收敛速度。理论上需满足对数降温（\(T_k = T_0 / \ln(k+1)\)）才能保证全局收敛，但实际应用中多采用指数降温（\(T_{k+1} = \alpha T_k\)）以平衡效率。

4. **马尔可夫链特性**：在每个温度下，算法执行固定步数的随机游走，形成一条有限长度的马尔可夫链。链长L需足够长以保证达到热平衡状态。

## 三、过程/示例
**问题**：求函数 \(f(x) = x^2\) 在区间 \([-10, 10]\) 上的最小值。

**参数设置**：初始温度 \(T_0 = 100\)，降温系数 \(\alpha = 0.9\)，每个温度迭代次数 \(L = 3\)，终止温度 \(T_{\min} = 0.1\)。

**执行步骤**：
1. **初始化**：随机选择初始解 \(x_0 = 8\)，\(f(8) = 64\)，当前最优解 \(x_{\text{best}} = 8\)。
2. **温度 \(T=100\)**：
   - 迭代1：生成邻域解 \(x_{\text{new}} = 6\)（随机扰动），\(\Delta E = 36 - 64 = -28 < 0\)，接受新解。当前解 \(x=6\)。
   - 迭代2：\(x_{\text{new}} = 9\)，\(\Delta E = 81 - 36 = 45 > 0\)，计算接受概率 \(P = e^{-45/100} \approx 0.637\)，随机数 \(r=0.2 < 0.637\)，接受。当前解 \(x=9\)。
   - 迭代3：\(x_{\text{new}} = 7\)，\(\Delta E = 49 - 81 = -32 < 0\)，接受。当前解 \(x=7\)。
3. **降温**：\(T = 100 \times 0.9 = 90\)。
4. **温度 \(T=90\)**：
   - 迭代1：\(x_{\text{new}} = 5\)，\(\Delta E = 25 - 49 = -24 < 0\)，接受。当前解 \(x=5\)。
   - 迭代2：\(x_{\text{new}} = 8\)，\(\Delta E = 64 - 25 = 39 > 0\)，\(P = e^{-39/90} \approx 0.648\)，\(r=0.5 < 0.648\)，接受。当前解 \(x=8\)。
   - 迭代3：\(x_{\text{new}} = 4\)，\(\Delta E = 16 - 64 = -48 < 0\)，接受。当前解 \(x=4\)。
5. **持续降温**：重复上述过程，温度逐渐降低，接受劣质解的概率越来越小。当 \(T=0.1\) 时，算法终止，输出最优解 \(x_{\text{best}} \approx 0\)（实际可能为接近0的小值）。

## 四、复杂度/性质分析
- **时间复杂度**：\(O(L \cdot K)\)，其中 \(L\) 为每个温度的迭代次数，\(K\) 为温度下降次数（\(K \approx \log_{\alpha}(T_{\min}/T_0)\)）。最好情况（初始解接近最优）和最坏情况（需充分探索）均为同一量级，但常数因子差异显著。
- **空间复杂度**：\(O(1)\)，仅需存储当前解、最优解和温度参数，与问题规模无关。
- **收敛性**：在满足对数降温策略和马尔可夫链无限长的理论条件下，算法以概率1收敛到全局最优。实际应用中，由于采用有限链长和指数降温，只能保证以高概率接近全局最优。
- **稳定性**：属于随机算法，单次运行结果具有随机性。通常需要多次独立运行取最优结果，或采用并行退火策略提高稳定性。
- **参数敏感性**：对初始温度 \(T_0\)、降温系数 \(\alpha\) 和链长 \(L\) 敏感。\(T_0\) 过小易陷入局部最优，过大则浪费计算资源；\(\alpha\) 过小导致降温过快，过大则收敛缓慢。

## 五、常见误区
1. **混淆退火与贪心算法**：贪心算法只接受更优解，而模拟退火以概率接受劣质解，这是两者本质区别。初学者常误以为模拟退火是“带随机扰动的贪心”。

2. **忽视温度调度的重要性**：认为只要使用Metropolis准则就能保证全局最优。实际上，不合理的降温策略（如线性降温）会导致算法退化为随机搜索或过早收敛。

3. **混淆模拟退火与遗传算法**：两者都是启发式随机优化算法，但模拟退火基于单点搜索和温度控制，遗传算法基于种群进化和交叉变异。模拟退火更适合连续优化问题，遗传算法更适合离散组合优化。

4. **错误理解接受概率公式**：认为接受概率仅与温度有关，忽略了能量差 \(\Delta E\) 的影响。实际上，\(e^{-\Delta E/T}\) 中 \(\Delta E\) 和 \(T\) 共同决定接受概率，且当 \(\Delta E\) 很大时，即使温度较高，接受概率也可能很小。

5. **忽略邻域生成策略**：认为只要随机扰动即可，未考虑邻域结构对算法性能的影响。对于不同问题，邻域生成方式（如连续问题的步长、组合问题的交换操作）直接影响搜索效率和解的质量。
