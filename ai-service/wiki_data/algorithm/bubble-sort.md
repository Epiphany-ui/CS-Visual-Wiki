# 冒泡排序

## 一、定义

冒泡排序（Bubble Sort）是一种基于比较的简单排序算法。它通过重复遍历待排序序列，依次比较相邻两个元素，若顺序错误则交换它们，直到整个序列有序。其名称源于较小的元素会像气泡一样逐渐“浮”到序列的顶端。

**核心伪代码**（对数组 `A[0..n-1]` 进行升序排序）：

```
for i = 0 to n-2 do
    for j = 0 to n-2-i do
        if A[j] > A[j+1] then
            swap(A[j], A[j+1])
        end if
    end for
end for
```

**数学描述**：设待排序序列为 `a₁, a₂, ..., aₙ`，冒泡排序通过一系列相邻交换操作，使得最终序列满足 `a₁ ≤ a₂ ≤ ... ≤ aₙ`。每次内层循环将当前未排序部分的最大值“冒泡”到其正确位置。

## 二、核心原理

1. **相邻比较与交换**：算法仅对相邻元素进行比较和交换。这一特性保证了每次交换都直接减少序列中的逆序对数量，且交换操作是局部且可逆的。

2. **多趟遍历**：每一趟遍历（外层循环一次）将当前未排序部分的最大元素移动到其最终位置。经过 `n-1` 趟后，所有元素归位。

3. **尾部有序性**：第 `k` 趟结束后，序列的最后 `k` 个元素已经处于正确位置，后续遍历无需再处理它们。这是内层循环上界递减的原因。

4. **提前终止优化**：若某一趟遍历中未发生任何交换，说明序列已经有序，算法可以提前结束。这一优化使冒泡排序在最好情况下具有线性时间复杂度。

5. **稳定性**：当相邻元素相等时，冒泡排序不交换它们，因此相等元素的相对顺序保持不变。这是冒泡排序作为稳定排序算法的关键性质。

## 三、过程/示例

以数组 `[5, 1, 4, 2, 8]` 为例，演示升序冒泡排序的完整过程。

**第一趟**（i=0，比较范围 j=0 到 3）：
- 比较 5 和 1：5 > 1，交换 → `[1, 5, 4, 2, 8]`
- 比较 5 和 4：5 > 4，交换 → `[1, 4, 5, 2, 8]`
- 比较 5 和 2：5 > 2，交换 → `[1, 4, 2, 5, 8]`
- 比较 5 和 8：5 < 8，不交换 → `[1, 4, 2, 5, 8]`
第一趟结束，最大值 8 已到达正确位置。

**第二趟**（i=1，比较范围 j=0 到 2）：
- 比较 1 和 4：1 < 4，不交换 → `[1, 4, 2, 5, 8]`
- 比较 4 和 2：4 > 2，交换 → `[1, 2, 4, 5, 8]`
- 比较 4 和 5：4 < 5，不交换 → `[1, 2, 4, 5, 8]`
第二趟结束，次大值 5 已归位。

**第三趟**（i=2，比较范围 j=0 到 1）：
- 比较 1 和 2：1 < 2，不交换 → `[1, 2, 4, 5, 8]`
- 比较 2 和 4：2 < 4，不交换 → `[1, 2, 4, 5, 8]`
此趟无交换，序列已有序，算法提前终止。

最终排序结果：`[1, 2, 4, 5, 8]`。

## 四、复杂度/性质分析

**时间复杂度**：
- **最坏情况**（逆序序列）：需要进行 `n-1` 趟，每趟比较次数递减，总比较次数为 `n(n-1)/2`，交换次数相同，时间复杂度为 **O(n²)**。
- **最好情况**（已有序且使用提前终止优化）：仅需一趟遍历，比较 `n-1` 次，无交换，时间复杂度为 **O(n)**。
- **平均情况**：平均比较次数和交换次数均为 `n(n-1)/4` 量级，时间复杂度为 **O(n²)**。

**空间复杂度**：仅需常数个额外变量（如用于交换的临时变量），空间复杂度为 **O(1)**，属于原地排序算法。

**稳定性**：稳定。当 `A[j] == A[j+1]` 时，条件 `A[j] > A[j+1]` 不成立，不会发生交换，因此相等元素的相对顺序得以保持。

**收敛性**：冒泡排序保证在有限步内收敛到有序序列。每一趟至少将一个元素放置到其最终位置，因此最多 `n-1` 趟后必然完成排序。

## 五、常见误区

1. **误认为冒泡排序与选择排序相同**：两者都通过多趟遍历将元素归位，但选择排序每趟寻找最小值（或最大值）并与当前位置交换，而冒泡排序通过相邻交换逐步移动元素。选择排序是不稳定的，冒泡排序是稳定的。

2. **忽略提前终止优化的作用**：未优化的冒泡排序总是执行 `n-1` 趟，即使序列已经有序。加入标志位后，最好情况时间复杂度可从 O(n²) 降至 O(n)，这是实现中必须注意的细节。

3. **混淆“冒泡”方向**：升序排序时，较大的元素向后“沉底”，较小的元素向前“浮起”。初学者常误以为冒泡排序是将大元素向前移动，实际上标准实现是将大元素向后移动。

4. **错误理解内层循环边界**：内层循环 `j` 的上界应为 `n-2-i`，而非 `n-2`。若忽略 `-i`，算法会重复比较已排序的尾部元素，虽然不影响正确性，但增加了不必要的比较次数。

5. **认为冒泡排序适合大规模数据**：由于 O(n²) 的平均时间复杂度，冒泡排序仅适用于小规模数据（如 n < 1000）或基本有序的数据。对于大规模随机数据，应选择快速排序、归并排序等更高效的算法。
