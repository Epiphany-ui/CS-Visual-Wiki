# 冒泡排序

## 一、定义

冒泡排序（Bubble Sort）是一种基于比较的简单排序算法。它通过重复遍历待排序序列，依次比较相邻两个元素，若顺序错误则交换它们，直到没有需要交换的元素为止。其名称来源于值较大的元素会像气泡一样逐渐“浮”到序列的末端。

**核心伪代码**（升序排序）：
```
BUBBLE-SORT(A, n)
    for i = 1 to n-1
        for j = 1 to n-i
            if A[j] > A[j+1]
                swap(A[j], A[j+1])
```

其中 `A[1..n]` 为待排序数组，`n` 为数组长度。内层循环每完成一轮，当前未排序部分的最大值就被放置到正确位置。

## 二、核心原理

1. **相邻比较与交换**：算法每次只比较相邻的两个元素。若前一个大于后一个（升序），则交换它们。这种局部操作保证了每轮遍历后，未排序部分的最大值被“冒泡”到该部分的末尾。

2. **多轮遍历**：每一轮遍历（外循环）将当前未排序区间内的最大值移动到正确位置。经过 `n-1` 轮后，所有元素均有序。

3. **提前终止优化**：若在某轮遍历中未发生任何交换，说明序列已经有序，算法可以提前结束。这是冒泡排序的一个重要优化特性。

4. **原地排序**：冒泡排序仅使用常数级别的额外空间（用于交换的临时变量），属于原地排序算法。

5. **稳定性**：当相邻元素相等时，算法不交换它们，因此相等元素的相对顺序保持不变。冒泡排序是稳定的排序算法。

## 三、过程/示例

以数组 `[5, 3, 8, 4, 2]` 为例，演示升序冒泡排序的完整过程：

**初始序列**：`[5, 3, 8, 4, 2]`

**第1轮**（i=1，需比较到第4个元素）：
- 比较5和3：5>3，交换 → `[3, 5, 8, 4, 2]`
- 比较5和8：5<8，不交换 → `[3, 5, 8, 4, 2]`
- 比较8和4：8>4，交换 → `[3, 5, 4, 8, 2]`
- 比较8和2：8>2，交换 → `[3, 5, 4, 2, 8]`
第1轮结束，最大值8已就位。

**第2轮**（i=2，需比较到第3个元素）：
- 比较3和5：3<5，不交换 → `[3, 5, 4, 2, 8]`
- 比较5和4：5>4，交换 → `[3, 4, 5, 2, 8]`
- 比较5和2：5>2，交换 → `[3, 4, 2, 5, 8]`
第2轮结束，次大值5已就位。

**第3轮**（i=3，需比较到第2个元素）：
- 比较3和4：3<4，不交换 → `[3, 4, 2, 5, 8]`
- 比较4和2：4>2，交换 → `[3, 2, 4, 5, 8]`
第3轮结束，值4已就位。

**第4轮**（i=4，需比较到第1个元素）：
- 比较3和2：3>2，交换 → `[2, 3, 4, 5, 8]`
第4轮结束，整个序列有序。

**最终结果**：`[2, 3, 4, 5, 8]`

## 四、复杂度/性质分析

1. **时间复杂度**：
   - **最坏情况**：数组完全逆序，每轮都需要进行最大次数的比较和交换。比较次数为 `(n-1)+(n-2)+...+1 = n(n-1)/2`，交换次数相同，故最坏时间复杂度为 **O(n²)**。
   - **最好情况**：数组已经有序，经过第一轮遍历后未发生任何交换，算法提前终止。此时仅进行 `n-1` 次比较，0次交换，时间复杂度为 **O(n)**。
   - **平均情况**：随机排列的数组，平均比较和交换次数约为 `n(n-1)/4`，时间复杂度为 **O(n²)**。

2. **空间复杂度**：仅需一个临时变量用于交换，空间复杂度为 **O(1)**。

3. **稳定性**：相等元素不交换，因此是 **稳定排序**。

4. **收敛性**：每轮遍历至少将一个元素放置到最终位置，因此最多经过 `n-1` 轮必然收敛到有序序列。

5. **适应性**：通过提前终止优化，冒泡排序在输入接近有序时表现良好，属于自适应排序算法。

## 五、常见误区

1. **误以为冒泡排序与选择排序相同**：两者都通过多轮遍历将元素放到正确位置，但选择排序每轮只找最小值（或最大值）并交换一次，而冒泡排序每轮进行多次相邻交换。选择排序是不稳定的，冒泡排序是稳定的。

2. **忽略提前终止优化**：许多初学者实现冒泡排序时未添加“是否发生交换”的标记，导致即使输入有序，算法仍执行完整的 `n-1` 轮遍历，时间复杂度退化为 O(n²)。

3. **混淆内层循环边界**：内层循环的终止条件应为 `n-i`（或 `n-i-1`，取决于索引起始值），而非 `n-1`。错误边界会导致已排序部分被重复比较，甚至数组越界。

4. **认为冒泡排序效率与插入排序相当**：虽然两者平均时间复杂度均为 O(n²)，但插入排序在数据接近有序时表现更好（常数因子更小），且插入排序的交换操作（实际为移动）通常比冒泡排序的交换更高效。

5. **误以为冒泡排序可以处理链表**：冒泡排序依赖相邻元素的随机访问和交换，在单向链表上实现时效率极低（需要反复遍历查找前驱节点），实际中链表排序更常使用归并排序。
