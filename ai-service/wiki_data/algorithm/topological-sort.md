# 拓扑排序

## 一、定义
拓扑排序（Topological Sorting）是对有向无环图（DAG, Directed Acyclic Graph）的所有顶点进行线性排序的一种算法，使得对于图中任意一条有向边(u, v)，顶点u在排序结果中都出现在顶点v之前。拓扑排序的核心数学表述为：给定一个DAG G = (V, E)，寻找一个排列π: V → {1, 2, …, |V|}，使得对任意(u, v) ∈ E，有π(u) < π(v)。拓扑排序的伪代码（基于Kahn算法）如下：

```
Kahn(G):
    L ← 空列表（存放排序结果）
    S ← 所有入度为0的顶点集合
    while S 非空:
        从S中移除一个顶点u
        将u添加到L末尾
        for 每个从u出发的边(u, v):
            将v的入度减1
            if v的入度变为0:
                将v加入S
    if L包含所有顶点:
        return L
    else:
        return "图中存在环，无法拓扑排序"
```

## 二、核心原理
1. **入度驱动机制**：拓扑排序的核心思想是利用顶点的入度（指向该顶点的边数）作为排序依据。入度为0的顶点表示没有任何前置依赖，可以优先输出。每输出一个顶点，就“移除”该顶点及其出边，从而更新其他顶点的入度状态。

2. **依赖关系建模**：拓扑排序本质上是对偏序关系（Partial Order）的线性扩展（Linear Extension）。在DAG中，边(u, v)表示u必须先于v执行，拓扑排序将这种偏序关系转化为全序关系。

3. **环检测能力**：拓扑排序天然具备检测有向环的能力。如果算法结束时输出的顶点数少于总顶点数，则说明图中存在环，无法进行拓扑排序。这一性质使得拓扑排序成为DAG验证的标准工具。

4. **不唯一性**：对于同一个DAG，可能存在多个合法的拓扑排序结果。当存在多个入度为0的顶点时，选择不同的顶点会导致不同的排序序列，但所有合法序列都满足偏序约束。

5. **逆拓扑排序**：通过反向处理（使用出度为0的顶点）或对拓扑排序结果取逆序，可以得到逆拓扑排序，用于解决“依赖后置”问题。

## 三、过程/示例
考虑一个课程依赖关系图，顶点表示课程，边表示前置课程要求：
- 顶点集：{A, B, C, D, E}
- 边集：{(A, C), (B, C), (C, D), (C, E), (D, E)}
- 初始入度：A:0, B:0, C:2, D:1, E:2

**执行步骤**：
1. 初始化：S = {A, B}（入度为0的顶点）
2. 从S中取出A，输出A。更新：C入度减1变为1，D、E不变。S = {B}
3. 从S中取出B，输出B。更新：C入度减1变为0，将C加入S。S = {C}
4. 从S中取出C，输出C。更新：D入度减1变为0，将D加入S；E入度减1变为1。S = {D}
5. 从S中取出D，输出D。更新：E入度减1变为0，将E加入S。S = {E}
6. 从S中取出E，输出E。S = ∅
7. 输出序列：A → B → C → D → E（验证：所有边均满足前置在后）

## 四、复杂度/性质分析
1. **时间复杂度**：基于Kahn算法或DFS的拓扑排序时间复杂度均为O(|V| + |E|)，其中|V|为顶点数，|E|为边数。最好、最坏和平均情况均为O(|V| + |E|)，因为算法必须遍历所有顶点和边一次。使用邻接表存储图时，每个顶点和每条边恰好被处理一次。

2. **空间复杂度**：O(|V|)，主要用于存储入度数组、队列/栈以及结果列表。邻接表本身占用O(|V| + |E|)空间，但通常不计入算法额外空间。

3. **稳定性**：拓扑排序本身不保证稳定性（即相等优先级元素的相对顺序）。当多个入度为0的顶点同时存在时，不同的实现（如使用队列或栈）会产生不同的顺序，但所有顺序都是合法的。

4. **收敛性**：拓扑排序是确定性的收敛算法，只要输入是DAG，算法必然在有限步内终止并输出完整排序。若输入包含环，算法同样会终止，但会报告错误。

5. **与DFS的关系**：基于DFS的拓扑排序通过记录顶点的完成时间（finish time），按完成时间降序输出即可得到拓扑排序。该方法的正确性基于DFS对DAG的深度优先遍历性质。

## 五、常见误区
1. **混淆拓扑排序与排序算法**：拓扑排序不是对数值大小进行排序，而是对具有依赖关系的元素进行线性化。它不要求元素具有可比性，只要求满足偏序约束。

2. **认为拓扑排序结果唯一**：初学者常误以为拓扑排序只有一种正确结果。实际上，只要满足所有边的方向约束，任何序列都是合法的。例如上例中B → A → C → D → E也是合法排序。

3. **忽略环检测的重要性**：拓扑排序只能应用于DAG。若输入图包含环，算法会失败。许多学习者忘记在实现中加入环检测逻辑，导致在非DAG上得到错误结果。

4. **混淆入度与出度的角色**：Kahn算法使用入度为0的顶点作为起点，而逆拓扑排序使用出度为0的顶点。初学者容易混淆这两种变体，或在实现中错误地使用出度代替入度。

5. **误认为拓扑排序适用于无向图**：拓扑排序严格定义在有向图上，无向图不存在“方向”概念，无法定义“前置”关系。对无向图应用拓扑排序是概念性错误。
