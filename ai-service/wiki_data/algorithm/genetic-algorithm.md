# 遗传算法

## 一、定义
遗传算法（Genetic Algorithm, GA）是一种模拟自然进化过程的随机搜索与优化算法，由John Holland于1975年提出。它通过模拟生物进化中的选择、交叉和变异等遗传操作，在解空间中迭代搜索最优解。

**核心数学框架**：设种群为$P(t) = \{x_1, x_2, ..., x_N\}$，其中$x_i$为个体（染色体），$f(x_i)$为适应度函数。遗传算法的基本迭代过程可表示为：
```
t = 0
初始化种群 P(0)
while (终止条件不满足):
    评估 P(t) 中每个个体的适应度 f(x_i)
    选择操作: P'(t) = Selection(P(t), f)
    交叉操作: P''(t) = Crossover(P'(t), pc)
    变异操作: P(t+1) = Mutation(P''(t), pm)
    t = t + 1
return 最优个体
```
其中$pc$为交叉概率，$pm$为变异概率。

## 二、核心原理
1. **编码与解码**：将问题的解空间映射到遗传算法的搜索空间。常用二进制编码（如将实数$[0,1]$映射为8位二进制串）、实数编码或排列编码。编码方式直接影响算法的搜索效率和解的质量。

2. **适应度函数**：衡量个体优劣的量化指标，是自然选择中的“生存能力”。适应度函数$f(x)$必须非负且最大化，对于最小化问题需通过变换$f(x) = C - g(x)$或$f(x) = 1/(1+g(x))$转换。

3. **选择机制**：模拟“适者生存”，常用轮盘赌选择（个体被选概率与适应度成正比）、锦标赛选择（随机抽取k个个体取最优）或排名选择。选择压力过大会导致早熟收敛，过小则搜索效率低。

4. **遗传操作**：交叉（Crossover）模拟基因重组，以概率$pc$交换父代部分基因；变异（Mutation）以概率$pm$随机改变基因位，维持种群多样性。二者平衡了算法的探索（Exploration）与开发（Exploitation）能力。

5. **精英保留策略**：将每代最优个体直接复制到下一代，保证算法单调收敛。这是防止最优解丢失的关键机制。

## 三、过程/示例
**问题**：求函数$f(x) = x^2$在$x \in [0, 31]$的最大值，使用二进制编码（5位二进制表示整数）。

**初始种群**：随机生成4个个体（染色体）：01101 (13), 11000 (24), 01000 (8), 10011 (19)

**第1代迭代**：
1. **适应度评估**：$f(13)=169, f(24)=576, f(8)=64, f(19)=361$，总和=1170
2. **选择**：轮盘赌概率分别为14.4%, 49.2%, 5.5%, 30.9%。假设选中个体：11000, 10011, 11000, 01101
3. **交叉**：随机配对(11000, 10011)和(11000, 01101)，交叉点位置3和2，产生子代：11011(27), 10000(16), 11101(29), 01000(8)
4. **变异**：以概率0.01变异，假设无变异发生

**新种群**：11011(27), 10000(16), 11101(29), 01000(8)，适应度分别为729, 256, 841, 64，平均适应度从292.5提升至472.5。

**第2代迭代**：选择后可能得到11101(29)和11011(27)等，继续进化直至收敛到最优解11111(31)，$f(31)=961$。

## 四、复杂度/性质分析
- **时间复杂度**：每代$O(N \cdot L + N \cdot f_{eval})$，其中$N$为种群规模，$L$为染色体长度，$f_{eval}$为适应度计算复杂度。总复杂度为$O(G \cdot N \cdot (L + f_{eval}))$，$G$为迭代代数。最好情况可快速收敛，最坏情况可能陷入随机搜索。
- **空间复杂度**：$O(N \cdot L)$，需存储当前种群和子代种群。
- **收敛性**：标准遗传算法不保证全局收敛，但带精英保留策略的GA以概率1收敛到全局最优（需无限时间）。实际应用中常采用早停策略。
- **稳定性**：受随机性影响，多次运行结果可能不同。通常需多次运行取最优或平均值。
- **参数敏感性**：对交叉概率$pc$（通常0.6-0.9）、变异概率$pm$（通常0.001-0.1）和种群规模$N$（通常20-100）敏感，需问题特定调参。

## 五、常见误区
1. **混淆遗传算法与进化策略**：遗传算法强调交叉操作，变异为辅助；进化策略（Evolution Strategies）主要依赖变异，交叉较少使用。二者编码方式也不同（GA常用二进制，ES常用实数向量）。

2. **认为遗传算法总能找到全局最优**：标准GA可能早熟收敛于局部最优。需结合精英保留、自适应参数或多种群策略提升全局搜索能力。

3. **忽视编码的重要性**：编码直接影响搜索空间结构和操作效率。例如，二进制编码中相邻整数的汉明距离可能很大（如01111→10000），导致变异效率低，此时Gray编码更优。

4. **盲目使用轮盘赌选择**：当适应度差异极大时，轮盘赌会导致超级个体主导种群，丧失多样性。此时应使用锦标赛选择或适应度缩放（如线性缩放、Sigma截断）。

5. **混淆适应度与目标函数**：适应度必须非负且最大化，而目标函数可能为负或需最小化。需通过变换将目标函数映射为适应度，且变换方式会影响选择压力。
