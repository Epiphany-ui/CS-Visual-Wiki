# 广度优先搜索

## 一、定义
广度优先搜索（Breadth-First Search, BFS）是一种用于遍历或搜索树或图数据结构的算法。它从选定的起始节点（根节点）出发，首先访问所有距离该节点为1的邻居节点，然后依次访问距离为2、3……的节点，直至遍历完所有可达节点。BFS的核心特征是“按层”进行探索，即先访问完当前层的所有节点后，再进入下一层。

在数学上，BFS可以形式化地描述为：给定图G=(V,E)和起始节点s∈V，BFS为每个节点v∈V计算其到s的最短路径距离δ(s,v)，并生成一棵以s为根的广度优先树。其核心伪代码如下：
```
BFS(G, s):
    for each vertex u ∈ G.V - {s}:
        u.color = WHITE
        u.d = ∞
        u.π = NIL
    s.color = GRAY
    s.d = 0
    s.π = NIL
    Q = ∅
    ENQUEUE(Q, s)
    while Q ≠ ∅:
        u = DEQUEUE(Q)
        for each v ∈ G.Adj[u]:
            if v.color == WHITE:
                v.color = GRAY
                v.d = u.d + 1
                v.π = u
                ENQUEUE(Q, v)
        u.color = BLACK
```
其中，color表示节点状态（WHITE未访问，GRAY已发现但邻接未完全探索，BLACK已完全探索），d记录到起点的距离，π记录前驱节点，Q为先进先出队列。

## 二、核心原理
1. **分层探索机制**：BFS的核心思想是“先近后远”。算法通过队列维护待访问节点，确保同一深度的节点被连续处理。当处理完第k层的所有节点后，队列中仅包含第k+1层的节点，从而保证按层推进。

2. **队列的先进先出特性**：队列是BFS的基石。新发现的节点被追加到队尾，而当前节点从队首取出。这种FIFO（First-In-First-Out）顺序天然实现了“先发现的节点先被探索其邻居”，等价于按距离递增的顺序访问节点。

3. **最短路径性质**：在无权图（所有边权为1）中，BFS首次访问某个节点时，所经过的路径就是从起点到该节点的最短路径。这一性质源于BFS按层扩展的特性——第k层的节点必然在访问完第k-1层所有节点后才被首次发现。

4. **广度优先树的构建**：算法执行过程中，每个节点（除起点外）被赋予一个前驱节点π，这些前驱关系构成一棵以s为根的树，称为广度优先树。该树包含了从s到所有可达节点的最短路径。

5. **节点着色机制**：通过WHITE、GRAY、BLACK三种颜色状态，BFS有效避免了重复访问和无限循环。WHITE表示未发现，GRAY表示已入队但邻居未完全探索，BLACK表示已完全处理。

## 三、过程/示例
考虑一个无向图，顶点集V={A,B,C,D,E}，边集E={(A,B),(A,C),(B,D),(C,D),(D,E)}。以A为起点执行BFS。

**初始化**：所有节点color=WHITE，d=∞，π=NIL。A.color=GRAY，A.d=0，A.π=NIL。队列Q=[A]。

**第1步**：出队A。遍历A的邻居B和C，均为WHITE。设置B.color=GRAY，B.d=1，B.π=A；C.color=GRAY，C.d=1，C.π=A。B和C入队，Q=[B,C]。A.color=BLACK。

**第2步**：出队B。遍历B的邻居A（已BLACK）和D（WHITE）。设置D.color=GRAY，D.d=2，D.π=B。D入队，Q=[C,D]。B.color=BLACK。

**第3步**：出队C。遍历C的邻居A（BLACK）和D（GRAY，已在队列中，不重复处理）。C.color=BLACK，Q=[D]。

**第4步**：出队D。遍历D的邻居B（BLACK）、C（BLACK）和E（WHITE）。设置E.color=GRAY，E.d=3，E.π=D。E入队，Q=[E]。D.color=BLACK。

**第5步**：出队E。遍历E的邻居D（BLACK）。E.color=BLACK，Q=∅。算法结束。

最终结果：A.d=0，B.d=1，C.d=1，D.d=2，E.d=3。广度优先树为A→B→D→E和A→C（C是A的直接子节点，D是B的子节点，E是D的子节点）。

## 四、复杂度/性质分析
**时间复杂度**：对于图G=(V,E)，BFS的时间复杂度为O(|V|+|E|)。每个节点入队出队各一次，耗时O(|V|)；每条边在探索其端点时被检查一次（无向图每条边检查两次），耗时O(|E|)。因此总时间为O(|V|+|E|)。无论图的结构如何，这一上界都是紧的。

**空间复杂度**：BFS需要存储队列、颜色数组、距离数组和前驱数组，空间复杂度为O(|V|)。在最坏情况下（如完全图），队列可能同时包含几乎所有节点。

**稳定性与收敛性**：BFS是确定性的——给定相同的图和起始节点，执行结果完全一致。在有限图中，BFS必然在有限步内终止（收敛），因为每个节点最多被访问一次。在无限图中，若从起点出发存在无限路径，BFS可能永不终止。

**最短路径性质**：在无权图中，BFS保证找到从起点到所有可达节点的最短路径（边数最少）。这一性质不依赖于图的连通性或稠密度。

**与DFS的对比**：BFS使用队列（空间O(V)），DFS使用栈或递归（空间O(h)，h为深度）。BFS适合找最短路径，DFS适合拓扑排序、连通分量等。

## 五、常见误区
1. **混淆BFS与DFS的遍历顺序**：BFS按“层”遍历，先访问离起点近的节点；DFS沿一条路径深入到底再回溯。初学者常误以为BFS也像DFS一样“一条路走到黑”，实际上BFS是“逐层推进”。

2. **误认为BFS能处理带权图的最短路径**：BFS仅适用于无权图或所有边权相等的图。对于带权图（如边权不同），BFS无法保证找到最短路径，此时应使用Dijkstra算法或Bellman-Ford算法。

3. **忽略图可能不连通**：若从起点出发无法到达某些节点，BFS仅遍历起点所在的连通分量。初学者常假设BFS能访问所有节点，实际需要检查所有节点的color状态，对未访问的节点重新启动BFS。

4. **错误处理重复访问**：在BFS中，节点首次被发现时即被标记为GRAY并加入队列，后续再次遇到时不应重复处理。常见错误是在出队时检查节点是否已访问，这可能导致同一节点多次入队，破坏算法正确性。

5. **混淆“距离”与“路径长度”**：BFS计算的d是边数（跳数），而非物理距离或权重和。在树或网格中，d对应层数；在社交网络中，d对应“六度分隔”中的度数。初学者有时会误以为d是欧几里得距离或其他度量。
