# 广度优先搜索

## 一、定义
广度优先搜索（Breadth-First Search，BFS）是一种用于遍历或搜索树或图数据结构的算法。它从选定的起始顶点出发，系统地探索所有距离该顶点为 $k$ 的顶点，然后才探索距离为 $k+1$ 的顶点。BFS 保证在无权图中找到从源点到所有可达顶点的最短路径（以边数计）。

**核心数学性质**：对于图 $G = (V, E)$ 和源点 $s \in V$，BFS 计算每个顶点 $v \in V$ 的距离 $\delta(s, v)$（从 $s$ 到 $v$ 的最短路径边数），并生成一棵广度优先树。

**伪代码**（基于队列实现）：
```
BFS(G, s):
    for each vertex u ∈ G.V - {s}:
        u.color = WHITE
        u.d = ∞
        u.π = NIL
    s.color = GRAY
    s.d = 0
    s.π = NIL
    Q = ∅
    ENQUEUE(Q, s)
    while Q ≠ ∅:
        u = DEQUEUE(Q)
        for each v ∈ G.Adj[u]:
            if v.color == WHITE:
                v.color = GRAY
                v.d = u.d + 1
                v.π = u
                ENQUEUE(Q, v)
        u.color = BLACK
```

## 二、核心原理
1. **层级推进策略**：BFS 按照顶点与源点的距离分层探索。所有距离为 $d$ 的顶点被完全探索后，才开始探索距离为 $d+1$ 的顶点。这种"逐层推进"的特性保证了最短路径性质。

2. **队列作为核心数据结构**：BFS 使用先进先出（FIFO）队列管理待访问顶点。队列保证了顶点按照"先被发现、先被探索"的顺序处理，这是实现层级推进的关键机制。

3. **三色标记法**：每个顶点有三种状态——白色（未发现）、灰色（已发现但邻接点未完全探索）、黑色（已完全探索）。灰色顶点构成探索的"前沿边界"，确保算法不会重复访问顶点。

4. **最短路径保证**：在无权图中，BFS 首次发现顶点 $v$ 时所记录的路径长度 $v.d$ 就是从源点到 $v$ 的最短路径长度。这一性质源于 BFS 按距离递增顺序发现顶点的特性。

5. **广度优先树**：BFS 在探索过程中记录每个顶点的前驱 $\pi$，形成一棵以源点为根的树（或森林），树中从根到任意节点的路径即为该节点在原始图中的最短路径。

## 三、过程/示例
考虑无向图 $G$，顶点集 $\{0,1,2,3,4,5\}$，边集 $\{(0,1), (0,2), (1,3), (2,3), (2,4), (3,5), (4,5)\}$。以顶点 0 为源点执行 BFS。

**初始化**：所有顶点白色，$d$ 为 $\infty$。源点 0 变灰，$d=0$，入队。队列：$[0]$。

**步骤 1**：出队 0，探索邻接点 1 和 2。两者均为白色，变灰，$d=1$，前驱设为 0，入队。0 变黑。队列：$[1, 2]$。

**步骤 2**：出队 1，探索邻接点 0（黑色，跳过）和 3（白色）。3 变灰，$d=2$，前驱为 1，入队。1 变黑。队列：$[2, 3]$。

**步骤 3**：出队 2，探索邻接点 0（黑）、3（灰，已发现但未探索，不处理）、4（白）。4 变灰，$d=2$，前驱为 2，入队。2 变黑。队列：$[3, 4]$。

**步骤 4**：出队 3，探索邻接点 1（黑）、2（黑）、5（白）。5 变灰，$d=3$，前驱为 3，入队。3 变黑。队列：$[4, 5]$。

**步骤 5**：出队 4，探索邻接点 2（黑）、5（灰，跳过）。4 变黑。队列：$[5]$。

**步骤 6**：出队 5，探索邻接点 3（黑）、4（黑）。5 变黑。队列空，算法结束。

**结果**：各顶点距离：$d[0]=0, d[1]=1, d[2]=1, d[3]=2, d[4]=2, d[5]=3$。广度优先树：$0 \to 1 \to 3 \to 5$ 和 $0 \to 2 \to 4$ 等路径。

## 四、复杂度/性质分析
**时间复杂度**：
- 邻接表表示：$O(|V| + |E|)$。每个顶点入队出队一次（$O(|V|)$），每条边在探索其端点时被检查一次（$O(|E|)$）。
- 邻接矩阵表示：$O(|V|^2)$。检查每个顶点的所有邻接点需要遍历整行。

**空间复杂度**：$O(|V|)$。队列最多存储 $|V|$ 个顶点，颜色和距离数组各需 $|V|$ 空间。

**关键性质**：
- **最短路径性**：在无权图中，BFS 正确计算所有可达顶点的最短路径长度。
- **完备性**：对于有限图，BFS 保证访问所有从源点可达的顶点。
- **最优性**：在无权图中，BFS 找到的路径是边数最少的路径。
- **非适应性**：BFS 不依赖边的权重，仅考虑边的存在性。

**与 DFS 对比**：BFS 使用队列（FIFO），DFS 使用栈（LIFO）；BFS 适合找最短路径，DFS 适合拓扑排序和连通分量检测；BFS 空间复杂度在最坏情况下（完全图）为 $O(|V|)$，DFS 为 $O(|V|)$（递归栈深度）。

## 五、常见误区
1. **混淆 BFS 与 Dijkstra 算法**：BFS 仅适用于无权图（或等权图），而 Dijkstra 算法处理带非负权重的图。初学者常错误地用 BFS 处理带权图的最短路径问题。

2. **忽略图不连通的情况**：BFS 从单个源点出发，只能访问该源点所在的连通分量。若需遍历整个图，必须对每个未访问顶点重复执行 BFS。

3. **错误理解"最短路径"的适用范围**：BFS 保证的是"边数最少"的最短路径，而非"权重和最小"的最短路径。在带权图中，边数最少的路径不一定权重最小。

4. **混淆灰色顶点与黑色顶点的处理**：灰色顶点已被发现但尚未完全探索，其邻接点可能包含未发现的白色顶点；黑色顶点所有邻接点都已被发现。错误地将灰色顶点当作已完全探索会导致遗漏路径。

5. **忽视 BFS 树与原始图的关系**：BFS 树中的边是原始图的子集，但原始图中可能存在不在 BFS 树中的边（称为"交叉边"或"横向边"）。这些边连接同一层或相邻层的顶点，不会影响最短路径距离的计算。
