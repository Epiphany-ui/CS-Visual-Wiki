# 插入排序

## 一、定义

插入排序（Insertion Sort）是一种基于比较的排序算法，其核心思想是通过构建有序序列，对未排序数据在已排序序列中从后向前扫描，找到相应位置并插入。该算法类似于人们整理扑克牌时的手动排序方法。

给定一个包含 n 个元素的数组 A[1..n]，插入排序的伪代码如下：

```
for j = 2 to n:
    key = A[j]
    i = j - 1
    while i > 0 and A[i] > key:
        A[i+1] = A[i]
        i = i - 1
    A[i+1] = key
```

其中，A[1..j-1] 始终保持有序状态，称为已排序子数组；A[j..n] 为待排序部分。

## 二、核心原理

1. **增量式构建有序序列**：算法从数组的第二个元素开始，逐步将每个新元素插入到前面已排序子数组的正确位置，最终使整个数组有序。

2. **原地排序（In-place）**：插入排序仅使用常数级别的额外空间（用于存储 key 和循环变量），直接在原数组上进行元素移动和插入操作。

3. **稳定排序**：当待插入元素与已排序序列中的元素相等时，算法将其插入到相等元素的后面，因此不会改变相等元素的相对顺序。

4. **自适应（Adaptive）**：对于已经部分有序的输入，插入排序的效率显著提高。当输入数组基本有序时，其性能接近线性。

5. **在线算法**：插入排序可以在接收数据的同时进行排序，即不必一次性获得全部数据，可以逐个处理新到达的元素。

## 三、过程/示例

以数组 [5, 2, 4, 6, 1, 3] 为例，演示插入排序的完整过程：

**初始状态**：A = [5, 2, 4, 6, 1, 3]

**第1轮（j=2）**：key = 2
- 比较 5 > 2，将 5 后移一位
- 数组变为 [5, 5, 4, 6, 1, 3]
- 将 key=2 插入位置 1
- 结果：[2, 5, 4, 6, 1, 3]

**第2轮（j=3）**：key = 4
- 比较 5 > 4，将 5 后移
- 数组变为 [2, 5, 5, 6, 1, 3]
- 比较 2 < 4，停止移动
- 将 key=4 插入位置 2
- 结果：[2, 4, 5, 6, 1, 3]

**第3轮（j=4）**：key = 6
- 比较 5 < 6，无需移动
- 直接插入原位置
- 结果：[2, 4, 5, 6, 1, 3]

**第4轮（j=5）**：key = 1
- 依次比较 6>1, 5>1, 4>1, 2>1，全部后移
- 数组变为 [2, 2, 4, 5, 6, 3]
- 将 key=1 插入位置 1
- 结果：[1, 2, 4, 5, 6, 3]

**第5轮（j=6）**：key = 3
- 比较 6>3, 5>3, 4>3，后移
- 数组变为 [1, 2, 4, 4, 5, 6]
- 比较 2 < 3，停止移动
- 将 key=3 插入位置 3
- 最终结果：[1, 2, 3, 4, 5, 6]

## 四、复杂度/性质分析

**时间复杂度**：
- **最好情况**：O(n)——当输入数组已经有序时，内层 while 循环每次比较一次即退出，总共进行 n-1 次比较。
- **最坏情况**：O(n²)——当输入数组完全逆序时，第 j 个元素需要与前面 j-1 个元素全部比较并移动，总比较次数为 1+2+...+(n-1) = n(n-1)/2。
- **平均情况**：O(n²)——对于随机排列的数组，平均比较次数约为 n²/4。

**空间复杂度**：O(1)，仅需常数级别的额外空间。

**稳定性**：稳定。当 A[i] == key 时，while 循环条件不成立，key 被插入到相等元素的后面。

**收敛性**：算法保证在有限步内终止，每次迭代至少将一个元素放置到最终位置（最坏情况下需要 n-1 轮）。

**比较与交换次数**：比较次数与逆序对数量直接相关；元素移动次数等于比较次数加上 n-1（每次插入的赋值操作）。

## 五、常见误区

1. **与选择排序混淆**：选择排序每次从未排序部分选出最小值放到已排序末尾，而插入排序是将当前元素插入到已排序部分的正确位置。两者虽然都是 O(n²) 算法，但插入排序在部分有序数据上表现更好，且是稳定排序。

2. **误认为插入排序总是从第一个元素开始**：实际上，第一个元素（索引为1）被视为已排序的初始序列，算法从第二个元素（j=2）开始处理。

3. **忽略 while 循环的短路特性**：在伪代码中，条件 `i > 0 and A[i] > key` 利用了逻辑与的短路求值。当 i 变为 0 时，不再比较 A[0] > key，避免了数组越界。

4. **混淆比较次数与移动次数**：在最坏情况下，比较次数和移动次数都是 O(n²)，但移动次数通常略大于比较次数，因为每次比较后可能伴随一次元素后移操作。

5. **认为插入排序不适合任何实际应用**：尽管平均时间复杂度为 O(n²)，但对于小规模数据（n ≤ 50）或基本有序的数据，插入排序的实际运行速度往往快于复杂的 O(n log n) 算法，因此常作为高级排序算法（如快速排序、归并排序）的底层优化。
