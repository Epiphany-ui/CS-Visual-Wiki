# 插入排序

## 一、定义

插入排序（Insertion Sort）是一种基于比较的排序算法，其核心思想是将待排序序列分为已排序和未排序两部分，每次从未排序部分取出第一个元素，将其插入到已排序部分的正确位置，直至所有元素有序。

给定长度为 \( n \) 的数组 \( A[1..n] \)，插入排序的伪代码如下：

```
for j = 2 to n:
    key = A[j]
    i = j - 1
    while i > 0 and A[i] > key:
        A[i+1] = A[i]
        i = i - 1
    A[i+1] = key
```

该算法通过逐步构建有序序列，最终使整个数组非递减排列。

## 二、核心原理

1. **分治思想**：插入排序将数组逻辑上划分为已排序区（左侧）和未排序区（右侧）。初始时，已排序区仅包含第一个元素，其余元素均属于未排序区。

2. **元素插入机制**：每次从未排序区取出第一个元素（称为“键值”），在已排序区中从右向左扫描，找到第一个小于等于该键值的元素位置，将键值插入其后。若已排序区所有元素均大于键值，则插入到最左端。

3. **就地排序**：插入排序仅使用常数级别的额外空间（如临时变量 `key`），所有操作在原数组上进行，属于原地排序算法。

4. **稳定性**：当键值与已排序区中某元素相等时，算法将其插入到该元素之后，不改变相等元素的相对顺序，因此插入排序是稳定的。

5. **在线性**：插入排序可以一边接收新数据一边排序，即支持在线处理。当新元素到达时，只需将其插入到当前已排序序列的适当位置即可。

## 三、过程/示例

以数组 `[5, 2, 4, 6, 1, 3]` 为例，演示插入排序的完整过程：

**初始状态**：已排序区 = `[5]`，未排序区 = `[2, 4, 6, 1, 3]`

**第1轮**（j=2）：取出 key = 2，与已排序区 `[5]` 比较。5 > 2，将5右移一位，得到 `[5, 5]`，将2插入到位置1。结果：`[2, 5, 4, 6, 1, 3]`

**第2轮**（j=3）：取出 key = 4，与已排序区 `[2, 5]` 从右向左比较。5 > 4，将5右移，得到 `[2, 5, 5]`；2 < 4，停止。将4插入到位置2。结果：`[2, 4, 5, 6, 1, 3]`

**第3轮**（j=4）：取出 key = 6，与已排序区 `[2, 4, 5]` 比较。5 < 6，无需移动，直接插入到位置4。结果：`[2, 4, 5, 6, 1, 3]`

**第4轮**（j=5）：取出 key = 1，与已排序区 `[2, 4, 5, 6]` 比较。所有元素均大于1，依次右移，得到 `[2, 4, 5, 6, 6]` → `[2, 4, 5, 5, 6]` → `[2, 4, 4, 5, 6]` → `[2, 2, 4, 5, 6]`，将1插入到位置1。结果：`[1, 2, 4, 5, 6, 3]`

**第5轮**（j=6）：取出 key = 3，与已排序区 `[1, 2, 4, 5, 6]` 比较。6 > 3，右移；5 > 3，右移；4 > 3，右移；2 < 3，停止。将3插入到位置3。结果：`[1, 2, 3, 4, 5, 6]`

排序完成。

## 四、复杂度/性质分析

- **时间复杂度**：
  - **最好情况**：输入数组已有序，每轮只需比较一次，无需移动元素，时间复杂度为 \( O(n) \)。
  - **最坏情况**：输入数组逆序，每轮需比较和移动所有已排序元素，总比较次数为 \( \sum_{j=2}^{n} (j-1) = \frac{n(n-1)}{2} \)，时间复杂度为 \( O(n^2) \)。
  - **平均情况**：输入随机排列，平均比较次数约为 \( \frac{n^2}{4} \)，时间复杂度为 \( O(n^2) \)。

- **空间复杂度**：仅使用常数个辅助变量，空间复杂度为 \( O(1) \)。

- **稳定性**：稳定排序，相等元素的相对顺序保持不变。

- **收敛性**：算法在有限步内终止，每轮至少将一个元素归位，经过 \( n-1 \) 轮后必然得到有序序列。

- **适应性**：插入排序对部分有序的数据效率较高，当逆序对数量较少时，实际运行时间接近线性。

## 五、常见误区

1. **混淆插入排序与选择排序**：插入排序每次将未排序区的第一个元素插入到已排序区的正确位置；选择排序每次从未排序区选出最小元素放到已排序区末尾。两者过程截然不同，初学者常因“逐步构建有序序列”的相似性而混淆。

2. **误认为插入排序总是 \( O(n^2) \)**：忽略最好情况下的线性复杂度。当输入数据基本有序时，插入排序效率很高，甚至优于某些 \( O(n \log n) \) 算法。

3. **错误理解稳定性**：认为插入排序不稳定。实际上，当键值与已排序区元素相等时，算法将其插入到该元素之后，不改变相对顺序，因此是稳定的。

4. **混淆“插入”与“交换”**：插入排序通过移动元素腾出位置，而非直接交换。初学者可能误以为每次比较后立即交换，导致实现效率低下。

5. **忽略边界条件**：在实现时，容易忘记处理键值小于已排序区所有元素的情况（即插入到位置1），导致数组越界或元素丢失。
