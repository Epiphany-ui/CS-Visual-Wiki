# 贪心算法

## 一、定义

贪心算法（Greedy Algorithm）是一种在每一步决策时都采取当前状态下最优（即最有利）的选择，从而希望最终得到全局最优解的算法设计范式。其核心数学描述为：给定一个优化问题，设解空间为 \( S \)，目标函数为 \( f: S \rightarrow \mathbb{R} \)，贪心算法通过构造一个选择序列 \( x_1, x_2, \ldots, x_n \)，使得每一步的选择 \( x_i \) 满足：

\[
x_i = \arg\max_{x \in C_i} g(x) \quad \text{或} \quad x_i = \arg\min_{x \in C_i} g(x)
\]

其中 \( C_i \) 是第 \( i \) 步的可行候选集，\( g \) 是局部评价函数。贪心算法的通用伪代码如下：

```
GREEDY(C)
    S ← ∅
    while C ≠ ∅ and solution not complete do
        x ← SELECT(C)   // 根据贪心策略选择最优候选
        if FEASIBLE(S ∪ {x}) then
            S ← S ∪ {x}
        C ← C \ {x}
    return S
```

## 二、核心原理

贪心算法的核心思想建立在以下三个关键机制之上：

1. **贪心选择性质（Greedy Choice Property）**：全局最优解可以通过一系列局部最优选择来构造。这意味着，一旦做出选择，就不需要回溯或重新考虑之前的决策。这是贪心算法能够正确工作的首要条件。

2. **最优子结构（Optimal Substructure）**：一个问题的最优解包含其子问题的最优解。如果一个问题同时满足贪心选择性质和最优子结构，则贪心算法可以保证得到全局最优解。

3. **无后效性（Memorylessness）**：当前的选择只影响未来的状态，而不受未来选择的影响。贪心算法在每一步只依赖当前信息，不关心过去的选择如何影响未来的可能性。

4. **局部评价函数**：需要一个明确定义的准则来比较候选解，该准则能够量化当前步骤中每个选项的“优劣”程度。

5. **可行性检查**：在每次选择后，必须验证当前部分解是否满足问题的约束条件，确保不会构造出无效解。

## 三、过程/示例

以经典的**找零问题**为例：假设有面值为 25、10、5、1 美分的硬币，需要找零 63 美分，要求使用最少数量的硬币。

**贪心策略**：每次选择不超过剩余金额的最大面值硬币。

**执行过程**：

- **初始状态**：剩余金额 = 63，硬币集合 = ∅
- **第1步**：最大可用硬币为 25，63 ≥ 25，选择 25。剩余金额 = 63 - 25 = 38，硬币集合 = {25}
- **第2步**：最大可用硬币为 25，38 ≥ 25，选择 25。剩余金额 = 38 - 25 = 13，硬币集合 = {25, 25}
- **第3步**：最大可用硬币为 25，13 < 25，跳过；下一个为 10，13 ≥ 10，选择 10。剩余金额 = 13 - 10 = 3，硬币集合 = {25, 25, 10}
- **第4步**：最大可用硬币为 10，3 < 10，跳过；下一个为 5，3 < 5，跳过；下一个为 1，3 ≥ 1，选择 1。剩余金额 = 3 - 1 = 2，硬币集合 = {25, 25, 10, 1}
- **第5步**：继续选择 1。剩余金额 = 2 - 1 = 1，硬币集合 = {25, 25, 10, 1, 1}
- **第6步**：继续选择 1。剩余金额 = 1 - 1 = 0，硬币集合 = {25, 25, 10, 1, 1, 1}

**最终结果**：使用 6 枚硬币（2 枚 25 美分、1 枚 10 美分、3 枚 1 美分）。对于美国硬币系统，贪心算法确实能得到最优解。

## 四、复杂度/性质分析

**时间复杂度**：
- **最好情况**：\( O(n) \)，当每次选择后问题规模迅速减小，或候选集已预排序。
- **最坏情况**：\( O(n \log n) \) 至 \( O(n^2) \)，取决于选择操作的实现。若每次选择需要扫描所有候选，则为 \( O(n^2) \)；若先排序再依次选择，则为 \( O(n \log n) \)。
- **平均情况**：通常为 \( O(n \log n) \)，因为排序是常见预处理步骤。

**空间复杂度**：
- 通常为 \( O(n) \)，用于存储候选集和部分解。若允许原地修改，可降至 \( O(1) \)。

**稳定性**：
- 贪心算法本身不涉及稳定性概念（这是排序算法的性质）。但贪心策略的选择顺序可能影响结果，当存在多个等价最优解时，不同实现可能得到不同解。

**收敛性**：
- 对于满足贪心选择性质的问题，算法保证在有限步内收敛到全局最优解。
- 对于不满足该性质的问题，贪心算法可能收敛到局部最优解，无法保证全局最优。

**正确性条件**：
- 必须同时满足贪心选择性质和最优子结构，才能保证全局最优。
- 许多问题（如背包问题的分数版本）满足这些性质，而另一些（如0-1背包问题）则不满足。

## 五、常见误区

1. **误以为贪心总是最优**：最常见的错误是认为贪心算法在所有优化问题上都能得到全局最优解。实际上，只有满足贪心选择性质和最优子结构的问题才能保证最优性。例如，0-1背包问题使用贪心（按价值/重量比）可能得到次优解。

2. **混淆贪心与动态规划**：两者都利用最优子结构，但贪心是“向前看一步”，做出不可撤销的选择；动态规划则考虑所有可能的选择，通过状态转移方程求解。贪心是动态规划的特例，但适用范围更窄。

3. **忽视排序预处理**：许多贪心算法（如活动选择、霍夫曼编码）需要先对候选集排序。忘记排序或使用错误的排序标准会导致算法失效。

4. **错误应用局部评价函数**：将局部最优等同于全局最优。例如，在最小生成树问题中，每次选择最小权值边（Kruskal算法）是正确的，但若改为选择最小权值顶点则错误。

5. **忽略可行性检查**：贪心算法在每一步必须验证当前选择是否可行。例如，在图的着色问题中，选择最小可用颜色时，必须确保不与相邻顶点冲突，否则可能构造出非法解。
