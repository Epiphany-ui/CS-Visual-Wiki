# 双指针

## 一、定义
双指针（Two Pointers）是一种在数组、链表等线性数据结构中，通过维护两个指针变量来协同遍历或搜索的算法技术。其核心思想是利用指针的移动策略（如相向移动、同向移动或快慢移动），在单次遍历中完成原本需要嵌套循环才能实现的任务。形式上，给定一个长度为n的序列A，双指针通常定义为两个索引变量i和j（0 ≤ i, j < n），通过调整i和j的移动规则，在O(n)时间内求解特定问题。典型伪代码如下：

```
i = 0, j = n-1  // 相向双指针
while i < j:
    if condition(A[i], A[j]):
        process(i, j)
        i++
    else:
        j--
```

## 二、核心原理
1. **单调性约束**：双指针算法依赖问题的单调性质。例如在有序数组中，当左指针右移时，右指针只能左移（或保持不动），这种单调移动保证了每个元素最多被访问两次，从而避免回溯。

2. **状态空间压缩**：暴力解法通常需要O(n²)枚举所有元素对，双指针通过指针的协同移动，将状态空间从二维压缩到一维，每次只考察当前指针位置对应的有限候选解。

3. **滑动窗口机制**：在同向双指针（快慢指针）中，两个指针构成一个动态窗口。窗口的扩张（快指针移动）和收缩（慢指针移动）遵循特定条件，常用于子数组/子串问题。

4. **收敛性保证**：双指针算法必然在有限步内终止。每次迭代至少有一个指针向终止条件移动一步，因此最多执行O(n)次迭代。

5. **无后效性**：指针的移动决策仅基于当前状态和问题约束，不依赖于未来未访问的元素，这保证了算法的正确性和局部最优性。

## 三、过程/示例
**问题**：在有序数组 `[1, 3, 5, 7, 9, 11]` 中找出两个数，使其和为 `12`。

**步骤**：
1. 初始化左指针 `i = 0`（指向1），右指针 `j = 5`（指向11）。
2. 计算 `sum = A[i] + A[j] = 1 + 11 = 12`，正好等于目标值，返回 `(1, 11)`。
3. 若 `sum < target`，则左指针右移（增大和）；若 `sum > target`，则右指针左移（减小和）。本例中直接命中目标。

**更复杂的例子**：目标值为 `10`。
1. `i=0, j=5`：`1+11=12 > 10`，右指针左移 → `j=4`（指向9）。
2. `i=0, j=4`：`1+9=10`，命中目标，返回 `(1, 9)`。

**目标值为 `8`**：
1. `i=0, j=5`：`12 > 8` → `j=4`（9）。
2. `i=0, j=4`：`10 > 8` → `j=3`（7）。
3. `i=0, j=3`：`8 = 8`，命中目标，返回 `(1, 7)`。

**目标值为 `4`**：
1. `i=0, j=5`：`12 > 4` → `j=4`（9）。
2. 持续左移右指针直到 `j=1`（3）：`1+3=4`，命中目标。

整个过程指针从未回溯，每个元素最多被访问一次。

## 四、复杂度/性质分析
- **时间复杂度**：最好情况O(1)（首次比较即命中），最坏和平均情况均为O(n)。每个指针最多移动n步，总移动次数不超过2n。
- **空间复杂度**：O(1)，仅需存储两个指针变量。
- **稳定性**：双指针算法本身不涉及元素交换，稳定性取决于具体实现（如用于排序时需额外注意）。
- **收敛性**：严格收敛，每次迭代至少有一个指针向边界移动，最多n次迭代后必然终止。
- **适用条件**：要求问题具有单调性（如数组有序）或可分解性（如滑动窗口问题）。不适用于需要全局回溯或元素间无约束关系的场景。

## 五、常见误区
1. **混淆双指针与二分查找**：二分查找每次将搜索区间减半，而双指针每次只移动一个指针一步。二分查找适用于单点搜索，双指针适用于配对或区间问题。

2. **忽视单调性前提**：在无序数组中使用相向双指针求两数之和，可能漏解。例如数组 `[3, 1, 2]` 目标和为4，双指针无法正确工作，必须先排序。

3. **指针越界错误**：在快慢指针中，快指针移动速度是慢指针的两倍，需确保快指针的下一步（如 `fast.next.next`）不为空，否则引发空指针异常。

4. **窗口收缩条件错误**：滑动窗口问题中，收缩窗口（移动慢指针）的条件必须是当前窗口不再满足约束，而非提前收缩。例如求最长无重复子串时，应在发现重复字符后才移动慢指针。

5. **误认为双指针总能替代哈希表**：双指针要求数据有序或具有单调性，而哈希表（如两数之和问题）可在无序数组中O(n)求解，但需要O(n)额外空间。两者各有优劣，需根据问题特性选择。
