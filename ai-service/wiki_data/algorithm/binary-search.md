# 二分查找

## 一、定义
二分查找（Binary Search），又称折半查找，是一种在有序数组中查找特定元素的高效算法。其核心思想是通过不断将搜索区间对半分割，逐步缩小目标元素可能存在的范围，直至找到目标或确定其不存在。给定一个长度为 \( n \) 的有序数组 \( A \) 和目标值 \( T \)，二分查找的数学描述如下：设搜索区间为 \([L, R]\)（初始时 \( L=0, R=n-1 \)），计算中点 \( mid = \lfloor (L+R)/2 \rfloor \)，若 \( A[mid] = T \) 则查找成功；若 \( A[mid] < T \) 则令 \( L = mid+1 \)；否则令 \( R = mid-1 \)。重复此过程直至 \( L > R \)。其伪代码为：

```
function binarySearch(A, n, T):
    L = 0
    R = n - 1
    while L <= R:
        mid = floor((L + R) / 2)
        if A[mid] == T:
            return mid
        else if A[mid] < T:
            L = mid + 1
        else:
            R = mid - 1
    return -1
```

## 二、核心原理
1. **有序性前提**：二分查找要求输入数组必须是有序的（通常为升序或降序）。这一前提保证了每次比较后可以确定目标位于中点的哪一侧，从而排除另一半区间。
2. **分治策略**：二分查找是分治思想的典型应用。它将原问题（在完整数组中查找）分解为规模更小的子问题（在左半或右半数组中查找），且子问题与原问题性质相同，通过递归或迭代求解。
3. **区间缩减**：每次迭代将搜索区间长度至少减半。设当前区间长度为 \( k \)，则下一次区间长度不超过 \( \lfloor k/2 \rfloor \)。这一性质直接决定了算法的高效性。
4. **比较操作**：每次迭代仅需一次比较（与中点元素比较），即可决定下一步方向。比较操作是算法中唯一的决策依据，因此比较函数的正确性至关重要。
5. **终止条件**：算法在两种情况下终止：找到目标（返回索引）或区间为空（\( L > R \)，返回失败标志）。终止条件的正确设计避免了死循环。

## 三、过程/示例
考虑有序数组 \( A = [2, 5, 8, 12, 16, 23, 38, 45, 56, 72] \)，长度 \( n=10 \)，目标值 \( T=23 \)。

- **步骤1**：初始化 \( L=0, R=9 \)。计算 \( mid = \lfloor (0+9)/2 \rfloor = 4 \)，\( A[4]=16 \)。因 \( 16 < 23 \)，目标在右半区间，更新 \( L=5 \)。
- **步骤2**：当前 \( L=5, R=9 \)。计算 \( mid = \lfloor (5+9)/2 \rfloor = 7 \)，\( A[7]=45 \)。因 \( 45 > 23 \)，目标在左半区间，更新 \( R=6 \)。
- **步骤3**：当前 \( L=5, R=6 \)。计算 \( mid = \lfloor (5+6)/2 \rfloor = 5 \)，\( A[5]=23 \)。因 \( A[5] = T \)，查找成功，返回索引5。

若目标值 \( T=10 \)：
- **步骤1**：同上，\( L=0, R=9, mid=4, A[4]=16 > 10 \)，更新 \( R=3 \)。
- **步骤2**：\( L=0, R=3, mid=1, A[1]=5 < 10 \)，更新 \( L=2 \)。
- **步骤3**：\( L=2, R=3, mid=2, A[2]=8 < 10 \)，更新 \( L=3 \)。
- **步骤4**：\( L=3, R=3, mid=3, A[3]=12 > 10 \)，更新 \( R=2 \)。
- **步骤5**：此时 \( L=3 > R=2 \)，区间为空，返回-1表示查找失败。

## 四、复杂度/性质分析
- **时间复杂度**：
  - **最好情况**：\( O(1) \)，即第一次比较就找到目标（中点恰好是目标）。
  - **最坏情况**：\( O(\log n) \)，每次比较后区间减半，最多需要 \( \lfloor \log_2 n \rfloor + 1 \) 次比较。
  - **平均情况**：\( O(\log n) \)，数学期望与最坏情况同阶。
- **空间复杂度**：迭代实现为 \( O(1) \)，仅需常数个辅助变量；递归实现为 \( O(\log n) \)（递归调用栈深度）。
- **稳定性**：二分查找本身不涉及元素交换，但若数组中有重复元素，标准实现可能返回任意一个匹配索引，而非第一个或最后一个。可通过修改边界条件实现稳定查找（如查找第一个等于目标的位置）。
- **收敛性**：算法保证在有限步内收敛。每次迭代区间长度严格减小，且终止条件明确，不会出现无限循环。需注意整数溢出问题：当 \( L \) 和 \( R \) 很大时，\( (L+R)/2 \) 可能溢出，建议使用 \( L + (R-L)/2 \) 计算中点。

## 五、常见误区
1. **数组未排序**：初学者常忽略有序性前提，在无序数组上使用二分查找导致错误结果。必须确保数组已排序，或自定义比较函数满足全序关系。
2. **边界条件错误**：循环条件写成 `while (L < R)` 而非 `while (L <= R)`，或更新边界时写成 `L = mid` 而非 `L = mid + 1`，可能导致死循环或漏查元素。例如，当区间长度为1时，若使用 `L < R` 会直接退出循环。
3. **中点计算溢出**：使用 `(L + R) / 2` 在 \( L+R \) 超过整型最大值时产生溢出。正确做法是 `L + (R - L) / 2` 或使用无符号右移。
4. **重复元素处理**：标准二分查找在重复元素中返回任意匹配位置，但实际需求常要求第一个或最后一个出现位置。此时需修改比较逻辑和边界更新策略（如查找第一个时，当 `A[mid] >= T` 时令 `R = mid`）。
5. **与线性查找混淆**：二分查找并非万能，对于小规模数据（如 \( n < 10 \)），线性查找因常数小可能更快；对于动态插入频繁的数据，维护有序性的成本可能超过查找收益。此外，二分查找不适用于链表等随机访问代价高的数据结构。
