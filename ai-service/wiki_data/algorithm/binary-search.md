# 二分查找

## 一、定义
二分查找（Binary Search），又称折半查找，是一种在有序数组中查找特定元素的高效算法。其核心思想是通过不断将搜索区间对半分割，逐步缩小目标元素可能存在的范围，直至找到目标或确定其不存在。给定一个已按非递减顺序排列的数组 `A[0..n-1]` 和待查找值 `target`，二分查找的伪代码如下：

```
function binary_search(A, target):
    left = 0
    right = n - 1
    while left <= right:
        mid = left + (right - left) // 2
        if A[mid] == target:
            return mid
        else if A[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1  // 未找到
```

该算法要求输入数据必须有序，这是其正确工作的前提条件。

## 二、核心原理
二分查找基于分治策略，其核心机制包含以下关键要点：

1. **有序性前提**：算法要求数组元素按单调非递减（或非递增）顺序排列。有序性保证了每次比较后可以确定性地排除一半元素，这是算法正确性的根本保证。

2. **区间收缩机制**：通过维护左边界 `left` 和右边界 `right` 定义当前搜索区间 `[left, right]`。每次取中间位置 `mid` 与目标值比较，根据比较结果将区间缩小为左半部分或右半部分，确保目标值始终在新区间内（若存在）。

3. **中点计算防溢出**：`mid = left + (right - left) // 2` 等价于 `(left + right) // 2`，但避免了 `left + right` 可能导致的整数溢出问题，这是工程实现中的重要细节。

4. **终止条件**：当 `left > right` 时搜索结束。此时若未找到目标，则说明目标不存在于数组中。该条件保证了算法在有限步内终止。

5. **不变式维护**：在每次迭代中，若目标值存在于数组中，则其索引必然在 `[left, right]` 区间内。这一不变式是算法正确性的逻辑基础。

## 三、过程/示例
考虑有序数组 `A = [2, 5, 8, 12, 16, 23, 38, 45, 56, 72]`，查找目标值 `target = 23`。

**初始状态**：`left = 0`，`right = 9`（数组长度 `n = 10`）。

**第1步**：计算 `mid = 0 + (9 - 0) // 2 = 4`，`A[4] = 16`。  
比较：`16 < 23`，目标在右半部分。更新 `left = mid + 1 = 5`。  
当前区间：`[5, 9]`。

**第2步**：计算 `mid = 5 + (9 - 5) // 2 = 7`，`A[7] = 45`。  
比较：`45 > 23`，目标在左半部分。更新 `right = mid - 1 = 6`。  
当前区间：`[5, 6]`。

**第3步**：计算 `mid = 5 + (6 - 5) // 2 = 5`，`A[5] = 23`。  
比较：`23 == 23`，找到目标。返回索引 `5`。

若查找 `target = 10`，则过程类似，最终 `left > right` 时返回 `-1`。

## 四、复杂度/性质分析
1. **时间复杂度**：
   - **最好情况**：`O(1)`，目标恰好位于第一次比较的中间位置。
   - **最坏情况**：`O(log n)`，每次比较后区间缩小一半，需进行 `⌊log₂n⌋ + 1` 次比较。
   - **平均情况**：`O(log n)`，对于随机分布的有序数组，平均比较次数约为 `log₂n`。

2. **空间复杂度**：`O(1)`，仅需常数个额外变量（`left`、`right`、`mid`），属于原地算法。

3. **稳定性**：二分查找本身不涉及元素移动，仅返回索引，因此不存在稳定性问题。但若数组中有重复元素，标准二分查找可能返回任意一个匹配元素的索引。

4. **收敛性**：算法严格单调收敛。每次迭代区间长度至少减少1，保证在有限步内终止。区间长度从 `n` 开始，经过 `k` 次迭代后不超过 `n/2^k`，当 `n/2^k < 1` 时算法必然终止。

5. **比较次数下界**：基于决策树模型，任何基于比较的查找算法在最坏情况下至少需要 `⌈log₂(n+1)⌉` 次比较，二分查找达到了这个理论下界。

## 五、常见误区
1. **混淆有序与无序**：初学者常试图在无序数组上使用二分查找，导致错误结果。必须确保输入数组已排序，否则算法失效。

2. **边界条件处理错误**：循环条件写成 `left < right` 而非 `left <= right`，或更新边界时直接使用 `mid` 而非 `mid ± 1`，可能导致死循环或遗漏元素。例如，当 `left = right` 时，若 `A[mid] != target`，不更新边界将导致无限循环。

3. **中点计算溢出**：使用 `(left + right) // 2` 在 `left + right` 超过整数范围时可能溢出，尤其在C/C++等语言中。应使用 `left + (right - left) // 2` 或无符号右移操作。

4. **重复元素处理**：标准二分查找在存在重复元素时可能返回任意匹配位置。若需查找第一个或最后一个出现位置，需修改算法（如找到目标后继续向左或向右搜索），而非直接返回。

5. **与线性查找混淆**：线性查找适用于无序数据，时间复杂度 `O(n)`；二分查找要求有序，但效率更高。初学者可能在小规模数据上误用二分查找，忽略了排序的额外开销（`O(n log n)`），导致整体效率反而不如线性查找。
