# 前缀和

## 一、定义
前缀和（Prefix Sum）是一种预处理技术，用于快速计算数组中任意连续子数组的和。对于一个长度为 \(n\) 的数组 \(A[0..n-1]\)，其前缀和数组 \(P[0..n-1]\) 定义为：
\[
P[i] = \sum_{k=0}^{i} A[k], \quad 0 \leq i < n
\]
即 \(P[i]\) 存储原数组从第一个元素到第 \(i\) 个元素（包含）的累积和。通常还会定义 \(P[-1] = 0\) 以简化边界处理。利用前缀和，子数组 \(A[l..r]\) 的和可表示为 \(P[r] - P[l-1]\)（当 \(l=0\) 时即为 \(P[r]\)）。

## 二、核心原理
1. **空间换时间**：通过一次 \(O(n)\) 的预处理，将后续每次子数组求和的时间从 \(O(n)\) 降为 \(O(1)\)。这是前缀和最核心的优化思想。
2. **差分性质**：前缀和与差分互为逆运算。给定前缀和数组 \(P\)，可通过差分 \(A[i] = P[i] - P[i-1]\) 恢复原数组。这一性质在区间更新问题中非常有用。
3. **可扩展性**：前缀和思想可推广到多维（如二维前缀和）、带权前缀和、异或前缀和等，用于解决不同运算下的区间查询问题。
4. **离线预处理**：前缀和是一种静态预处理技术，适用于原数组固定、多次查询的场景。若原数组动态变化，需结合树状数组或线段树等数据结构。
5. **边界统一性**：通过定义 \(P[-1]=0\) 或 \(P[0]=A[0]\) 并额外存储 \(P[-1]\)，可将所有子数组求和公式统一为 \(P[r] - P[l-1]\)，避免分类讨论。

## 三、过程/示例
考虑数组 \(A = [3, 1, 4, 1, 5, 9]\)，计算其前缀和数组 \(P\)。

**步骤1**：初始化 \(P[0] = A[0] = 3\)。
**步骤2**：\(P[1] = P[0] + A[1] = 3 + 1 = 4\)。
**步骤3**：\(P[2] = P[1] + A[2] = 4 + 4 = 8\)。
**步骤4**：\(P[3] = P[2] + A[3] = 8 + 1 = 9\)。
**步骤5**：\(P[4] = P[3] + A[4] = 9 + 5 = 14\)。
**步骤6**：\(P[5] = P[4] + A[5] = 14 + 9 = 23\)。

最终得到 \(P = [3, 4, 8, 9, 14, 23]\)。

**查询示例**：求子数组 \(A[1..4] = [1, 4, 1, 5]\) 的和。
使用公式：\(P[4] - P[0] = 14 - 3 = 11\)，与直接计算 \(1+4+1+5=11\) 一致。
求子数组 \(A[2..5] = [4, 1, 5, 9]\) 的和：\(P[5] - P[1] = 23 - 4 = 19\)，验证正确。

## 四、复杂度/性质分析
- **时间复杂度**：
  - 预处理：\(O(n)\)，需遍历原数组一次。
  - 单次查询：\(O(1)\)，仅需两次数组访问和一次减法。
  - 最好/最坏/平均情况均为 \(O(n)\) 预处理 + \(O(1)\) 查询。
- **空间复杂度**：\(O(n)\)，需存储与原数组等长的前缀和数组。若允许修改原数组，可原地计算，空间复杂度降为 \(O(1)\)。
- **稳定性**：前缀和本身是确定性的数值计算，不存在不稳定问题。但在浮点数场景下，累积误差可能随 \(n\) 增大而放大。
- **收敛性**：不适用。前缀和是精确的代数运算，不涉及迭代或逼近过程。
- **正确性**：基于加法结合律和交换律，前缀和公式严格成立，无近似或误差。

## 五、常见误区
1. **混淆前缀和与滑动窗口**：前缀和用于静态数组的区间和查询，而滑动窗口用于动态维护连续子数组的某种性质（如和、最大值）。前缀和无法处理窗口滑动时的增量更新。
2. **忽略边界条件**：计算 \(A[l..r]\) 时，若 \(l=0\)，应直接返回 \(P[r]\)，而非 \(P[r] - P[-1]\)。初学者常忘记处理 \(l=0\) 的情况，导致数组越界或错误结果。
3. **误用于动态数组**：若原数组频繁更新（如单点修改），每次更新后需重新计算整个前缀和数组，时间复杂度退化为 \(O(n)\)。此时应使用树状数组或线段树。
4. **混淆一维与二维前缀和公式**：二维前缀和 \(P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + A[i][j]\)，子矩阵和公式为 \(P[r2][c2] - P[r1-1][c2] - P[r2][c1-1] + P[r1-1][c1-1]\)。初学者常漏掉减去的重叠部分。
5. **忽视数据类型溢出**：当数组元素和超过整数类型范围时，前缀和可能溢出。在C++/Java等语言中，应使用 `long long` 或 `BigInteger` 等大数类型。
