# Dijkstra算法

## 一、定义

Dijkstra算法是一种用于求解**单源最短路径问题**（Single-Source Shortest Paths）的经典算法，由荷兰计算机科学家Edsger W. Dijkstra于1956年提出。该算法适用于**所有边权为非负**的有向图或无向图，能够计算从给定源节点到图中所有其他节点的最短路径及其距离值。

算法的核心数学形式可表述为：维护一个距离数组`dist[]`，其中`dist[v]`表示当前已知的从源节点`s`到节点`v`的最短路径长度。初始时，`dist[s]=0`，其余节点`dist[v]=∞`。算法反复从尚未确定最短路径的节点集合中，选取`dist`值最小的节点`u`，然后对`u`的所有出边`(u, v, w)`执行**松弛操作**：若`dist[u] + w < dist[v]`，则更新`dist[v] = dist[u] + w`。重复此过程直至所有节点都被处理。

伪代码表示如下：
```
function Dijkstra(Graph, source):
    dist[source] ← 0
    for each vertex v in Graph:
        if v ≠ source: dist[v] ← ∞
        prev[v] ← undefined
    Q ← set of all vertices
    
    while Q is not empty:
        u ← vertex in Q with min dist[u]
        remove u from Q
        for each neighbor v of u:
            alt ← dist[u] + weight(u, v)
            if alt < dist[v]:
                dist[v] ← alt
                prev[v] ← u
    return dist[], prev[]
```

## 二、核心原理

1. **贪心策略**：算法每一步都从尚未处理的节点中选择当前距离最小的节点，认为该节点的最短路径已经确定。这一策略之所以正确，依赖于边权非负的假设——任何后续路径都不可能使已选节点的距离变得更小。

2. **松弛操作**：松弛是算法的基本操作单元，通过检查经过当前节点是否能缩短到其他节点的路径来逐步逼近最优解。每次松弛都可能更新距离值和前驱节点。

3. **最优子结构**：最短路径的子路径也是最短路径。若`s→...→u→v`是`s`到`v`的最短路径，则`s→...→u`必然是`s`到`u`的最短路径。这一性质保证了贪心选择的正确性。

4. **已确定节点集的不变性**：一旦节点`u`被从优先队列中取出，其`dist[u]`值即为最终的最短距离，后续不会再被更新。这是算法能够终止且保证正确性的关键。

5. **非负边权的必要性**：若存在负权边，已确定节点的最短路径可能被后续发现的负权路径所更新，导致算法失效。这是Dijkstra算法与Bellman-Ford算法的根本区别。

## 三、过程/示例

考虑以下有向图，求从节点A到其他所有节点的最短路径：

```
边集：A→B(4), A→C(2), B→C(1), B→D(5), C→D(8), C→E(10), D→E(2)
```

**初始化**：`dist[A]=0`，`dist[B]=∞`，`dist[C]=∞`，`dist[D]=∞`，`dist[E]=∞`。优先队列Q包含所有节点。

**第1步**：取出`dist`最小的节点A(0)。松弛A的出边：
- A→B：`0+4=4 < ∞`，更新`dist[B]=4`
- A→C：`0+2=2 < ∞`，更新`dist[C]=2`
当前距离：A(0已定), B(4), C(2), D(∞), E(∞)

**第2步**：取出当前最小节点C(2)。松弛C的出边：
- C→D：`2+8=10 < ∞`，更新`dist[D]=10`
- C→E：`2+10=12 < ∞`，更新`dist[E]=12`
当前距离：A(0已定), C(2已定), B(4), D(10), E(12)

**第3步**：取出节点B(4)。松弛B的出边：
- B→C：`4+1=5 > 2`，不更新
- B→D：`4+5=9 < 10`，更新`dist[D]=9`
当前距离：A(0), C(2), B(4已定), D(9), E(12)

**第4步**：取出节点D(9)。松弛D的出边：
- D→E：`9+2=11 < 12`，更新`dist[E]=11`
当前距离：A(0), C(2), B(4), D(9已定), E(11)

**第5步**：取出节点E(11)。无出边，算法结束。

**最终结果**：A→A:0, A→B:4(路径A→B), A→C:2(路径A→C), A→D:9(路径A→B→D), A→E:11(路径A→B→D→E)

## 四、复杂度/性质分析

**时间复杂度**：
- 使用**邻接矩阵**+**简单数组**实现：O(V²)，其中V为节点数。每次查找最小距离节点需O(V)，共V次；每条边松弛一次，共E次。
- 使用**邻接表**+**二叉堆**实现：O((V+E)logV)。每次堆操作O(logV)，共V次提取最小和E次松弛更新。
- 使用**邻接表**+**斐波那契堆**实现：O(VlogV+E)，理论最优，但常数较大。

**空间复杂度**：O(V+E)用于存储图（邻接表），加上O(V)用于距离数组和前驱数组。

**性质分析**：
- **正确性**：在非负边权条件下，算法保证找到所有节点的最短路径。
- **稳定性**：算法本身不涉及排序，但若使用优先队列，相同距离的节点处理顺序不影响最终结果。
- **收敛性**：算法在有限步内终止，最多执行V次提取操作和E次松弛操作。
- **适应性**：不能处理负权边，不能检测负权环。

## 五、常见误区

1. **误认为Dijkstra算法可以处理负权边**：这是最常见的错误。当存在负权边时，已确定最短路径的节点可能被后续发现的负权路径更新，破坏贪心策略的正确性。此时应使用Bellman-Ford算法或SPFA。

2. **混淆Dijkstra算法与BFS**：BFS用于无权图的最短路径（每条边权视为1），而Dijkstra算法是BFS在加权图上的推广。BFS使用队列，Dijkstra使用优先队列。

3. **忽略堆优化的重要性**：在稠密图中，使用二叉堆的O((V+E)logV)可能不如简单数组的O(V²)高效。初学者常盲目使用堆优化而不考虑图的规模。

4. **错误理解“已确定节点”的含义**：节点被从优先队列取出时其距离值即确定，后续不应再被更新。若在实现中允许对已取出节点再次松弛，会导致算法错误。

5. **混淆最短路径与最小生成树**：Dijkstra算法求的是从源点到各点的最短路径（路径和最小），而Prim算法求的是连接所有节点的最小生成树（总权值最小）。两者虽然都使用贪心策略和优先队列，但目标函数和更新规则完全不同。
