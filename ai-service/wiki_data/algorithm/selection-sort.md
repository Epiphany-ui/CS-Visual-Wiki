# 选择排序

## 一、定义
选择排序（Selection Sort）是一种基于比较的原地排序算法。其核心思想是：每次从未排序部分中选出最小（或最大）元素，将其放置到已排序部分的末尾。对于长度为n的数组A，其伪代码可表示为：

```
for i = 0 to n-2:
    min_idx = i
    for j = i+1 to n-1:
        if A[j] < A[min_idx]:
            min_idx = j
    swap(A[i], A[min_idx])
```

该算法通过n-1轮选择与交换完成排序，每轮确定一个元素的最终位置。

## 二、核心原理
1. **分治思想**：将数组逻辑划分为已排序区（左侧）和未排序区（右侧），初始时已排序区为空，未排序区为整个数组。

2. **极值选择机制**：每轮遍历未排序区，通过线性扫描找到最小元素的下标，这是算法唯一的信息获取方式。

3. **原地交换**：找到最小元素后，将其与未排序区的第一个元素交换。该操作仅需常数级额外空间，无需辅助数组。

4. **不稳定性**：当最小元素与待交换位置存在相同值元素时，交换可能破坏相等元素的相对顺序。例如数组[5a, 3, 5b]中，第一轮会将5a与3交换，导致5a出现在5b之后。

5. **输入不敏感性**：无论数组初始状态如何，每轮都必须完整遍历未排序区，导致算法性能与输入数据分布无关。

## 三、过程/示例
以数组[29, 10, 14, 37, 13]为例演示升序排序：

**初始状态**：已排序区[]，未排序区[29, 10, 14, 37, 13]

**第1轮**（i=0）：
- 扫描未排序区[29,10,14,37,13]，最小值为10（下标1）
- 交换A[0]与A[1] → [10, 29, 14, 37, 13]
- 已排序区[10]，未排序区[29,14,37,13]

**第2轮**（i=1）：
- 扫描未排序区[29,14,37,13]，最小值为13（下标4）
- 交换A[1]与A[4] → [10, 13, 14, 37, 29]
- 已排序区[10,13]，未排序区[14,37,29]

**第3轮**（i=2）：
- 扫描未排序区[14,37,29]，最小值为14（下标2）
- 无需交换（自身即为最小）→ [10, 13, 14, 37, 29]
- 已排序区[10,13,14]，未排序区[37,29]

**第4轮**（i=3）：
- 扫描未排序区[37,29]，最小值为29（下标4）
- 交换A[3]与A[4] → [10, 13, 14, 29, 37]
- 排序完成

## 四、复杂度/性质分析
1. **时间复杂度**：
   - 最好/最坏/平均情况：均为O(n²)。比较次数恒为n(n-1)/2，交换次数为n-1次。与冒泡排序相比，选择排序的交换次数显著减少（冒泡排序最坏需n(n-1)/2次交换）。

2. **空间复杂度**：O(1)，仅需一个临时变量存储最小元素下标。

3. **稳定性**：不稳定。如前述[5a,3,5b]示例，排序后变为[3,5b,5a]，破坏了原始顺序。

4. **收敛性**：每轮至少将一个元素放置到最终位置，因此最多n-1轮后必然收敛。

5. **适应性**：无适应性。即使输入已有序，仍需执行全部比较操作，无法提前终止。

## 五、常见误区
1. **与冒泡排序混淆**：两者虽同为O(n²)算法，但选择排序每轮仅交换一次，而冒泡排序可能多次交换。在数据移动成本高的场景下，选择排序更优。

2. **误认为稳定**：许多初学者认为“选择最小元素交换”不会破坏稳定性，但实际当存在重复值时，交换操作可能将靠前的重复元素移至后方。

3. **忽略自身交换情况**：当最小元素恰好在待交换位置时，部分实现仍执行交换操作（虽不影响结果但增加开销），优化实现应跳过此情况。

4. **与插入排序的对比误区**：两者均维护已排序区，但选择排序是“选择极值插入末尾”，而插入排序是“将新元素插入已排序区的正确位置”。插入排序在部分有序数据上性能更好。

5. **错误认为可提前终止**：与冒泡排序不同，选择排序无法通过检测某轮无交换来提前结束，因为即使数组已有序，仍需完成所有比较才能确认。
