# 选择排序

## 一、定义
选择排序（Selection Sort）是一种基于比较的就地排序算法。其核心思想是：在未排序序列中找到最小（或最大）元素，将其存放到已排序序列的起始位置，然后从剩余未排序元素中继续寻找最小（或最大）元素，重复此过程直至所有元素排序完毕。

**伪代码描述**：
```
SELECTION-SORT(A, n)
    for i = 1 to n-1
        min_idx = i
        for j = i+1 to n
            if A[j] < A[min_idx]
                min_idx = j
        swap(A[i], A[min_idx])
```

其中，`A`为待排序数组，`n`为数组长度。算法通过两层循环实现：外层循环控制已排序区间的边界，内层循环在未排序区间中寻找最小值索引。

## 二、核心原理
1. **分治思想**：将数组逻辑上分为已排序区间（左侧）和未排序区间（右侧）。初始时已排序区间为空，未排序区间为整个数组。

2. **极值选择**：每次从未排序区间中通过线性扫描找出最小（或最大）元素，这是算法最耗时的操作，时间复杂度为O(n²)。

3. **交换定位**：将找到的最小元素与未排序区间的第一个元素交换，从而将该元素归入已排序区间。每次交换确定一个元素的最终位置。

4. **原地操作**：除交换时使用的临时变量外，不需要额外存储空间，空间复杂度为O(1)。

5. **不稳定性**：选择排序是不稳定的排序算法。当存在重复元素时，交换操作可能改变相同元素的相对顺序。例如，数组[5a, 5b, 1]排序后变为[1, 5b, 5a]，原顺序被破坏。

## 三、过程/示例
以数组`[29, 10, 14, 37, 13]`为例，演示升序排序过程：

**初始状态**：`[29, 10, 14, 37, 13]`（已排序区间为空）

**第1轮**（i=0）：
- 在未排序区间`[29, 10, 14, 37, 13]`中扫描，找到最小值`10`（索引1）
- 交换`A[0]`与`A[1]`：`[10, 29, 14, 37, 13]`
- 已排序区间：`[10]`

**第2轮**（i=1）：
- 在未排序区间`[29, 14, 37, 13]`中扫描，找到最小值`13`（索引4）
- 交换`A[1]`与`A[4]`：`[10, 13, 14, 37, 29]`
- 已排序区间：`[10, 13]`

**第3轮**（i=2）：
- 在未排序区间`[14, 37, 29]`中扫描，找到最小值`14`（索引2）
- 最小值已在正确位置，无需交换：`[10, 13, 14, 37, 29]`
- 已排序区间：`[10, 13, 14]`

**第4轮**（i=3）：
- 在未排序区间`[37, 29]`中扫描，找到最小值`29`（索引4）
- 交换`A[3]`与`A[4]`：`[10, 13, 14, 29, 37]`
- 已排序区间：`[10, 13, 14, 29]`

**最终结果**：`[10, 13, 14, 29, 37]`

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好情况：O(n²) —— 即使数组已有序，仍需执行完整的比较操作。
  - 最坏情况：O(n²) —— 逆序数组时，比较次数不变。
  - 平均情况：O(n²) —— 比较次数恒为n(n-1)/2，与输入数据分布无关。
  
- **空间复杂度**：O(1) —— 仅需常数级额外空间用于交换操作。

- **稳定性**：不稳定。如前述示例，交换操作可能破坏相等元素的相对顺序。

- **收敛性**：算法保证在n-1轮后完成排序，每轮至少将一个元素归位，具有确定的收敛性。

- **比较次数**：固定为n(n-1)/2次，交换次数为n-1次（最好情况）到n-1次（最坏情况，实际交换次数与逆序对数量无关）。

- **适应性**：不具有适应性，即算法性能不随输入数据的有序程度而改变。

## 五、常见误区
1. **混淆选择排序与冒泡排序**：两者都是O(n²)的简单排序，但选择排序每轮只交换一次，而冒泡排序每轮可能多次交换。选择排序的交换次数远少于冒泡排序，但比较次数相同。

2. **误认为选择排序是稳定的**：这是最常见的错误。虽然可以通过修改实现（如使用链表或稳定化处理）使其稳定，但标准的选择排序实现是不稳定的。

3. **认为最好情况复杂度更低**：选择排序的比较次数固定为n(n-1)/2，与输入数据是否有序无关。这与插入排序不同，后者在有序情况下复杂度为O(n)。

4. **混淆选择排序与堆排序**：两者都基于选择思想，但堆排序通过堆数据结构将选择最小值的时间复杂度从O(n)优化到O(log n)，从而将整体复杂度降至O(n log n)。

5. **误认为选择排序可用于大规模数据**：由于O(n²)的时间复杂度，选择排序仅适用于小规模数据（n<1000）。对于大规模数据，应使用归并排序、快速排序等更高效的算法。
