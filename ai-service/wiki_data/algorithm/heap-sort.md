# 堆排序

## 一、定义
堆排序（Heapsort）是一种基于比较的排序算法，它利用堆这种数据结构进行排序。堆是一个近似完全二叉树的结构，并同时满足堆性质：对于最大堆，每个节点的值都大于或等于其子节点的值；对于最小堆，每个节点的值都小于或等于其子节点的值。

堆排序的核心过程可描述为：首先将无序数组构建成一个最大堆（或最小堆），然后反复将堆顶元素（最大值或最小值）与堆的最后一个元素交换，并调整剩余元素使其重新满足堆性质，直至堆中仅剩一个元素。

**伪代码（最大堆排序）：**
```
HEAPSORT(A)
    BUILD-MAX-HEAP(A)
    for i = A.length downto 2
        swap A[1] with A[i]
        heap-size = heap-size - 1
        MAX-HEAPIFY(A, 1)
```

其中 `BUILD-MAX-HEAP` 将数组构建为最大堆，`MAX-HEAPIFY` 维护堆性质。

## 二、核心原理
1. **堆的数组表示**：堆通常用数组实现。若根节点索引为1，则对于索引为 `i` 的节点，其左子节点索引为 `2i`，右子节点索引为 `2i+1`，父节点索引为 `⌊i/2⌋`。这种表示方式使得堆操作无需指针，仅通过索引计算即可完成。

2. **最大堆性质**：对于最大堆，任意节点 `A[i]` 满足 `A[PARENT(i)] ≥ A[i]`。这意味着堆顶元素是整个数组的最大值，且从根到任意叶节点的路径上，元素值单调递减。

3. **MAX-HEAPIFY 操作**：这是堆排序的核心子程序。当节点 `i` 的左、右子树均为最大堆，但 `A[i]` 可能小于其子节点时，`MAX-HEAPIFY` 通过将 `A[i]` 与较大的子节点交换，并递归处理被交换的子节点，使以 `i` 为根的子树重新满足最大堆性质。其时间复杂度为 `O(log n)`。

4. **建堆过程**：从最后一个非叶节点（索引 `⌊n/2⌋`）开始，自底向上对每个节点执行 `MAX-HEAPIFY`。由于叶节点本身满足堆性质，只需处理内部节点。建堆的时间复杂度为 `O(n)`。

5. **排序过程**：建堆后，堆顶元素为最大值。将其与堆的最后一个元素交换，堆大小减1，然后对新的根节点执行 `MAX-HEAPIFY` 恢复堆性质。重复此过程，每次取出当前最大值，最终得到升序排列的数组。

## 三、过程/示例
对数组 `[4, 10, 3, 5, 1]` 进行升序堆排序。

**步骤1：建最大堆**
- 初始数组（索引从1开始）：`[4, 10, 3, 5, 1]`
- 最后一个非叶节点索引：`⌊5/2⌋ = 2`（值为10）
- 对索引2（值10）：左子5，右子1，10 ≥ 5且10 ≥ 1，无需调整
- 对索引1（值4）：左子10，右子3，最大子为10，交换4和10 → `[10, 4, 3, 5, 1]`
- 对索引2（值4）：左子5，右子1，最大子为5，交换4和5 → `[10, 5, 3, 4, 1]`
- 建堆完成：`[10, 5, 3, 4, 1]`

**步骤2：排序**
- 交换堆顶10与末尾1 → `[1, 5, 3, 4, 10]`，堆大小减为4
- 对根节点1执行MAX-HEAPIFY：左子5，右子3，交换1和5 → `[5, 1, 3, 4, 10]`
- 对索引2（值1）：左子4，右子无，交换1和4 → `[5, 4, 3, 1, 10]`
- 交换堆顶5与末尾1 → `[1, 4, 3, 5, 10]`，堆大小减为3
- 对根节点1执行MAX-HEAPIFY：左子4，右子3，交换1和4 → `[4, 1, 3, 5, 10]`
- 交换堆顶4与末尾3 → `[3, 1, 4, 5, 10]`，堆大小减为2
- 对根节点3执行MAX-HEAPIFY：左子1，无需交换
- 交换堆顶3与末尾1 → `[1, 3, 4, 5, 10]`，堆大小减为1
- 排序完成：`[1, 3, 4, 5, 10]`

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好情况：`O(n log n)`（无论输入如何，建堆和排序过程固定）
  - 最坏情况：`O(n log n)`（所有元素逆序时，MAX-HEAPIFY仍为`O(log n)`）
  - 平均情况：`O(n log n)`（建堆`O(n)`，n-1次MAX-HEAPIFY各`O(log n)`）
- **空间复杂度**：`O(1)`（原地排序，仅需常数级额外空间）
- **稳定性**：不稳定。例如数组 `[5, 5, 3]`，建堆后第一个5与3交换，可能破坏相等元素的相对顺序。
- **收敛性**：堆排序保证在有限步内完成，每次迭代至少将一个元素放置到最终位置，因此必然收敛。
- **比较次数**：约为 `2n log n` 量级，常数因子略高于快速排序，但无退化风险。

## 五、常见误区
1. **混淆堆与堆排序**：堆是一种数据结构，支持插入、删除等操作；堆排序是排序算法，利用堆结构实现。堆排序不涉及堆的插入操作，仅使用建堆和调整。

2. **误认为建堆时间复杂度为`O(n log n)`**：实际建堆时间复杂度为`O(n)`。虽然对每个非叶节点调用`MAX-HEAPIFY`，但节点高度越低，调用次数越多，但每次调整时间随高度递减，总和为线性。

3. **混淆最大堆与最小堆的排序结果**：使用最大堆得到升序排列（每次取出最大值放末尾），使用最小堆得到降序排列（每次取出最小值放末尾）。初学者常误以为最大堆直接输出升序。

4. **忽略索引从1开始的约定**：许多教材采用1-based索引，若用0-based索引，子节点公式变为`2i+1`和`2i+2`，父节点为`⌊(i-1)/2⌋`。混淆索引方式会导致数组越界或逻辑错误。

5. **认为堆排序是稳定排序**：堆排序在交换堆顶与末尾元素时，可能将相同键值的元素交换到不同位置，破坏稳定性。这与归并排序等稳定算法形成对比。
