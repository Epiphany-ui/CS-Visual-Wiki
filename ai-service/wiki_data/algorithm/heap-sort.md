# 堆排序

## 一、定义
堆排序（Heapsort）是一种基于比较的排序算法，它利用堆这种数据结构所设计。堆是一个近似完全二叉树的结构，并同时满足堆性质：即子节点的键值或索引总是小于（或大于）它的父节点。堆排序通过构建最大堆（Max-Heap）或最小堆（Min-Heap），然后反复取出堆顶元素并调整堆来完成排序。

核心伪代码（最大堆实现升序排序）：
```
HEAPSORT(A)
    BUILD-MAX-HEAP(A)
    for i = A.length downto 2
        swap A[1] with A[i]
        A.heap-size = A.heap-size - 1
        MAX-HEAPIFY(A, 1)
```

其中 `BUILD-MAX-HEAP` 将无序数组构建成最大堆，`MAX-HEAPIFY` 维护堆性质。

## 二、核心原理
1. **堆结构**：堆是一棵完全二叉树，通常用数组表示。对于节点 `i`（1-indexed），其左子节点为 `2i`，右子节点为 `2i+1`，父节点为 `⌊i/2⌋`。

2. **堆性质**：最大堆中，每个节点的值都大于或等于其子节点的值，即 `A[PARENT(i)] ≥ A[i]`。最小堆则相反。

3. **建堆（Build-Heap）**：从最后一个非叶子节点（`⌊n/2⌋`）开始，自底向上对每个节点调用 `MAX-HEAPIFY`，确保以该节点为根的子树满足最大堆性质。时间复杂度为 O(n)。

4. **堆化（Heapify）**：当某个节点的值小于其子节点时，将其与较大的子节点交换，并递归地对交换后的子树执行相同操作，直到满足堆性质或到达叶子节点。

5. **排序过程**：建堆后，堆顶元素（最大值）与数组末尾元素交换，堆大小减1，然后对新的根节点执行 `MAX-HEAPIFY` 恢复堆性质。重复此过程直到堆大小为1，数组即有序。

## 三、过程/示例
以数组 `[4, 10, 3, 5, 1]` 为例，演示升序堆排序（最大堆）：

**初始数组**：`[4, 10, 3, 5, 1]`（索引从1开始）

**步骤1：建最大堆**
- 从 `i=⌊5/2⌋=2` 开始（值为10），其子节点为5和1，10≥5且10≥1，无需调整。
- `i=1`（值为4），子节点为10和3。10最大，交换4和10 → `[10, 4, 3, 5, 1]`
- 对交换后的位置 `i=2`（值为4）执行 `MAX-HEAPIFY`，子节点5和1，5最大，交换4和5 → `[10, 5, 3, 4, 1]`
- 建堆完成：`[10, 5, 3, 4, 1]`

**步骤2：排序**
- 交换堆顶10与末尾1 → `[1, 5, 3, 4, 10]`，堆大小减为4
- 对根节点（值为1）执行 `MAX-HEAPIFY`：子节点5和3，5最大，交换1和5 → `[5, 1, 3, 4, 10]`
- 对位置2（值为1）执行 `MAX-HEAPIFY`：子节点4，4最大，交换1和4 → `[5, 4, 3, 1, 10]`
- 交换堆顶5与末尾1（堆大小3）→ `[1, 4, 3, 5, 10]`
- 对根节点 `MAX-HEAPIFY`：子节点4和3，4最大，交换1和4 → `[4, 1, 3, 5, 10]`
- 交换堆顶4与末尾3（堆大小2）→ `[3, 1, 4, 5, 10]`
- 对根节点 `MAX-HEAPIFY`：子节点1，无需交换
- 交换堆顶3与末尾1（堆大小1）→ `[1, 3, 4, 5, 10]`
- 排序完成：`[1, 3, 4, 5, 10]`

## 四、复杂度/性质分析
- **时间复杂度**：
  - 最好情况：O(n log n)（所有元素相等时仍需堆化）
  - 最坏情况：O(n log n)（完全逆序或正序均相同）
  - 平均情况：O(n log n)
  - 建堆阶段 O(n)，每次 `MAX-HEAPIFY` 为 O(log n)，共 n-1 次，故总复杂度为 O(n log n)

- **空间复杂度**：O(1)（原地排序，仅需常数级额外空间）

- **稳定性**：不稳定。堆排序在交换堆顶与末尾元素时可能改变相等元素的相对顺序。例如 `[5a, 5b, 3]` 建堆后 `[5a, 5b, 3]`，交换5a与3后 `[3, 5b, 5a]`，5a移到了5b后面。

- **收敛性**：算法保证在有限步内终止，每次迭代至少将一个元素放到最终位置，最多 n-1 次迭代。

- **比较次数**：约为 2n log n + O(n)，比快速排序略多，但避免了最坏情况退化。

## 五、常见误区
1. **混淆堆与堆栈**：堆排序中的“堆”是数据结构（完全二叉树），与内存管理中的“堆栈”无关。

2. **误认为建堆是 O(n log n)**：建堆的严格时间复杂度为 O(n)，因为从下往上调整时，高度越低的节点数量越少，总工作量是线性的。

3. **忽略索引偏移**：实现时若数组从0开始索引，子节点公式应为 `2i+1` 和 `2i+2`，父节点为 `⌊(i-1)/2⌋`，初学者常在此出错。

4. **认为堆排序稳定**：堆排序是不稳定的，因为堆顶与末尾元素的交换可能破坏相等元素的原始顺序，这与归并排序等稳定算法不同。

5. **混淆最大堆与最小堆用途**：升序排序使用最大堆（每次取出最大值放末尾），降序排序使用最小堆。初学者常误以为升序需用最小堆。
