# CS Visual Wiki: 计算机科学交互式可视化百科平台
> 基于 Manim + 大语言模型打造的「知识可视化 + 交互式创作」在线教育平台，让抽象的数理与计算机知识，变成可交互、可编辑、可拆解的动态动画。

## 📖 项目简介
CS Visual Wiki 不再是一个单纯的「Manim 视频生成工具」，而是面向学生、开发者与教育者的**广义计算机科学可视化百科平台**。

传统在线教育以「被动观看视频」「静态文字阅读」为主，本项目通过大模型 + Manim 动画引擎，将抽象的算法、数学、通信原理等知识转化为可动态演示的可视化内容，并提供「划词生成 → 在线编辑 → AI 调试 → 渲染预览 → 社区分享」的完整闭环。用户既可以作为学习者浏览百科、直观理解知识点，也可以作为创作者修改动画、沉淀自己的可视化学习笔记。

项目愿景：用可视化技术降低硬核知识的认知门槛，打造一个「人人能看懂、人人能创作」的交互式知识社区。

## ✨ 核心特性
- **划词即可视化**：在百科词条中选中任意文本，一键生成对应 Manim 动画，上下文自动注入保证知识准确性
- **三栏式创作工作台**：左侧代码编辑器 + 右侧视频渲染区 + 底部 AI 对话面板，实现编辑、预览、调试无缝衔接
- **代码-动画双向锚定**：点击代码行自动跳转对应动画片段，拖动视频进度条同步高亮对应代码
- **RAG 增强的知识生成**：基于向量知识库的词条上下文注入，动画内容严格贴合百科定义，减少大模型幻觉
- **零代码参数调节**：支持通过滑块、下拉框修改动画速度、配色、参数范围，无需手写代码即可定制效果
- **结构化知识体系**：覆盖算法、数据结构、高等数学、线性代数、通信原理等领域，词条模块化设计，关联知识图谱
- **UGC 社区生态**：支持作品保存、Fork 衍生、社区发布、学习笔记创作，形成内容自运转飞轮
- **帧级动画调试**：支持单步前进/后退、断点暂停，逐帧拆解算法执行与数学推导过程

## 🧩 核心功能模块

### 1. 可视化百科知识库 (Visual Wiki)
平台的知识底座，替代传统静态文字百科，实现「定义 + 推导 + 可视化」三位一体。
- **结构化词条**：每个词条拆分为定义、核心原理、推导过程、典型示例、常见误区 5 个模块，每个模块支持独立生成可视化动画
- **划词触发生成**：选中文本后悬浮唤起「生成可视化」按钮，自动识别知识类型（算法过程/几何概念/公式推导）并匹配对应 Prompt 模板
- **知识图谱联动**：词条内关联概念自动添加超链接，底部推荐相关可视化内容，形成网状知识导航
- **多粒度动画**：内置 3 种动画规格——10s 概念示意、30s 过程演示、120s 完整推导，匹配不同学习场景
- **官方精选内容**：官方预先生产高频知识点的高质量动画，保证新用户开箱即有优质内容可浏览

### 2. 交互式创作工作台 (Interactive Workbench)
平台的核心交互单元，承接从百科跳转而来的创作需求。
- **三栏标准布局**
  - 左栏：基于 Monaco Editor 的在线 IDE，支持 Python 语法高亮、代码折叠、错误标红、行号显示
  - 右上栏：视频渲染区，支持一键渲染、进度条控制、单帧步进、倍速播放、导出 MP4/GIF
  - 右下栏：AI 对话面板，支持多轮对话修改代码，内置快捷指令（修复报错、放慢动画、添加标注、更换配色）
- **代码-动画双向锚定**：代码行与动画时间轴自动映射，点击代码跳转对应画面，拖动进度条高亮对应代码
- **动画步进调试器**：支持设置断点、单步执行，逐帧拆解算法步骤与推导过程
- **零代码参数面板**：可视化滑块调节核心参数（动画速度、元素数量、坐标轴范围、配色主题），系统自动同步修改代码
- **版本历史管理**：自动保存每次渲染的代码版本，支持一键回滚

### 3. 知识体系与学习路径
从「工具查询」延伸到「体系化学习」，覆盖备考与系统学习场景。
- **定向学习路径**：如「算法入门 40 讲」「线性代数可视化通关」「通信原理核心串讲」，将词条串联为体系化课程
- **备考专属专区**：针对考研数学、408 计算机统考等场景，提供经典教辅的动态化解析
- **每日一练打卡**：每日推送一个 30 秒以内的可视化知识点 + 思考题，适配碎片化学习节奏
- **交互式习题**：每节配套实操作业（如「修改动画演示冒泡排序优化版本」），提交后自动渲染校验

### 4. UGC 社区与内容生态
让用户成为内容生产者，构建内容飞轮。
- **词条变体机制**：用户修改后的动画可发布为对应词条的「社区变体」，同一知识点下沉淀多视角可视化内容
- **Fork 衍生机制**：一键复刻他人作品到个人工作台进行二次创作，自动保留原作者署名，类 GitHub 开源协作模式
- **交互式学习笔记**：支持插入官方/个人动画 + 文字讲解，创建可交互的学习笔记，支持公开分享与私密保存
- **社区广场**：按热度、评分、最新发布展示优质社区内容，支持点赞、收藏、评论
- **创作激励体系**：优质内容可获得平台积分、会员权益与现金分成，吸引高校创作者与专业讲师入驻

### 5. 个人空间与作品管理
- 个人作品库：保存所有生成与修改的动画项目，支持分类归档
- 收藏夹：收藏百科词条与社区优质作品
- 学习记录：自动记录浏览、学习、打卡进度，生成学习数据看板
- 作品分享：生成专属分享链接，支持外链直接播放动画

## 🏗️ 技术架构

### 技术栈总览
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + TypeScript + Monaco Editor + Pinia | 在线 IDE 与交互界面 |
| 后端 | FastAPI + Python 3.10+ | 高性能异步 API 服务 |
| AI 引擎 | DeepSeek Chat API + Ollama + ChromaDB | 大模型对话 + 本地 Embedding + 向量知识库 |
| 渲染引擎 | Manim Community v0.18.0 | 数学动画生成与渲染 |
| 任务队列 | Celery + Redis | 异步渲染任务调度，支持并发与超时控制 |
| 关系型数据库 | SQLite（开发）/ PostgreSQL（生产） | 用户、词条、作品等结构化数据存储 |
| 部署 | Docker + Nginx | 容器化部署与反向代理 |

### 核心数据流
1. 用户在百科页选中文本触发可视化请求 → 后端携带词条上下文调用 DeepSeek API 生成 Manim 代码
2. 代码返回前端，跳转工作台并自动填充到编辑器
3. 用户点击渲染 → 后端提交任务到渲染队列 → Manim 执行渲染 → 返回视频流与播放地址
4. 用户在对话面板发送修改指令 → 后端携带当前代码 + 历史对话调用大模型 → 返回修改后代码并同步到编辑器
5. 用户保存作品 → 结构化数据存入关系库，代码文件存入对象存储

## 🚀 快速开始

### 环境要求
- Python ≥ 3.10
- Node.js ≥ 18
- Ollama 本地服务（用于 Embedding，需拉取 `nomic-embed-text` 模型）
- Manim Community v0.18.0 及其运行依赖（FFmpeg 等）
- Redis（用于异步任务队列）
- DeepSeek API Key

### 本地部署步骤

#### 1. 克隆项目
```bash
git clone <项目仓库地址>
cd cs-visual-wiki
```

#### 2. 配置后端环境
```bash
cd backend

# 安装 Python 依赖
pip install -r requirements.txt

# 复制并填写环境配置
cp .env.example .env
# 编辑 .env，填入 DEEPSEEK_API_KEY、Redis 地址等配置
```

#### 3. 初始化知识库与数据库
```bash
# 初始化关系型数据库
python scripts/init_wiki_data.py

# 构建百科向量知识库
python scripts/build_kb.py
```

#### 4. 启动后端服务
```bash
# 启动主 API 服务
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 新开终端，启动渲染任务 Worker
celery -A app.services.render_worker worker --loglevel=info
```

#### 5. 启动前端服务
```bash
cd ../frontend

# 安装依赖
npm install

# 启动开发服务
npm run dev
```

启动完成后，访问 `http://localhost:5173` 即可进入平台。

## 🎯 核心使用流程
1. **浏览百科**：进入可视化百科首页，选择感兴趣的领域（算法/数学/通信），打开对应词条
2. **触发生成**：在词条正文中选中想要可视化的段落，点击悬浮的「生成可视化动画」按钮
3. **进入工作台**：代码生成完成后自动跳转工作台，左侧编辑器填充生成的 Manim 代码
4. **渲染预览**：点击右上区域的「开始渲染」按钮，等待渲染完成后在线播放动画
5. **AI 调试修改**：在右下对话框输入修改需求（如「放慢动画速度」「添加文字标注」），AI 自动修改代码并同步到编辑器
6. **手动编辑代码**：直接在左侧编辑器修改代码，支持语法高亮与错误提示，修改后可重新渲染
7. **保存与分享**：登录后可保存作品到个人空间，或发布到社区广场供其他用户浏览与 Fork

## 🛠️ 开发与扩展指南

### 新增百科词条
1. 在 `scripts/wiki_data/` 目录下创建 Markdown 格式的词条文件
2. 按照规范填写词条元信息（分类、标签、难度）与正文内容
3. 执行 `python scripts/build_kb.py` 更新向量知识库
4. 重启后端服务后即可在前端访问

### 扩展 Prompt 模板
项目内置多类知识的生成 Prompt，可在 `backend/app/utils/prompt_templates/` 下新增或优化：
- `algorithm_generation.tpl` 算法类动画生成模板
- `math_generation.tpl` 数学概念推导模板
- `code_fix.tpl` 代码修复调试模板
- 新增模板后在 `prompt_manager.py` 中注册即可自动匹配调用

### 自定义 Manim 组件库
平台内置通用 Manim 组件（坐标系、标注样式、通用动画模板），可在 `backend/app/utils/manim_components/` 中扩展：
- 封装高频复用的图形、动画、布局组件
- 生成代码时自动注入组件导入语句，提升生成代码的一致性与可维护性
