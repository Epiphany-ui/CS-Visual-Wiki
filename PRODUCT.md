# CS-Visual-Learn 产品业务文档
> 让抽象概念，动起来

---

## 一、产品概述

### 1.1 产品定位
CS-Visual-Learn 是一款**面向计算机科学与数学领域的交互式知识可视化学习平台**。基于 Manim 动画引擎与大语言模型技术，将教材中抽象的算法、数据结构、数学定理转化为直观可交互的动画，同时提供零代码/低代码的动画创作能力，打造"学习-理解-创作-分享"的完整闭环。

### 1.2 核心价值主张
- **对学习者**：告别"看书就懂，做题就废"，通过动画直观理解抽象概念，降低CS/数学的学习门槛
- **对创作者**：不需要掌握复杂的Manim语法，通过自然语言或模板参数，快速生成专业级教学动画
- **对教育者**：一键生成可视化教学素材，降低课件制作成本

### 1.3 目标用户画像
| 用户类型 | 核心需求 | 占比 |
|---------|---------|------|
| 计算机/数学专业大学生 | 理解课堂难点、备考复习 | 60% |
| 考研/转码备考者 | 针对性学习数据结构、算法、高数 | 20% |
| 编程/数学爱好者 | 直观理解有趣的概念（如傅里叶变换） | 15% |
| 教育工作者/UP主 | 快速制作教学动画素材 | 5% |

### 1.4 产品愿景
成为CS/数学领域的"可视化维基百科"，让每一个抽象概念都有对应的直观动画，让知识看得见。

## 三、核心功能模块详细说明

### 3.1 首页与发现
**定位**：用户进入产品的第一印象，快速找到感兴趣的内容

| 功能点 | 详细说明 |
|--------|---------|
| 顶部搜索栏 | 全局搜索，支持搜索知识点、动画、模板、用户 |
| 快速生成入口 | 大输入框，直接输入需求一键生成动画（核心转化入口） |
| 热门动画轮播 | 展示近期最火的高质量动画 |
| 分类导航 | 算法、数据结构、高等数学、线性代数、概率论、物理等分类入口 |
| 精选推荐 | 官方推荐的优质内容、新上架模板 |
| 最新作品 | 社区用户最新公开的作品 |

### 3.2 百科知识模块
**定位**：结构化的知识体系，可视化维基百科

| 功能点 | 详细说明 |
|--------|---------|
| 分类浏览 | 按学科分类（算法/数学/数据结构...）、按难度分级（入门/中等/困难） |
| 词条详情 | 标准五段式内容：定义→核心原理→过程示例→复杂度分析→常见误区 |
| 配套动画 | 每个词条绑定1~N个演示动画，看文字的同时直接看动画 |
| 相关词条推荐 | 基于知识图谱推荐前置知识、后续知识、相似概念 |
| 词条搜索 | 关键词模糊搜索，支持拼音、别名搜索 |
| 词条贡献 | 用户可以编辑改进词条，提交PR审核后合并 |
| 来源标注 | 所有词条标注来源（维基百科/教材/用户贡献） |

### 3.3 动画沙箱（核心创作工具）
**定位**：专业用户的创作工作台，三栏式布局

| 功能点 | 详细说明 |
|--------|---------|
| AI对话助手 | 左侧边栏，自然语言描述需求，AI生成/修改Manim代码，支持多轮对话 |
| 代码编辑器 | 中间栏，Monaco编辑器，语法高亮、自动补全、错误提示 |
| 实时预览 | 右侧栏，代码修改后一键渲染，实时预览动画效果 |
| 代码自动修复 | 渲染报错时，AI自动分析错误日志，一键修复代码 |
| 单步调试 | 支持逐帧播放、暂停、拖动进度条，代码与动画双向锚定 |
| 参数调节面板 | 可视化调节动画参数（速度、数组大小、颜色等），不用改代码 |
| 版本历史 | 自动保存代码版本，随时回退到历史版本 |
| 导出功能 | 导出MP4视频、GIF动图、Manim源码 |
| 保存/发布 | 保存到我的作品，或公开发布到社区/画廊 |

### 3.4 模板库（零代码创作）⭐
**定位**：降低创作门槛，普通人也能做动画

| 功能点 | 详细说明 |
|--------|---------|
| 模板分类 | 排序算法、树遍历、函数图像、概率分布、几何变换等分类 |
| 模板详情 | 模板介绍、效果预览视频、参数说明、使用次数 |
| 参数化生成 | 用户只需要填写可视化表单参数，一键生成动画 |
| 模板预览 | 调整参数后实时预览缩略效果 |
| 进阶编辑 | 从模板生成后，可以一键跳转到沙箱进行深度修改 |
| 用户贡献模板 | 优质创作者可以提交自己做的模板，官方审核后上架 |
| 模板排行榜 | 按使用次数、好评率排序 |

**第一批官方模板（MVP阶段）**：
1. 排序算法可视化（支持冒泡/选择/插入/快排/归并切换，可调数组大小、速度）
2. 一元函数图像绘制（输入任意函数表达式，可调坐标范围）
3. 二叉树遍历（前/中/后/层序遍历，自定义树结构）
4. 正态分布与中心极限定理（可调样本量、分布参数）
5. 傅里叶级数方波分解（可调项数）

### 3.5 精选画廊
**定位**：高质量内容展示厅，产品门面

| 功能点 | 详细说明 |

### 3.6 社区交流
**定位**：创作者与学习者的交流平台

### 3.7 备考学习系统
**定位**：面向应试场景的系统化学习路径

## 四、核心业务流程

### 4.1 学习流程（普通用户最常用）
```
用户进入首页 → 搜索知识点 → 阅读百科词条 → 播放配套动画
→ 拖动进度条反复观看 → 学习相关词条 → （可选）沙箱中调试参数 → 打卡完成
```

### 4.2 零代码创作流程（普通用户）
```
进入模板库 → 选择模板 → 填写参数 → 点击生成 → 预览调整
→ 导出/保存作品
```

### 4.3 AI创作流程（进阶用户）
```
进入沙箱 → 输入自然语言需求 → AI生成代码 → 渲染预览
→ AI自动修复错误/多轮修改 → 手动调整代码 → 保存发布
```

### 4.4 二次创作流程
```
画廊看到作品 → Fork到自己沙箱 → 修改代码/参数 → 重新渲染
→ 发布新作品（自动标注来源）
```

## 五、核心技术能力

1. **AI代码生成引擎**：RAG增强的Manim代码生成，多轮对话修改，自动错误修复
2. **Manim渲染引擎**：异步队列渲染，多质量支持，缓存机制，资源隔离
3. **RAG知识库系统**：百科+优质代码向量检索，增量更新，提升生成准确率
4. **参数化模板引擎**：占位符替换，参数校验，零代码生成
5. **异步任务调度**：任务状态管理，失败重试，进度推送，优先级队列

## 六、核心设计原则

1. **学习优先，创作其次**：首先是学习工具，其次才是创作工具
2. **零代码优先**：能用模板解决的不让用户写代码，能用自然语言解决的不让用户学语法
3. **内容质量优先**：官方内容严格把关，社区内容靠推荐算法筛选
4. **渐进式复杂度**：看动画→调参数→改模板→写代码，难度逐步提升
5. **核心引擎零修改**：ai_engine.py保持稳定，新功能在外层封装
